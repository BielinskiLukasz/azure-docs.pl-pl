# Azure Active Directory でのユーザーの作成または編集

Microsoft クラウド サービスにアクセス権を持つすべてのユーザーについて、Azure Active Directory (Azure AD) のアカウントを作成する必要があります。 また、ユーザー アカウントは変更したり、不要になった場合は削除したりできます。 既定では、ユーザーには管理者のアクセス許可はありませんが、必要に応じて割り当てることができます。

## ユーザーの作成

1. クリックして **Active Directory**, 、組織のディレクトリの名前をクリックします。
2.  **ユーザー** ] ページで [ **ユーザーの追加**します。
3.  **このユーザーについてお聞かせ** ] ページの **Type of User**, 、いずれかを選択します。
    1. **組織内の新しいユーザー** – 新しいユーザー アカウントを作成して、ディレクトリ内で管理することを示します。
    2. **既存の Microsoft アカウントを持つユーザー** – Microsoft アカウントを使用して Azure にアクセスする共同管理者と Azure リソースで共同作業するために、ディレクトリに既存の Microsoft アカウントを追加することを示します。
    3. **別の Azure AD ディレクトリ内のユーザー** – 別の Azure AD ディレクトリに属しているディレクトリにユーザー アカウントを追加することを示します。 その別のディレクトリ内のユーザーを選択するには、そのディレクトリのメンバーである必要があります。
4. 選択したオプションに応じて、ユーザー名、またはそのユーザーがサインインに使用する Microsoft アカウント名のどちらかを入力します。
5. ユーザーに対する **プロファイル** ] ページで、ユーザーの姓を提供して、姓、名、ユーザー フレンドリ名とロール] ドロップダウン メニューからユーザーのロール。 ユーザーと管理者の役割の詳細については、次を参照してください。 [Azure AD で管理者ロールの割り当て](active-directory-assign-admin-roles.md)します。 指定するかどうか **多要素認証を有効にする**です。
6.  **一時パスワードの取得** ] ページで [ **作成**します。

組織が複数のドメインを使用している場合は、ユーザー アカウントを作成するときに起きる、次の問題について理解しておく必要があります。

- 作成、たとえば、後で geoffgrisso@contoso.com geoffgrisso@contoso.onmicrosoft.com 場合は、ドメイン間で同じユーザー プリンシパル名 (UPN) を持つユーザー アカウントを作成できます。
- 後に geoffgrisso@contoso.onmicrosoft.com geoffgrisso@contoso.com を作成することはできません。

## ユーザーの編集

編集しようとしているユーザーがオンプレミスの Active Directory サービスと同期している場合、エラー メッセージが表示され、この手順を使用してユーザーを編集することができなくなります。 このユーザーを編集するには、ローカルの Active Directory 管理ツールを使用します。

Azure クラシック ポータルでユーザーを編集するには:

1. クリックして **Active Directory**, 、組織のディレクトリの名前をクリックします。
2.  **ユーザー** ] ページで、編集するユーザーの表示名をクリックします。
3. 変更が完了し、クリックして **保存**します。

## ユーザーのパスワードのリセット

1. クリックして **Active Directory**, 、組織のディレクトリの名前をクリックします。
2.  **ユーザー** ] ページで、編集するユーザーの表示名をクリックします。
3. クリックして、ポータルの下部にある、 **パスワードのリセット**します。
4. パスワードのリセットのダイアログで、クリックして **リセット**します。
5. チェック マークをクリックして、パスワードがリセットされたことを確認します。

## 外部ユーザーの作成

Azure AD では、別の Azure AD ディレクトリからのユーザーや、Microsoft アカウントを持つユーザーを、Azure AD ディレクトリに追加することもできます。 各ユーザーは最大 20 個までの異なるディレクトリのメンバーとなることができます。

別のディレクトリから追加されたユーザーは、外部ユーザーです。 外部ユーザーは、ディレクトリに既に存在するユーザーと、テスト環境などで、新しいアカウントと資格情報を使ってサインインしなくても共同作業することができます。 サインインしてのメンバーであるその他のすべてのディレクトリの認証の動作と、外部ユーザーは自分のホーム ディレクトリによって認証されます。

外部ユーザーを作成するには、ユーザーを Azure クラシック ポータルで、作成 **Type of User**, [ **別の Azure AD ディレクトリ内のユーザー**します。

## 外部ユーザーの管理と制限事項

あるディレクトリのユーザーを新しいディレクトリに追加すると、そのユーザーは新しいディレクトリで外部ユーザーとなります。 最初に、表示名とユーザー名がユーザーの「ホーム ディレクトリ」からコピーされ、追加先のディレクトリの外部ユーザーに設定されます。 それ以降は、外部のユーザー オブジェクトのすべてのプロパティは完全に独立したものになります。ホーム ディレクトリ内のユーザーに変更を加えても (ユーザーの名前の変更や役職の追加など)、それらの変更は追加先のディレクトリ内の外部ユーザー アカウントには反映されません。

2 つのオブジェクト間の関連は、ユーザーが常に自分のホーム ディレクトリにより、または Microsoft アカウントを使って認証されるということのみとなります。 この理由から、外部ユーザー アカウントにはパスワードをリセットするオプションや、多要素認証を有効にするオプションは表示されません。現在のところ、ユーザーのサインイン時に評価されるのは、ホーム ディレクトリまたは Microsoft アカウントの認証ポリシーのみです。

> [AZURE.NOTE]
> ディレクトリ内の外部ユーザーを無効にすることもできます。これによりディレクトリへのアクセスはブロックされます。

ユーザーがホーム ディレクトリ内で削除されたか、または Microsoft アカウントが取り消されたとしても、外部ユーザーはディレクトリ内に引き続き残ります。 ただし、それ以降は、ホーム ディレクトリでの認証や Microsoft アカウントを使った認証ができなくなるため、外部ユーザーはディレクトリ内のリソースにアクセスできません。

複数のディレクトリの管理者であるユーザーは、それらのディレクトリを Microsoft Azure クラシック ポータルで管理できます。 ただし、Office 365 などの他のアプリケーションでは、別のディレクトリ内の外部ユーザーとしてサービスを割り当てたりサービスにアクセスしたりすることは、現在のところできません。 将来的には、開発者に対して、開発するアプリを複数のディレクトリのメンバーであるユーザーに対応可能とするためのガイダンスを提供する予定です。

現在のところ次の点での制限事項があります。管理者が同意を与えることができるのは、ホーム ディレクトリ内のマルチ テナント アプリケーションに対してのみです。また、管理者は、ホーム ディレクトリのアクセス パネル経由で SaaS アプリおよび SSO に対してのみ、プロビジョニングできます。 Microsoft アカウント ユーザーにも、マルチテナント アプリケーションに同意を与えられない、またはアクセス パネルを使用できないという同じ制限事項があります。

## ゲスト

A **ゲスト** がユーザーの種類が「ゲスト」に設定されたディレクトリ内のユーザーです。 通常のユーザーの [ユーザーの種類 (User Type)] は、ディレクトリのメンバーであることを示す「メンバー」です。 ゲストは、自分のディレクトリの外部ユーザーとリソースを共有するときに作成されます。たとえば、Microsoft アカウントを Azure サブスクリプションに追加するときや、SharePoint で外部ユーザーと文書を共有するときなどに作成されます。

ゲストは、ディレクトリ内で限定された権限のセットを持ちます。 これらの権限では、ディレクトリ内の他のユーザーに関する情報を検出するゲストの能力は制限されますが、作業するリソースに関連付けられているユーザーやグループとの対話はできます。 たとえば、Azure サブスクリプションに割り当てられているゲストは、Azure サブスクリプションに関連付けられている他のユーザーやグループを参照できます。 サブスクリプションへのアクセス権が付与されている、ディレクトリ内の他のユーザーについて、完全な電子メール アドレスがわかっていれば、それらのユーザーを検索することもできます。 ゲストは、他のユーザーのプロパティについて、制限されたセットしか表示できません。 表示できるプロパティは、表示名、電子メール アドレス、ユーザー プリンシパル名 (UPN)、およびサムネイル写真に制限されます。

## ユーザー アクセス ポリシーの構成

 **構成** ディレクトリのタブには、外部ユーザーに関するアクセスを制御するオプションが含まれています。 これらのオプションを変更するには、ディレクトリのグローバル管理者が完全版の Azure クラシック ポータルの UI を使用する必要があります (Windows PowerShell や API では変更できません)。
開くには、 **構成** Azure クラシック ポータルでタブをクリックして **Active Directory**, 、ディレクトリの名前をクリックします。

![][1]

次に、外部ユーザーに関するアクセスを制御するオプションを編集できます。

![][2]

既定では、ゲストを列挙できない、ディレクトリの内容にユーザーやグループには表示されませんので、 **メンバー リスト**します。 ユーザーの完全なメール アドレスを入力してユーザーを検索してから、アクセス権を付与できます。 ゲストに関する既定の制限のセットは以下のとおりです。

- ディレクトリ内のユーザーやグループの列挙はできません。
- ユーザーのメール アドレスが分かっている場合は、そのユーザーの詳細情報の一部を表示できます。
- グループ名が分かっている場合は、そのグループの詳細情報の一部を表示できます。

ユーザーやグループの詳細情報の一部をゲストが表示できる機能を利用すると、他者を招待したり、連携者の詳細を表示したりできます。
