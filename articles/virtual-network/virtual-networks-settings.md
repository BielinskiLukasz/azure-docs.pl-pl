# Virtual Network (VNet) のプロパティの管理方法
VNet の設定を表示および変更するには、Azure ポータルを使用します。

## VNet  設定の表示

1. 管理ポータルにログオンします。

1. ナビゲーション ウィンドウで **ネットワーク**, で VNet の名前をクリックして、 **名前** 列です。

1. クリックして **ダッシュ ボード** を次の情報を表示します。

    - **仮想ネットワーク:** これは、仮想ネットワークとローカル ネットワークと DNS サーバーと共にサブネットです。

    - **データ、送信データ:** クロスプレミス接続がある場合に表示します。 これは仮想ネットワークで送受信されるデータです。

    - **ゲートウェイの IP アドレス:** ゲートウェイを構成した場合に表示します。 これはゲートウェイ IP アドレスです。 この IP アドレスを使用して、クロスプレミス接続用に VPN デバイスを構成します。

    - **リソース:** これらは、この仮想ネットワークに接続されている Azure のリソースです。

    - **概要:** を迅速に、次の情報を検索します。

        - **クライアント VPN パッケージのダウンロード:** このパッケージは、作成するときにポイント対サイト間 VPN 接続を使用します。 クライアントに関連するパッケージをダウンロードできます。

        - **VPN デバイス スクリプトのダウンロード:** サイト間クロスプレミス接続の VPN デバイスの構成に役立つこれらのテンプレートを使用します。

        - **status**

        - **サブスクリプション ID**
        
        - **仮想ネットワーク ID**
        
        - **location**

1. クリックして **構成** を次の情報を表示します。

    - **dns サーバー:** DNS サーバーと、対応する IP アドレス、仮想ネットワーク内のクライアントが名前解決に使用されますが表示されます。

    - **ポイント対サイト接続:** ポイント対サイト接続用に仮想ネットワークを構成した場合に表示します。

    - **サイト対サイト接続:** サイト対サイト接続用に仮想ネットワークを構成した場合に表示します。

    - **ローカル ネットワーク:** サイト対サイト接続があるかどうかはこれを示します。 これは選択されたローカル ネットワークで、オンプレミス サイトを示します。 このローカル ネットワークの設定を構成できます **ネットワーク** 上、 **ローカル ネットワーク** ページです。
    
    - **仮想ネットワーク アドレス空間:** アドレス空間と仮想ネットワークのサブネット。

1. クリックして **証明書** を次の情報を表示します。

    - **アップロードされたルート証明書:** ポイント対サイト VPN の構成
    
    - **証明書の拇印**
    
    - **証明書の有効期限**
    
    - **アップロード:** ルート証明書のアップロード
    
    - **[削除]:** 選択されているルート証明書の削除

## VNet 設定の編集

仮想ネットワークをデプロイした後は、特定の設定のみを変更できます。 次の手順で、管理ポータルを使用して仮想ネットワークの構成設定を編集できます。

1. 管理ポータルにログオンします。

1. ナビゲーション ウィンドウで **ネットワーク**, 、仮想ネットワーク名をクリックして、 **名前** 列です。

1. クリックして **構成**します。

    -  **Dns サーバー**, 、名前解決に DNS サーバーを追加することができます。 仮想ネットワークに仮想マシンをデプロイした後で DNS サーバーを追加する場合、新しく追加した DNS サーバーを仮想マシンが認識できるように、各仮想マシンを再起動する必要があります。
    
    -  **ポイント対サイト接続** ] セクションで、ポイント対サイト接続に対して指定するアドレス空間を構成することができます。
    
    -  **サイト間接続** ] セクションで、サイト間の設定を構成することができます。
    
    -  **仮想ネットワーク アドレス空間** ] セクションで、追加のアドレス空間とサブネットを追加することができます。 サブネットを追加するに **サブネットを追加** し、使用するアドレス空間を構成します。 クロスプレミス接続用のゲートウェイ サブネットを追加するには、クリックして **ゲートウェイ サブネットの追加**します。 仮想ネットワークに対してゲートウェイ サブネットを 1 つだけ追加できます。

1. クリックして **保存** 構成の変更を保存するには、画面の下部にあります。
