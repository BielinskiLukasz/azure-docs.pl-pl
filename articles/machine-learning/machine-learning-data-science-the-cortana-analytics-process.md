# Cortana Analytics Process (CAP) とは何か

Cortana Analytics Process (CAP) は、高度な分析技術を利用してインテリジェントなアプリケーションを構築する一連のステップを体系的に示したデータ サイエンス手法です。 キャップの手順は、 **ガイダンス** 問題を定義、関連するデータの分析、ビルドし、予測モデルを評価およびインテリジェントなアプリケーションでこれらのモデルを展開する方法についてです。 

次の手順を示します **Cortana Analytics Process**:  

![CAP ワークフロー](./media/machine-learning-data-science-the-cortana-analytics-process/CAP-workflow.png)

プロセスは、 **反復的な**: 理解新規および既存のモデルの微調整が進化し、順番に完了した手順のやり直しが必要ですか。 組織の既存の開発とプロジェクト計画のプロセスが **簡単に作成し直した** CAP 定義されている一連の手順を使用します。 

プロセスの手順が図を作成したり、リンク、 [CAP ラーニング パス](https://azure.microsoft.com/documentation/learning-paths/cortana-analytics-process/) と以下で説明します。  

## 準備ステップ 

## 1.  分析プロジェクトの計画を立てる 

分析プロジェクトへの取り組みで最初に行うことは、ビジネスの目標と問題を明らかにすることです。 観点で指定されている **ビジネス要件**します。 それらの要件を満たすために分析で予測すべき主要なビジネス変数 (売上予測、注文が詐欺である確率など) を特定することが、このステップの核となる目的です。 通常の理解を深めてするうえで不可欠で、追加の計画が、 **データ ソース** 分析の観点から、プロジェクトの目標に対処するために必要です。 たとえば、既存のシステムで問題を解決し、プロジェクトの目標を達成するために、従来とは異なる種類のデータを収集、記録しなければならないことはよくあります。 ガイダンスについては、次を参照してください。 [Cortana Analytics プロセスの環境を計画](machine-learning-data-science-plan-your-environment.md) と [Azure Machine Learning で高度な分析のシナリオ](machine-learning-data-science-plan-sample-scenarios.md)します。  

## 2.  分析環境のセットアップ 

Cortana Analytics Process の分析環境はいくつかの要素で構成されます。それらの構成要素を次に示します。 

- **データ ワークスペース** 分析とモデリングのため、データをステージング場所 
-  **処理インフラストラクチャ** 処理前の探索、およびデータのモデリング
-  **ランタイム インフラストラクチャ** を分析モデルを運用し、インテリジェントなクライアントに、モデルを使用するアプリケーションを実行します。  

セットアップすべき分析インフラストラクチャが帰属する環境は多くの場合、主要な運用システムからは独立しています。 しかし多くの場合、そのインフラストラクチャによって、社内だけでなく社外のさまざまなシステムから得られるデータを活用することができます。 分析インフラストラクチャには、純粋なクラウド環境、オンプレミス環境、または両者のハイブリッド環境があります。 オプションについては、次を参照してください。 [Cortana Analytics プロセスで使用するデータ サイエンス環境を設定](machine-learning-data-science-environment-setup.md)します。

## 分析ステップ:  

## 1.分析環境にデータを取り込む 

最初のステップは、データを処理できる分析環境に、社内または社外のさまざまなソースから関連データを取り込むことです。  **形式** ソースでデータの送信先で必要な形式から異なる場合があります。 したがって、取り込みツールで何らかのデータ変換が必要になる場合もあります。 オプションについては、次を参照してください [分析用のストレージ環境にデータを読み込む。](machine-learning-data-science-ingest-data.md)

だけでなく、初期のデータの取り込み、多くのインテリジェントなアプリケーションは、継続的な学習プロセスの一部としてデータを定期的に更新するために必要です。 設定してそのため、 **データ パイプライン** またはワークフロー。 ソリューションのデプロイ先となるインテリジェント アプリケーションには、その分析モデルの再構築と再評価を含む反復的なプロセスの一部として、このデータ パイプラインまたはワークフローが適用されます。 たとえば、 [Azure Data Factory を使って SQL azure、内部設置型 SQL server からデータを移動](machine-learning-data-science-move-sql-azure-adf.md)します。


## 2.データの調査と前処理 

次の手順は現在も調査して、データの理解を深めを取得するその **概要統計が表示** , 、リレーションシップ、および、このような手法を使用して **視覚エフェクト**します。 これの発行 **データ品質** と欠損値、データ型の不一致、一貫性のないデータのリレーションシップなどの整合性を解決します。 生データは、前処理変換でクリーンアップされてその後の分析やモデリングに使用されます。 詳細については、次を参照してください。 [拡張のデータを準備するタスクの機械学習](machine-learning-data-science-prepare-data.md)します。


## 3.データの特徴を明らかにする 

ドメイン専門家と共同でのデータの科学者には、データ セットの主要なプロパティをキャプチャして、計画中に特定の重要なビジネス変数を予測する最適な使用できる、機能を特定する必要があります。 こうした新たな特徴は、既存のデータから導くことができる場合もあれば、さらに別のデータを収集することが必要になる場合もあります。 このプロセスと呼ばれる **特徴エンジニア リング** 、効果的な予測分析システム構築の主な手順の 1 つです。 このステップでは、特定分野の専門知識とデータの調査ステップから得られた知見とを創造的に組み合わせる必要があります。 ガイダンスについては、次を参照してください。 [特徴エンジニア リング Cortana 分析プロセスで](machine-learning-data-science-create-features.md)します。


## 4.予測モデルを作成する 

プランニング ステップではビジネス要件を明らかにしました。そのビジネス要件によって特定された主要な変数は、データ サイエンティストが分析モデルを構築し、クリーニングと素性選択の済んだデータを使用して予測します。 機械学習システムをサポートして複数 **アルゴリズムをモデリング** をさまざまなケースに適用します。 ガイダンスについては、次を参照してください。 [Microsoft Azure Machine Learning のアルゴリズムを選択する方法](machine-learning-algorithm-choice,md)します。

データ サイエンティストは、予測タスクに最適なモデルを選択する必要があります。また、最適な結果を得るためには、複数のモデルから得られた結果を組み合わせなければならない場合も少なくありません。 モデリングに使用する入力データは通常、無作為に分割されて次の 3 つのデータセットが構築されます。

- トレーニング データセット 
- 検証データセット 
- テスト データセット 

使用してモデルを構築、 **トレーニング データ セット**します。 モデルを実行し、予測のエラーの測定で (チューニング パラメーター) を持つモデルの最適な組み合わせが選択されている、 **の検証データ セット**します。 最後に、 **テスト データ セット** 独立したデータのトレーニング用や、モデルの検証に使用されなかったことを選択したモデルのパフォーマンスを評価するために使用します。  手順については、次を参照してください。 [Azure Machine Learning でモデルのパフォーマンスを評価する方法](machine-learning-evaluate-model-performance.md)します。


## 5.モデルをデプロイして利用する 

できる一連の適切に動作するモデルを取得したら **計算** 他のアプリケーションを使用します。 ビジネス要件によって、作成した予測のいずれか **リアルタイム** または、 **バッチ** 単位です。 公開する必要は、モデルに計算する、 **API インターフェイスを開く** さまざまなアプリケーションから簡単に使用するこのようなオンライン web サイト、スプレッドシート、ダッシュ ボード、または基幹業務とバックエンド アプリケーションです。 参照してください [Azure Machine Learning web サービスを配置](machine-learning-publish-a-machine-learning-web-service.md)します。

## まとめと次のステップ

 [Cortana Analytics Process](https://azure.microsoft.com/documentation/learning-paths/cortana-analytics-process/) 反復処理されるステップのシーケンスとしてモデル化されている **ガイダンスを提供** 高度な分析を使用して、インテリジェントなアプリケーションをビルドするために必要なタスクにします。 見出したタスクをマイクロソフトの各種テクノロジを使って遂行する方法も、それぞれのステップから得られます。 

キャップが特定の種類の規定されていません、 **ドキュメント** 成果物、データの探索、モデリング言語と、評価の結果を文書化することをお勧めおよび分析できるようにすることに関連するコードを保存するために必要なときに反復処理します。 これによって、同様のデータや予測タスクを伴う他のアプリケーションに取り組むときに、同じ分析作業を再利用することができます。

完全なエンド ツー エンドのチュートリアルでのプロセス内のすべての手順を確認 **特定のシナリオ** も用意されています。 参照してください [の Cortana Analytics プロセスの操作: SQL Server を使用して](machine-learning-data-science-process-sql-walkthrough.md) と [の Cortana Analytics プロセスの操作: HDInsight Hadoop クラスターを使用して](machine-learning-data-science-process-hive-walkthrough.md)します。
