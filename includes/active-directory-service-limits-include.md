Oto ograniczenia używania i inne limity usługi dotyczące usługi Azure Active Directory (Azure AD).

| Kategoria | Limit |
| --- | --- |
| Katalogi | Jeden użytkownik może być członkiem lub gościem maksymalnie 500 katalogów usługi Azure AD.<br/>Jeden użytkownik może utworzyć maksymalnie 20 katalogów. |
| Domeny | Możesz dodać maksymalnie 900 nazw domen zarządzanych. Jeśli konfigurujesz wszystkie domeny federacyjne w celu federacji lokalnej z usługą Active Directory, możesz dodać maksymalnie 450 nazw domeny w każdym katalogu. |
|Resources |<ul><li>Użytkownicy bezpłatnej wersji usługi Azure Active Directory mogą utworzyć maksymalnie 50 000 zasobów usługi Azure AD w jednym katalogu. Jeśli masz co najmniej jedną zweryfikowaną domenę, domyślny przydział usługi Azure AD dla Twojej organizacji zostanie rozszerzony do 300 000 zasobów usługi Azure AD. Ten limit usługi nie jest związany z limitem warstwy cenowej 500 000 zasobów na stronie cennik usługi Azure AD. Aby wyjść poza domyślny przydział, należy skontaktować się z pomocą techniczną firmy Microsoft.</li><li>Użytkownik niebędący administratorem może utworzyć nie więcej niż 250 zasobów usługi Azure AD. Zarówno aktywne zasoby, jak i usunięte zasoby, które są dostępne do przywrócenia, są liczone do tego przydziału. Tylko usunięte zasoby usługi Azure AD, które zostały usunięte mniej niż 30 dni temu są dostępne do przywrócenia. Usunięte zasoby usługi Azure AD, które nie są już dostępne do przywrócenia, są liczone do tego przydziału o wartości jednej czwartej przez 30 dni. Jeśli masz programistów, którzy mogą wielokrotnie przekraczać ten przydział w trakcie swoich regularnych obowiązków, możesz [utworzyć i przypisać rolę niestandardową](../articles/active-directory/users-groups-roles/roles-quickstart-app-registration-limits.md) z uprawnieniami do tworzenia nieograniczonej liczby rejestracji aplikacji.</li></ul> |
| Rozszerzenia schematu |<ul><li>Rozszerzenia typu string mogą mieć maksymalnie 256 znaków. </li><li>Rozmiar rozszerzeń typu binary jest ograniczony do 256 bajtów.</li><li>Tylko 100 wartości rozszerzenia, *we wszystkich* typach i *wszystkich* aplikacjach, można zapisać do dowolnego pojedynczego zasobu usługi Azure AD.</li><li>Jednostki User, Group, TenantDetail, Device, Application i ServcicePrincipal można rozszerzyć za pomocą jednowartościowych atrybutów typu String lub Binary.</li><li>Rozszerzenia schematu są dostępne tylko w interfejsie API programu Graph w wersji 1.21 — wersja zapoznawcza. Aby zarejestrować rozszerzenie, należy przyznać aplikacji prawa dostępu do zapisu.</li></ul> |
| Aplikacje |Właścicielami jednej aplikacji może być maksymalnie 100 użytkowników. |
|Manifest aplikacji |W manifeście aplikacji można dodać maksymalnie 1200 wpisów. |
| Grupy |<ul><li>Użytkownik może utworzyć maksymalnie 250 grup w organizacji usługi Azure AD.</li><li>Organizacja usługi Azure AD może mieć maksymalnie 5000 grup dynamicznych.<li>Właścicielami jednej grupy może być maksymalnie 100 użytkowników.</li><li>Dowolna liczba zasobów usługi Azure AD może być członkami jednej grupy.</li><li>Użytkownik może być członkiem dowolnej liczby grup.</li><li>Liczba członków w grupie, których można zsynchronizować z lokalnej usługi Active Directory do usługi Azure Active Directory przy użyciu programu Azure AD Connect, jest ograniczona do 50 000 członków.</li><li>Grupy zagnieżdżone w usłudze Azure AD nie są obsługiwane we wszystkich scenariuszach</li></ul><br/> W tej chwili są to obsługiwane scenariusze z grupami zagnieżdżenia.<ul><li> Jedna grupa może zostać dodana jako członek innej grupy i można osiągnąć zagnieżdżanie grup.</li><li> Oświadczenia o członkostwie w grupie (gdy aplikacja jest skonfigurowana do odbierania oświadczeń o członkostwie grupy w tokenie, uwzględnione są grupy zagnieżdżone, których członkiem jest zalogowany użytkownik)</li><li>Dostęp warunkowy (podczas określania zakresu zasad dostępu warunkowego do grupy)</li><li>Ograniczanie dostępu do samodzielnego resetowania hasła</li><li>Ograniczanie użytkowników, którzy mogą do łączyć usługi Azure AD i rejestracji urządzenia</li></ul><br/>Następujące scenariusze NIE obsługuje grup zagnieżdżonych:<ul><li> Przypisanie roli aplikacji (przypisywanie grup do aplikacji jest obsługiwane, ale grupy zagnieżdżone w bezpośrednio przypisanej grupie nie będą miały dostępu), zarówno w celu uzyskania dostępu, jak i inicjowania obsługi administracyjnej</li><li>Licencjonowanie oparte na grupach (automatyczne przypisywanie licencji wszystkim członkom grupy)</li><li>Grupy usługi Office 365.</li></ul> |
| Serwer proxy aplikacji | <ul><li>Maksymalnie 500 transakcji na sekundę na aplikację proxy aplikacji</li><li>Maksymalnie 750 transakcji na sekundę dla organizacji usługi Azure AD</li></ul><br/>Transakcja jest definiowana jako pojedyncze żądanie http i odpowiedź dla unikatowego zasobu. Po ograniczeniu klienci otrzymają odpowiedź 429 (zbyt wiele żądań). |
| Panel dostępu |<ul><li>Nie ma limitu liczby aplikacji na użytkownika, które są widoczne w panelu dostępu. Dotyczy to przypisanych do użytkowników licencji usługi Azure AD Premium lub pakietu Enterprise Mobility Suite.</li><li>Panel dostępu zawiera maksymalnie 10 kafelków aplikacji dla każdego użytkownika. Ten limit dotyczy użytkowników, którym przypisano licencje dla planu licencji bezpłatnej usługi Azure AD. Przykładowe kafelki aplikacji Box, Salesforce i Dropbox. Ten limit nie dotyczy kont administratorów.</li></ul> |
| Raporty | W dowolnym raporcie można wyświetlić lub pobrać maksymalnie 1000 wierszy. Dodatkowe dane są obcinane. |
| Jednostki administracyjne | Zasób usługi Azure AD może być członkiem nie więcej niż 30 jednostek administracyjnych. |
| Role i uprawnienia administratora | <ul><li>Nie można dodać grupy jako [właściciela](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#object-ownership).</li><li>Nie można przypisać grupy do [roli](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles).</li><li>Nie można ograniczyć możliwości odczytu informacji katalogowych innych użytkowników poza przełącznikiem w całej organizacji usługi Azure AD, aby wyłączyć dostęp wszystkich użytkowników niebędących administratorami do wszystkich informacji katalogowych (nie zalecane). Więcej informacji na temat uprawnień domyślnych [tutaj](https://docs.microsoft.com/azure/active-directory/fundamentals/users-default-permissions?context=azure/active-directory/users-groups-roles/context/ugr-context#to-restrict-the-default-permissions-for-member-users).</li><li>Może upłynąć do 15 minut lub wylogowania się/zalogowania się, zanim nastąpią dodatki do członkostwa i odwołania roli administratora.</li></ul> |
