# サービス構成とプロファイルを管理する方法

## 概要

クラウド サービスを発行すると、Visual Studio は、サービス構成とプロファイルという 2 種類の構成ファイルに構成情報を保存します。 サービス構成 (.cscfg ファイル) には、Azure クラウド サービスのデプロイ環境の設定が保存されます。 Azure は、クラウド サービスを管理するときに、これらの構成ファイルを使用します。 その一方で、プロファイル (.azurePubxml ファイル) には、クラウド サービスの発行設定が保存されます。 この設定は、発行ウィザードで選択された内容の記録であり、Visual Studio によってローカルで使用されます。 ここでは、両方の種類の構成ファイルを使用する方法について説明します。

## サービス構成

複数のサービス構成を作成し、それらを各デプロイ環境に使用できます。 たとえば、ローカル環境用のサービス構成を作成して Azure アプリケーションの実行とテストに使用し、別のサービス構成を運用環境用に作成します。

これらのサービス構成は、要件に基づいて追加、削除、名前変更、および変更できます。 次の図に示すように、Visual Studio からこれらのサービス構成を管理できます。

![Manage Service Configurations](./media/vs-azure-tools-service-configurations-and-profiles-how-to-manage/manage-service-config.png)

開くことも、 **構成を管理** ロールのプロパティ ページから] ダイアログ ボックス。 Azure プロジェクトで、ロールのプロパティを開くには、そのロールのショートカット メニューを開くにして **プロパティ**します。  **設定** ] タブで、展開、 **サービス構成** 一覧をクリックして **管理** を開くには、 **構成を管理** ] ダイアログ ボックス。

### サービス構成を追加するには

1. ソリューション エクスプ ローラーで開き、Azure プロジェクトのショートカット メニューをクリックして **構成を管理**します。

     **サービス構成の管理** ] ダイアログ ボックスが表示されます。

1. サービス構成を追加するには、既存の構成のコピーを作成する必要があります。 これを行うには、構成を選択、名の一覧からコピーし、をクリックする **コピーを作成**します。

1. (省略可能)サービス構成に別の名前を付けて、名の一覧から新しいサービス構成を選択し、をクリックし、 **の名前を変更**します。  **名前** テキスト ボックスに、このサービス構成を使用して、クリックして使用する名前を入力 **OK**します。

    &Gt;.cscfg という新しいサービス構成ファイル。<New Name>.cscfg は、ソリューション エクスプ ローラーで Azure プロジェクトに追加されます。


### サービス構成を削除するには

1. ソリューション エクスプ ローラーで開き、Azure プロジェクトのショートカット メニューをクリックして **構成を管理**します。

     **サービス構成の管理** ] ダイアログ ボックスが表示されます。

1. サービス構成を削除するから削除する構成を選択、 **名** を一覧表示し、クリックして **削除**します。 この構成を削除することを確認するダイアログ ボックスが表示されます。

1. クリックして **削除**します。

     ソリューション エクスプローラーの Azure プロジェクトからサービス構成ファイルが削除されます。


### サービス構成の名前を変更するには

1. ソリューション エクスプ ローラーで開き、Azure プロジェクトのショートカット メニューをクリックして **構成を管理**します。

     **サービス構成の管理** ] ダイアログ ボックスが表示されます。

1. サービス構成の名前を変更するから新しいサービスの構成を選択、 **名** を一覧表示し、クリックして **の名前を変更**します。  **名前** テキスト ボックスに、このサービス構成を使用して、クリックして使用する名前を入力 **OK**します。

    ソリューション エクスプローラーの Azure プロジェクトでサービス構成ファイルの名前が変更されます。

### サービス構成を変更するには

- サービス構成を変更する場合は、Azure プロジェクトで変更するロールのショートカット メニューを開き、[プロパティ] をクリックします。 参照してください [方法: Visual Studio で Azure クラウド サービスの役割を構成する](https://msdn.microsoft.com/library/azure/hh369931.aspx) の詳細。

## プロファイルを使用してさまざまな設定を組み合わせる

プロファイルを使用すると、自動的に記入すること、 **発行ウィザード** さまざまな目的の設定の組み合わせが異なります。 たとえば、デバッグ用にプロファイルを 1 つ作成し、リリース ビルド用に別のプロファイルを作成します。 その場合は、 **デバッグ** プロファイルが必要があります **IntelliTrace** 有効になっていると **デバッグ** 選択すると、構成と **リリース** プロファイルが必要があります **IntelliTrace** 無効になっていると **リリース** 構成を選択しました。 また、別のプロファイルを使用して、別のストレージ アカウントでサービスをデプロイすることもできます。

初めてウィザードを実行すると、既定のプロファイルが作成されます。 Visual Studio を下にある Azure プロジェクトに追加 .azurePubXml の拡張子を持つファイルにプロファイルを格納する、 **プロファイル** フォルダーです。 後でウィザードを実行するときに手動で別の選択を指定すると、ファイルは自動的に更新されます。 次の手順を実行する前に、クラウド サービスを少なくとも 1 回発行する必要があります。

### プロファイルを追加するには

1. Azure プロジェクトのショートカット メニューを開き、をクリックし、 **発行**します。

1. 横に、 **profile を対象に** ] をクリックして、 **プロファイルの保存** として次の図は、ボタンをクリックします。 プロファイルが自動的に作成されます。

    ![Create an new profile](./media/vs-azure-tools-service-configurations-and-profiles-how-to-manage/create-new-profile.png)

1. プロファイルが作成されたら **< 管理… >** で、 **profile を対象に** ] ボックスの一覧です。

     **プロファイルの管理** として次の図は、ダイアログ ボックスが表示されます。

    ![Manage Profiles Dialog](./media/vs-azure-tools-service-configurations-and-profiles-how-to-manage/manage-profiles.png)

1.  **名前** ボックスの一覧をプロファイルを選択してクリックして **コピーの作成**します。

1. 選択、 **閉じる** ] ボタンをクリックします。

    新しいプロファイルが [ターゲット プロファイル] ボックスの一覧に表示されます。

1.  **Profile を対象に** 一覧で、先ほど作成したプロファイルをクリックします。 選択したプロファイルで選択されている内容が、発行ウィザードの設定項目に入力されます。

1. クリックして、 **前** と **次** 発行ウィザードの各ページを表示し、このプロファイルの設定をカスタマイズするボタンです。 参照してください [Azure アプリケーションの発行ウィザード](http://go.microsoft.com/fwlink/p/?LinkID=623085) についてです。

1. 設定のカスタマイズが完了したら、クリックして **次** の設定] ページに戻ります。 これらの設定を使用して、サービスを発行するとき、またはクリックした場合、プロファイルが保存した **保存** プロファイルの一覧の横にあります。

### プロファイルを削除するには

1. Azure プロジェクトのショートカット メニューを開き、をクリックし、 **発行**します。

1.  **Profile を対象に** 一覧で、クリックして **管理**します。

1.  **プロファイルの管理** ] ダイアログ ボックスで、削除、およびをクリックするプロファイルをクリックして **削除**します。

1. [確認] ダイアログ ボックスが表示されますが、クリックして **OK**します。

1. クリックして **閉じる**します。

### プロファイルを変更するには

1. Azure プロジェクトのショートカット メニューを開き、をクリックし、 **発行**します。

1.  **Profile を対象に** リストを変更するプロファイルをクリックします。

1. をクリックして、 **前** と **次** を発行ウィザードの各ページを表示し、必要な設定を変更します。 参照してください [Azure アプリケーションの発行ウィザード](http://go.microsoft.com/fwlink/p/?LinkID=623085) についてです。

1. 設定の変更が完了したら、クリックして **次** に戻る、 **設定** ページです。

1. (省略可能)クリックして **発行** 新しい設定を使用してクラウド サービスを発行します。 この時点でクラウド サービスを発行せずに発行ウィザードを閉じると、変更をプロファイルに保存するかどうかをたずねるメッセージが表示されます。
