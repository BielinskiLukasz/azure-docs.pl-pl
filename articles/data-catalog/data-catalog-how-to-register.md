# データ ソースの登録方法

## はじめに
**Microsoft Azure Data Catalog** 登録のシステムとエンタープライズ データ ソースの検出のシステムとして機能する完全に管理されたクラウド サービスです。 つまり、 **Azure Data Catalog** こそすべての検出、理解、およびデータ ソースを使用して、ユーザーと企業が既存のデータから複数の値を取得します。 データ ソースを使用して探索可能にするための第一歩と **Azure Data Catalog** は、そのデータ ソースを登録します。
## データ ソースの登録
登録は、データ ソースからメタデータを抽出しには、そのデータをコピーするプロセス、 **Azure Data Catalog** サービスです。 データは現在存在する場所に残り、現在のシステムの管理者とポリシーの制御下に留まります。

データ ソースを登録するには、単に起動、 **Azure Data Catalog** からのデータ ソース登録ツール、 **Azure Data Catalog** ポータルです。 職場または学校アカウント (ポータルへのログインに使用するのと同じ Azure Active Directory 資格情報) でログインし、登録するデータ ソースを選択します。
参照してください、 [Azure Data Catalog を使ってみる](data-catalog-get-started.md) 詳細な手順についてのチュートリアルです。

データ ソースが登録されると、Catalog はその場所を追跡し、メタデータのインデックスを作成して、ユーザーがデータ ソースを検索、参照、検出し、その場所を使用して任意のアプリケーションまたはツールで接続できるようにします。

## サポートされているソース
現在のプレビューで **Azure Data Catalog** これらのデータ ソースとオブジェクトの種類の登録をサポートします。

* SQL Server Database Engine のテーブルとビュー
* Oracle データベースのテーブルとビュー
* SQL Server Analysis Services Multidimensional Dimension、Measure、KPI
* SQL Server Analysis Services 表形式テーブル
* SQL Server Reporting Services レポート
* Azure Storage BLOB およびディレクトリ
* Azure Data Lake Store
* HDFS のファイルとディレクトリ
* Apache Hive テーブル
* Teradata テーブルと Teradata ビュー

> [AZURE.NOTE] SQL Server のサポートは、Microsoft Azure SQL Database および Microsoft Azure SQL Data Warehouse にも含まれます。

<br/>

> [AZURE.NOTE] SQL Server Reporting Services のサポートは、ネイティブ モード サーバーのみに – SharePoint モードはサポートされていません

<br/>


## 構造メタデータ
データ ソースを登録するとき、登録ツールは選択されたオブジェクトの構造に関する情報を抽出します。これは、構造メタデータと呼ばれます。

すべてのオブジェクトについて、この構造メタデータには、データを検出するユーザーがその情報を使用して、任意のクライアント ツールのオブジェクトに接続できるように、オブジェクトの場所が含まれます。 他の構造メタデータとしては、オブジェクトの名前と型、属性/列の名前、データの型などがあります。

## 記述メタデータ
データ ソースから抽出されるコア構造メタデータに加えて、データ ソース登録ツールは記述メタデータも抽出します。 SQL Server Analysis Services および SQL Server Reporting Services の場合、これはこれらのサービスによって公開される Description プロパティから取得され+ます。 SQL Server の場合は、ms_description 拡張プロパティを使用して提供される値が抽出されます。 Oracle データベースでは、データ ソース登録ツールは ALL_TAB_COMMENTS ビューから COMMENTS 列を抽出します。

データ ソースから抽出される記述メタデータに加えて、ユーザーはデータ ソース登録ツールを使用して記述メタデータを入力することもできます。 ユーザーはタグを追加でき、登録されているオブジェクトの専門家を識別できます。 すべてこの記述メタデータをコピー、 **Azure Data Catalog** 、構造メタデータと共にサービスです。

## プレビューの組み込み

既定では、メタデータのみがデータ ソースから抽出し、にコピー、 **Azure Data Catalog** が、データ ソースが含まれているデータのサンプルを見ることによって容易に多くの場合、なりますについて理解します。

 **Azure Data Catalog** データ ソース登録ツールにより、ユーザーを各テーブルおよび登録されているビューのデータのスナップショット プレビューを含めます。 ユーザーが登録の間にプレビューの組み込みを選択した場合、登録ツールは各テーブルおよびビューから最大 20 個のレコードを組み込みます。 このスナップショットは、構造メタデータおよび記述メタデータと共に Catalog にコピーされます。


> [AZURE.NOTE]  幅の広いテーブルの列の数が多い、プレビューに含まれている 20 未満のレコードがあります。


## データ プロファイルの組み込み

プレビューの組み込みと同じよう役に立つのコンテキストが提供のユーザーのデータ ソースの検索 **Azure Data Catalog**, 、データ プロファイルを含むことができますも置くことが検出されたデータ ソースを理解します。

 **Azure Data Catalog** データ ソース登録ツールにより、ユーザーを各テーブルおよび登録されているビューのデータ プロファイルを含めます。 ユーザーが登録時にデータ プロファイルの取り込みを選択すると、登録ツールは各テーブルとビューのデータに関する次のような集計情報を組み込みます。

* オブジェクト内の行数とデータのサイズ
* データとオブジェクト スキーマに対して最新の更新が行われた日付
* 列における null レコードの数と重複しない値の数
* 列の最小値、最大値、平均値、および標準偏差値

これらの統計は、構造メタデータおよび記述メタデータと共に Catalog にコピーされます。

> [AZURE.NOTE]  テキストと日付の列では、データ プロファイルでの平均や標準偏差の統計情報は含まれません。


## 登録の更新

データ ソースの登録が検出できるようにで、 **Azure Data Catalog** メタデータおよび登録時に抽出オプションのプレビューを使用します。 Catalog のデータ ソースを更新する必要がある場合は (たとえば、オブジェクトのスキーマが変更された場合、もともと除外されていたテーブルを含める必要がある場合、ユーザーがプレビューに含まれるデータを更新したい場合など)、データ ソース登録ツールを再実行できます。

既に登録されているデータ ソースを再登録すると、マージ "upsert" 操作が実行されます。既存のオブジェクトは更新され、新しいオブジェクトは作成されます。 使用してユーザーが指定されている任意のメタデータ、 **Azure Data Catalog** ポータルは維持されます。

## 概要
データ ソースの登録 **Azure Data Catalog** 、そのデータ ソースを簡単に検出および理解、Catalog サービスにデータ ソースから構造的でわかりやすいメタデータをコピーしています。 データ ソースが登録されると、その注釈を付けることができますし、管理、および使用して検出された、 **Azure Data Catalog** ポータルです。
