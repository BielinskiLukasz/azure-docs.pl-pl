# 名前付き認証資格情報の設定

Visual Studio から Azure にアプリケーションを発行したり、既存のクラウド サービスを監視するには、Azure への要求を認証するために使用できる資格情報を指定する必要があります。 Visual Studio には、こうした資格情報を指定するためのサインインのインターフェイスがいくつか設けられています。 たとえば、サーバー エクスプ ローラーからすることができます、ショートカット メニューを開き、 **Azure** ノードを選択して **Azure への接続**します。 サインインすると、Azure のアカウントに関連付けられたサブスクリプション情報が Visual Studio で利用できるようになるため、それ以上の設定は不要です。

Azure Tools では、サブスクリプション ファイル (.publishsettings ファイル) を使用して資格情報を指定する従来の方法もサポートされます。 このトピックでは、Azure SDK 2.2 で現在もサポートされているこの方法について説明します。

Azure に対する認証には、次の項目が必要です。

- サブスクリプション ID

- 有効な X.509 v3 証明書

>[AZURE.NOTE] X.509 v3 証明書のキーの長さは 2048 ビット以上である必要があります。 この要件を満たさない証明書や無効な証明書は、Azure によってすべて拒否されます。

Visual Studio では、サブスクリプション ID が証明書データと共に資格情報として使用されます。 該当する資格情報はサブスクリプション ファイル (.publishsettings ファイル) 内で参照され、このファイルに証明書の公開キーが格納されます。 サブスクリプション ファイルには、複数のサブスクリプションの資格情報を含めることができます。

サブスクリプション情報を編集することができます、 **サブスクリプションの新規/編集** ダイアログ ボックスで、このトピックで後述します。

証明書を作成する場合は、ことができます」の手順を参照してください。 [の作成と Azure の管理証明書をアップロード](https://msdn.microsoft.com/library/windowsazure/gg551722.aspx) 管理ポータルに証明書を手動でアップロードします。

>[AZURE.NOTE] Visual Studio は、クラウド サービスの管理に必要となるこれらの資格情報では、Azure ストレージ サービスに対する要求を認証するために必要な同じ資格情報はありません。

## Visual Studio の認証資格情報の変更またはエクスポート

設定、変更、またはエクスポートできますで、認証資格情報、 **新しいサブスクリプション** ] ダイアログ ボックスは、次の操作のいずれかの操作を行う場合に表示されます。

-  **サーバー エクスプ ローラー**, 、ショートカット メニューを開き、 **Azure** ] ノードを選択 **サブスクリプションの管理**, 、選択、 **証明書** タブをクリックし、選択、 **新規** または **編集** ] ボタンをクリックします。

- Azure クラウド サービスを発行すると、 **Azure アプリケーションの発行** ウィザードで、選択 **管理** で、 **サブスクリプションを選択する** を一覧表示し、証明書] タブを選択し、[、 **新規** または **編集** ] ボタンをクリックします。

次の手順を前提としていますが、 **新しいサブスクリプション** ] ダイアログ ボックスが開いています。

### Visual Studio で認証資格情報を設定するには

1.  **既存の証明書を選択して** 認証一覧については、証明書を選択します。

1. 選択、 **完全パスをコピー** ] ボタンをクリックします。証明書 (.cer ファイル) のパスは、クリップボードにコピーされます。

    >[AZURE.IMPORTANT] Visual Studio から Azure アプリケーションを発行するには、管理ポータルにこの証明書をアップロードする必要があります。

1. 証明書を Microsoft Azure 管理ポータルにアップロードするために、次の手順を実行します。

    1. [Azure ポータル] リンクを選択します。

          [Azure 管理ポータル](http://go.microsoft.com/fwlink/?LinkID=213885) が開きます。

    1. クリックして、Microsoft アカウントを使用して、Azure 管理ポータルにサインイン、 **Cloud Services** ] ボタンをクリックします。

    1. 必要なクラウド サービスを選択します。

        そのサービスのページが表示されます。

    1.  **証明書** ] タブで、選択、 **アップロード** ] ボタンをクリックします。

    1. 作成した .cer ファイルのフル パスを貼り付け、指定したパスワードを入力します。
