# <a name="classic-deployment-model"></a>Klasyczny Model wdrażania

Klasyczny model wdrażania jest starszej generacji ziemi tryb wdrażania na platformie Azure wymusza vCPU globalnego limitu przydziału dla maszyn wirtualnych i zestawów skalowania maszyn wirtualnych. Klasycznego modelu wdrażania nie jest już zalecane i teraz zostało zastąpione przez model usługi Resource Manager. Aby dowiedzieć się więcej na temat tych dwóch wdrażania modeli i korzystać z usługi Resource Manager, zobacz strony modelu wdrażania usługi Resource Manager Po utworzeniu nowej subskrypcji jest do niej przypisany domyślny limit przydziału procesorów wirtualnych. W dowolnym momencie nowej maszyny Wirtualnej jest można wdrożyć przy użyciu klasycznego modelu wdrażania, sumę użycia nowych i istniejących wirtualnych procesorów CPU, we wszystkich regionach nie może przekraczać limit przydziału procesorów wirtualnych, zatwierdzone dla klasycznego modelu wdrażania. Dowiedz się więcej na temat limitów przydziału w subskrypcji platformy Azure i strona limity usługi.

Możesz poprosić o zwiększenie limitu procesorów wirtualnych dla klasycznego modelu wdrażania przy użyciu pomoc + bloku pomocy technicznej lub użycia + przydziału bloku, w portalu.
