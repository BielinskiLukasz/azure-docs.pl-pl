<a name="seller-insights-definitions"></a>Definicje informacji o sprzedawcy
=======================

Poniższa tabela zawiera definicje dotyczące wielu terminów używanych w usłudze informacje o sprzedaży.

|  **Mandat**                 |  **Definicja**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Typ licencji platformy Azure**                               | Typ umowy licencyjnej używanej przez klientów do kupowania platformy Azure.  Znany również jako kanał.                                                  |
| **Typ licencji platformy Azure: dostawca rozwiązań w chmurze**      | Klient końcowy nabywa platformę Azure i ofertę portalu Marketplace za pomocą swojego dostawcy rozwiązań w chmurze, który działa jako odsprzedawca.                 |
| **Typ licencji platformy Azure: Enterprise**                   | Klient końcowy nabywa platformę Azure i ofertę w portalu Marketplace za pomocą Umowa Enterprise podpisanej bezpośrednio w firmie Microsoft.                  |
| **Typ licencji platformy Azure: przedsiębiorstwo przez odsprzedawcę**  | Klient końcowy nabywa platformę Azure i ofertę portalu Marketplace przez odsprzedawcę, który ułatwia ich Umowa Enterprise firmie Microsoft.     |
| **Typ licencji platformy Azure: płatność zgodnie z rzeczywistym użyciem**                | Klient końcowy nabywa platformę Azure i ofertę Marketplace w ramach umowy "płatność zgodnie z rzeczywistym użyciem" podpisanej bezpośrednio z firmą Microsoft.                |
| **Kwota opłaty (DW)**                              | Kwota naliczana dla klienta w *walucie klienta* (DW) dla rozliczeń.                                                                 |
| **Kwota opłaty (komputer)**                               | Kwota naliczana dla klienta w *walucie wypłaty* (komputer).                                                                      |
| **Data opłaty**                                      | Data obliczenia opłaty klienta (zazwyczaj bezpośrednio po okresie użytkowania).                                             |
| **Nazwa wystąpienia chmury**                              | Microsoft Cloud, w którym wystąpiło wdrożenie maszyny wirtualnej.                                   |
| **Nazwa wystąpienia chmury: globalna Azure**                | Publiczna globalna chmura firmy Microsoft.                           |
| **Nazwa wystąpienia chmury: Azure Government**        | Specyficzne dla instytucji rządowe chmury firmy Microsoft dla jednego z następujących rządów: Chiny, Niemcy lub Stany Zjednoczone Ameryki.                           |
| **Stan kolekcji**  | W przypadku określonej opłaty jest najnowszy stan cyklu rozliczeń i zbierania danych.  Na przykład stan może być zbieranie w toku, zbierane lub zwrotne.                  |
| **Rozmiar podstawowy**  | Liczba rdzeni wirtualnych prezentowanych przez maszynę wirtualną, na której jest uruchomiona oferta.               |
| **Miasto klienta**  | Nazwa miasta podana przez klienta. Może to być inne niż miasto w subskrypcji platformy Azure klienta.  |
| **Język komunikacji klienta**        | Język preferowany przez klienta do komunikacji.                                  |
| **Nazwa firmy klienta**                  | Nazwa firmy podana przez klienta. Może to być inne niż miasto w subskrypcji platformy Azure klienta.                                  |
| **Kraj klienta**                       | Nazwa kraju/regionu podana przez klienta.  Może to być inne niż kraj/region w subskrypcji platformy Azure klienta.                               |
| **Waluta klienta (DW)**                 | Waluta preferowana przez klienta na potrzeby cen i rozliczeń.                            |
| **Adres E-mail klienta**                         | Adres e-mail dostarczony przez klienta końcowego.  Może to być inne niż adres e-mail w subskrypcji platformy Azure klienta.                   |
| **Imię klienta**                              | Imię i nazwisko dostarczone przez klienta.  Może się to różnić od nazwy podanej w subskrypcji platformy Azure klienta.                    |
| **Nazwisko klienta**                               | Nazwisko dostarczone przez klienta.  Może się to różnić od nazwy podanej w subskrypcji platformy Azure klienta.                    |
| **Identyfikator klienta**                               | Unikatowy identyfikator przypisany do klienta.  Klient może mieć zero lub więcej subskrypcji portalu Azure Marketplace.                    |
| **Typ płatności klienta**                            | Typ instrumentu płatniczego używany przez klienta.  Na przykład może to być karta, faktura lub sprawdzanie elektroniczne.                             |
| **Kod pocztowy klienta**                             | Kod pocztowy dostarczony przez klienta.  Może to być inne niż kod pocztowy podany w subskrypcji platformy Azure klienta.                                                                 |
| **Stan klienta**                                   | Stan (adres) dostarczony przez klienta.  Może to być inne niż nazwisko podane w subskrypcji platformy Azure klienta.                                                                 |
| **Data pozyskania**                                    | Pierwszy dzień zakupionej oferty przez klienta.                 |
| **Data utraty**                                        | Ostatnia data anulowania ostatniego zakupionej oferty przez klienta.                                                         |
| **Szacowana opłata rozszerzona (DW)**                   | Szacowana rozszerzona opłata za liczbę jednostek użytkowania dla danej jednostki SKU (w walucie klienta). Ta wartość nie może być dokładna ze względu na błędy zaokrągleń lub obcinania.                            |
| **Szacowana opłata rozszerzona (komputer)**                   | Szacowana rozszerzona opłata za liczbę jednostek użytkowania dla danej jednostki SKU w oparciu o konwersję wymiany walut na użycie jest obliczana (w walucie wydawcy). Ta wartość nie może być dokładna ze względu na błędy zaokrągleń lub obcinania.                  |
| **Szacowana wypłata (komputer)**                            | Szacowana płatność dla liczby jednostek użycia dla danej jednostki SKU, oparta na konwersji wymiany walutowej na datę obliczenia użycia (w walucie wydawcy). Ta wartość nie może być dokładna ze względu na błędy zaokrągleń lub obcinania.                        |
| **Szacowana cena (komputer)**                             | Szacowana cena jednostki użycia dla danej jednostki SKU, oparta na konwersji wymiany walutowej na użycie, jest obliczana (w walucie wydawcy). Ta wartość nie może być dokładna ze względu na błędy zaokrągleń lub obcinania.                                             |
| **Jest nowym klientem**                                  | Wartość informuje o tym, czy jest to nowy klient zużywający lub Kupujący jedną lub większą liczbę ofert w portalu Marketplace po raz pierwszy (lub nie).  Wartość będzie równa "yes", jeśli jest to ten sam miesiąc kalendarzowy dla "Data pozyskania".  Ta wartość będzie równa "No", jeśli klient kupił dowolne oferty przed zgłoszonym miesiącem kalendarzowym.                                                       |
| **Jest jednostką SKU wersji zapoznawczej**                                  | Wartość będzie informować o tym, czy jest oznaczona jako wersja zapoznawcza. Wartość będzie równa "yes", jeśli jednostka SKU została odpowiednio oznaczona, a tylko subskrypcje platformy Azure autoryzowane przez użytkownika, można wdrożyć i korzystać z tego obrazu.  Wartość będzie równa "No", jeśli jednostka SKU nie została zidentyfikowana jako "wersja zapoznawcza".                                          |
| **Czy kontakt promocyjny jest wybierany**                   | Ta wartość informuje o tym, czy klient aktywnie zabrał w celu skontaktowania się z wydawcami. W tej chwili nie prezentujemy opcji klientom, dlatego w całej tablicy podano wartość "No". Po wdrożeniu tej funkcji rozpocznie się aktualizowanie.                                          |
| **Typ licencji Marketplace**                         | Metoda rozliczeń oferty portalu Marketplace.                             |
| **Typ licencji Marketplace: rozliczane za pomocą platformy Azure**   | Firma Microsoft jest Twoim agentem dla tej oferty w witrynie Marketplace i wekslom klientów w Twoim imieniu. (Karta kredytowa PAYG lub faktura w przedsiębiorstwie)       |
| **Typ licencji Marketplace: korzystanie z własnej licencji** | Maszyna wirtualna wymaga klucza licencji dostarczonego przez klienta do wdrożenia. Firma Microsoft nie nalicza opłat za użycie ofert w portalu Marketplace. |
| **Typ licencji Marketplace: bezpłatnie**                   | Oferta portalu Marketplace jest konfigurowana bezpłatnie dla wszystkich użytkowników. Firma Microsoft nie nalicza opłat za użycie tej oferty w portalu Marketplace.    |
| **Typ licencji Marketplace: Firma Microsoft jako odsprzedawca**  | Firma Microsoft jest odsprzedawcą dla tej oferty portalu Marketplace.                               |
| **Nazwa firmy dla subskrypcji witryny Marketplace**  | Nazwa firmy podana przez klienta w ramach subskrypcji platformy Azure.                           |
| **Identyfikator subskrypcji witryny Marketplace**            | Unikatowy identyfikator skojarzony z subskrypcją platformy Azure używany przez klienta do zakupu oferty portalu Marketplace.  Wcześniej był to identyfikator GUID subskrypcji platformy Azure.  W niektórych przypadkach, takich jak kolejność SAAS, subskrypcja nie jest wymagana.           |
| **Opłata firmy Microsoft (DW)**                               | Opłata od firmy Microsoft w odniesieniu do transakcji w walucie klienta.                              |
| **Nazwa oferty**                                | Nazwa oferty portalu Marketplace.                                                             |
| **Typ oferty**                                | Typ oferty Microsoft Marketplace.  Na przykład może to być aplikacja zarządzana, licencje lub kontenery maszyn wirtualnych.                           |
| **Data anulowania zamówienia**                         | Data anulowania zamówienia witryny Marketplace.                                                       |
| **Identyfikator zamówienia**                                  | Unikatowy identyfikator zamówienia klienta usługi portalu Marketplace.  Oferty oparte na użyciu maszyny wirtualnej nie są skojarzone z kolejnością.                 |
| **Zamówienie daty zakupu**                       | Data utworzenia zamówienia witryny Marketplace.                                                        |
| **Zamówiona ilość**                       | Liczba zamówień dla tej samej jednostki SKU skojarzonej z subskrypcją platformy Azure.  W przypadku zamówień SAAS reprezentuje liczbę stanowisk skojarzonych z tym wystąpieniem zamówienia.                                            |
| **Stan zamówienia**                              | Stan zamówienia witryny Marketplace w momencie ostatniego odświeżenia danych.  Na przykład zamówienie może być aktywne lub anulowane.             |
| **Stan zamówienia: aktywny**                           | Klient kupił zamówienie i nie anulował jego zamówienia.   |
| **Stan zamówienia: anulowane**                        | Klient kupił wcześniej zamówienie, a następnie anulował jego zamówienie.        |
| **Kwota wypłaty (komputer)**                             | Kwota zapłacona dla Ciebie w walucie preferowanej wypłaty (komputer).                                       |
| **Waluta wypłaty (komputer)**                           | Waluta użyta do wypłat.                                                                |
| **Data wypłaty**                                    | Data wysłania żądania płatności od firmy Microsoft do wybranej instytucji finansowej.                                                                         |
| **Stan wypłaty**                                  | Stan wypłaty w momencie ostatniego odświeżenia danych.                                   |
| **Stan wypłaty: w toku**               | Transakcja nie jest gotowa do wypłaty. (Aby uzyskać więcej informacji, zobacz stan kolekcji)                             |
| **Status wypłaty: Wypłata**                          | Transakcja została uwzględniona w obliczeniu ostatniej wypłaty. Wartości dodatnie są płatne, a wartości ujemne są netted od łącznej kwoty należnej. |
| **Stan wypłaty: nadchodząca wypłata**                   | Transakcja jest gotowa do wypłaty i zostanie uwzględniona w kolejnym obliczeniu wypłaty.                      |
| **Cena (DW)**                                   | Cena jednostki użycia dla danej jednostki SKU (w walucie klienta).                                                                  |
| **Adres E-mail dostawcy**                                 | Adres e-mail dostawcy, który jest powiązany z relacją między firmą Microsoft a klientem końcowym. Jeśli klient jest przedsiębiorstwem do odsprzedawcy, zostanie odsprzedawca.  Jeśli jest używany dostawca rozwiązań w chmurze (CSP), będzie on dostawcą usług kryptograficznych.                                                                 |
| **Nazwa dostawcy**                                 | Nazwa dostawcy występującego w relacji między firmą Microsoft a klientem końcowym. Jeśli klient jest przedsiębiorstwem do odsprzedawcy, zostanie odsprzedawca.  Jeśli jest używany dostawca rozwiązań w chmurze (CSP), będzie on dostawcą usług kryptograficznych.                                                                 |
| **Identyfikator URI zasobu**                                  | Ścieżka, w której wdrożono artefakt wdrożenia.                                            |
| **SKU**                                           | Nazwa jednostki SKU zdefiniowana podczas publikowania. Oferta może mieć wiele jednostek SKU, ale jednostka SKU może być skojarzona tylko z jedną ofertą.              |
| **Typ rozliczeń jednostki SKU**                                 | Metoda rozliczania jednostki SKU.                                                                                                             |
| **Typ rozliczeń SKU: Dostarcz własną licencję**         | Maszyna wirtualna lub usługa wymaga klucza licencji dostarczonego przez klienta do wdrożenia. Firma Microsoft nie nalicza opłat za użycie tych ofert w portalu Marketplace.   |
| **Typ rozliczeń SKU: Microsoft jako odsprzedawca**          | Firma Microsoft jest odsprzedawcą dla tej jednostki SKU.   |
| **Typ rozliczeń SKU: bezpłatny**                           | Jednostka SKU jest skonfigurowana bezpłatnie dla wszystkich użytkowników. Firma Microsoft nie rozlicza klientów za użycie tej jednostki SKU.                           |
| **Typ rozliczeń SKU: płatny**                           | Firma Microsoft jest Twoim agentem dla tej jednostki SKU i obciąża klientów w Twoim imieniu. (Karta kredytowa PAYG lub faktura w przedsiębiorstwie)                   |
| **Typ rozliczeń SKU: wersja próbna**                          | Klient jest w okresie próbnym i zostanie przekonwertowane na płatne, jeśli nie anuluje ani nie usunie.                                         |
| **Kwota podatku (DW)**                                  | Kwota podatku zastosowana do rachunku klienta w walucie klienta (DW).                                                               |
| **Data transakcji**                                 | Data transakcji zarejestrowana w raporcie wypłaty.                                                                           |
| **Typ transakcji**                               | Typ transakcji, która jest raportowana.  Na przykład może to być opłata, zwrot lub korekta wypłaty.                                                                                        |
| **Data zakończenia okresu próbnego**                                 | Data zakończenia okresu próbnego dla tego zamówienia lub zostanie ona zakończona.                                                                           |
| **Data użycia**                                       | Data wykorzystania klienta.                                                                                                             |
| **Ilość użycia**                                       | Zgłoszone użycie klienta w jednostce SKU. W przypadku obrazów maszyn wirtualnych rekordy użycia reprezentują użycie dla zgłoszonego okresu dla rozmiaru maszyny wirtualnej i jednostki SKU.                                               |
| **Informacje o użyciu**                                       | Identyfikator dla co najmniej jednego dnia użycia klienta dla danej jednostki SKU skojarzonej z wpisem w raporcie wypłaty.                                               |
| **Typ użycia**                                       | Opis mierzonego użycia. (Znormalizowane użycie lub surowe użycie)                                                              |
| **Typ użycia: znormalizowane użycie**                | Godziny użycia znormalizowane do konta dla liczby rdzeni maszyn wirtualnych wykorzystywanych w użyciu: [liczba rdzeni maszyn wirtualnych] x [godz. użycia pierwotnego]. Maszyny wirtualne oznaczone jako "SHAREDCORE" wykorzystują 1/6 (lub 0,1666) jako mnożnik [liczba rdzeni maszyn wirtualnych].                                        |
| **Typ użycia: surowe użycie**                        | Godziny użycia dla oferty portalu Marketplace.  Jest on prezentowany w postaci "Calendar" godzin.  Jest to liczba uruchomionych maszyn wirtualnych.                           |
| **Jednostki użycia**                                   | Jednostka miary dla podanego użycia. Maszyny wirtualne są zawsze mierzone przy użyciu jednostek miary co godzinę.                               |
| **Rozmiar maszyny wirtualnej**                                       | Przedstawia rozmiar sprzętu maszyny wirtualnej wyrównany do oferty platformy Azure. Przykłady obejmują `Basic_A0`, `Standard_A11`, `Standard_D12`i `Standard_G4`.   |
|  |  |
