# <a name="co-sell-option-in-partner-center"></a>Opcja współsprzedaży w Centrum partnerów

Dostarczanie informacji na karcie Co-sell jest całkowicie opcjonalne. Jest to wymagane do osiągnięcia co-sell Ready i IP Co-sell Ready status. Podane informacje będą używane przez zespoły sprzedaży firmy Microsoft, aby dowiedzieć się więcej o rozwiązaniu podczas oceny jego dopasowania do potrzeb klienta. Nie jest dostępny bezpośrednio dla klientów. Aby uzyskać więcej informacji, zobacz [sprzedaż za pośrednictwem firmy Microsoft](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-new-saas-offer#sell-through-microsoft).

Opcja współsprzedaży w Centrum partnerów jest obecnie dostępna dla niektórych typów ofert. Obsługa dodatkowych typów ofert będzie dostępna w czasie.

| **Typ oferty** | **Strona co-sell dostępna w Centrum partnerów** |
|    -----------------------    |    -----------------------------  |
| Oprogramowanie jako usługa (SaaS) |     Ogólna dostępność około kwietnia 2020 r.     |
| Dynamics 365 for Customer Engagement & PowerApps      |     Dostępny teraz      |
| Dynamics 365 for Operations     |     Dostępny teraz       |
| Dynamics 365 Business Central      |  Ogólna dostępność około kwietnia 2020 r.    |
| Azure Application     |     Ogólna dostępność około kwietnia 2020 r.     |
| Maszyna wirtualna platformy Azure      |     Do ogłoszenia       |
| Kontener platformy Azure     |     Do ogłoszenia       |
| Usługa konsultingowa      |     Do ogłoszenia       |
| Moduł brzegowy IoT      |     Do ogłoszenia       |
| Aplikacja usługi Power BI     |     Do ogłoszenia       |
|  |

## <a name="listing"></a>Aukcji

Oferty sprzedaży współsprzedają pomogą zespołom sprzedaży firmy Microsoft sprzedawać twoje rozwiązanie szerszemu gronu odbiorców.

- Wybierz platformy firmy Microsoft, z którymi jest zbudowana oferta, rozszerza lub integruje się z nią.
- Wybierz segmenty rynku, na które kierujesz ofertę.
- Wybierz [typ rozwiązania](#solution-type) dla swojej oferty.
- Wybierz maksymalnie trzy [obszary rozwiązania](#solution-areas) i podobszady.

### <a name="solution-type"></a>Typ rozwiązania

Typy rozwiązań pomagają zdefiniować scenariusze, które oferta jest przeznaczona do rozwiązania.

| **Typ rozwiązania**    | **Opis**  |
| :------------------- | :-------------------|
| Urządzenie (sprzęt) | Oferta obejmująca budowę lub sprzedaż sprzętu od producenta urządzenia. |
| IP (aplikacja) | Aplikacje lub inne materiały chronione prawami autorskimi licencjonowane do użytku klienta. Na przykład: program CRM, który może być licencjonowany i instalowany lokalnie. |
| Usługa | Praktyczna wiedza specjalistyczna dla konkretnego jednorazowego projektu, często dostarczana przez konsultantów. Na przykład: konfigurowanie bazy danych klientów dla klienta (z klientem przyjmującym odpowiedzialność za obsługę bazy danych po dostarczeniu). |
| Usługa zarządzana | Praktyczna wiedza specjalistyczna w zakresie projektu opartego na chmurze, zwykle na bieżąco. Na przykład: Zapewnienie platformy i narzędzi do uruchamiania bazy danych online, z bieżącym zarządzaniem zapewnianym przez dostawcę usług zarządzanych. |

### <a name="solution-areas"></a>Obszary rozwiązań

Obszary rozwiązań pomagają w dalszym definiowaniu rozwiązania.

Możesz dodać maksymalnie trzy obszary rozwiązania dla swojej oferty. Dla każdego obszaru rozwiązania można wybrać maksymalnie dwa podobszary rozwiązania. Aby wybrać wiele podkategorii, użyj klawisza `Ctrl` (w systemie Windows) lub klawisza Command (w systemie Mac OS).

## <a name="contacts"></a>Kontakty

Informacje kontaktowe umożliwiają zespołom sprzedaży firmy Microsoft i sprzedawcom kanałów żądanie dodatkowych informacji od odpowiedniego zasobu w organizacji. Informacje kontaktowe są dostępne dla wszystkich zespołów sprzedaży firmy Microsoft. Jeśli zdecydujesz się udostępnić swoją ofertę w programie CSP, te informacje kontaktowe będą również dostępne dla sprzedawców kanałów.

Ten dokument zawiera wskazówki dotyczące różnych ról dla partnerów i jak wybrać między nimi. Kontakty można podać, pobierając szablon CSV z Centrum partnerów, wypełniając każdy wiersz informacjami o kontakcie i zaimportując ten plik CSV. Importowanie pliku CSV spowoduje zastąpienie istniejących kontaktów.

Można również wyeksportować istniejącą listę kontaktów oferty, a następnie wprowadzić zmiany w tym pliku.

* *Nazwa (wymagana)*: Nazwa kontaktu.
* *Adres e-mail (wymagany)*: adres e-mail kontaktu.
* *Stanowisko (wymagane)*: Stanowisko.
* *Rola (wymagana)*: Użyj jednej z następujących ról w tym polu w szablonie.

    | **Roli**    | **Opis**  |
    | :------------------- | :-------------------|
    | *:::no-loc text="Partner marketing":::* | Ta rola koncentruje się na marketingu rozwiązania i współpracy nad działaniami marketingowymi z zespołami sprzedaży firmy Microsoft i sprzedawcami kanałów. Głównym punktem kontaktowym dla zaangażowania marketingowego i oferowania treści z listą, takich jak opisy produktów, obrazy i filmy. |
    | *:::no-loc text="Partner sales":::* | Ta rola koncentruje się na sprzedaży rozwiązania i współpracy przy sprzedaży z zespołami sprzedaży firmy Microsoft i sprzedawcami kanałów. W przypadku rozwiązań co-sell należy wskazać co najmniej jeden kontakt sprzedażowy partnera na region, w którym chcesz być Co-sell Ready. Ten sam kontakt handlowy partnera może obejmować wiele regionów. |
    | *:::no-loc text="Partner technical sales":::* | Obsługuje zagadnienia dotyczące architektury technicznej i wdrażania podczas cyklu sprzedaży, integracji po sprzedaży i okresów wdrażania. |
    | *:::no-loc text="Partner customer success manager":::* | Zazwyczaj obsługuje klientów po wdrożeniu, aby pomóc im w pełni wykorzystać swoje rozwiązanie i zwiększyć jego zasięg w organizacji klienta. |

* *Kraje/regiony (wymagane):*  Podczas wypełniania szablonu należy użyć [dwóch liter kodów kraju/regionu, jak podano](./commercial-marketplace-co-sell-countries.md). Jeśli kontakt obejmuje wszystkie kraje/regiony, użyj trzyliterowego kodu "OOO". Jeśli kontakt obejmuje więcej niż jeden kraj/region, wprowadź każdy z dwóch kodów liter oddzielonych przecinkiem (na przykład wpisz "US, CA, FR" bez cudzysłowów w szablonie).

    Kraje/regiony powinny odzwierciedlać terytorium każdego kontaktu. Zespoły sprzedaży firmy Microsoft i sprzedawcy kanałów będą wykorzystywać te informacje do żądania informacji lub współpracy w zakresie sprzedaży w określonym kraju/regionie.

* *Stany/prowincje (opcjonalnie):*  Podczas wypełniania szablonu należy użyć formatu XX-XX wymienionego w [tabeli Stany/prowincje](./commercial-marketplace-co-sell-states.md).
