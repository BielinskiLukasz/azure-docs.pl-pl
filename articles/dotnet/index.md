# <a name="azure-libraries-for-net"></a>Biblioteki platformy Azure dla platformy .NET

Użyj bibliotek w zestawie .NET SDK platformy Azure, aby zarządzać usługami platformy Azure i wykorzystywać je w swoich aplikacjach.

## <a name="packages"></a>Pakiety

> [!TIP]
> [Biblioteki Fluent](https://azure.microsoft.com/blog/simpler-azure-management-libraries-for-net/) (z końcówką **\*.Fluent**) poprawiają środowisko deweloperskie, zapewniając zorientowany na obiekty interfejs API wyższego poziomu zoptymalizowany pod kątem czytelności i możliwości zapisu. Określają elementy wymagane i opcjonalne oraz elementy, których nie można modyfikować. Biblioteki te mogą być uruchamiane wraz z bibliotekami innymi niż Fluent, więc możesz użyć pakietów Fluent, jeśli wolisz tę składnię. [Microsoft.Azure.Management.Fluent](https://www.nuget.org/packages/Microsoft.Azure.Management.Fluent) to pakiet zbiorczy zawierający wszystkie biblioteki zarządzania Fluent.

## <a name="installation"></a>Instalacja

### <a name="visual-studio"></a>Visual Studio

Jeśli korzystasz z programu Visual Studio, użyj **konsoli menedżera pakietów NuGet**, aby zaimportować pakiet do projektu.

1. Przy otwartym rozwiązaniu programu Visual Studio uruchom konsolę, klikając pozycję **Narzędzia**, **Menedżer pakietów NuGet**, a następnie **Konsola menedżera pakietów**.  

    ![Konsola menedżera pakietów](media/index/package-manager.png)

2. W oknie konsoli użyj polecenia cmdlet **Install-Package**, aby pobrać i zainstalować pakiet NuGet.  Aby na przykład uwzględnić najnowszą wersję [biblioteki zarządzania zasobami platformy Azure](http://www.nuget.org/packages/Microsoft.Azure.Management.ResourceManager) dla programu .NET:

    ```powershell
    Install-Package Microsoft.Azure.Management.ResourceManager -Pre 
    ``` 
    Aby użyć określonej wersji, uwzględnij numer wersji w następujący sposób:

    ```powershell
    Install-Package Microsoft.Azure.Management.ResourceMananger -Version 1.4.0-preview
    ``` 

### <a name="other-editors"></a>Inne edytory

Jeśli używasz platformy .NET Core z programem Visual Studio Code (lub innym edytorem), edytuj plik csproj, aby dodać pakiet jako element **PackageReference**.  W tym przykładzie użyto konkretnej wersji elementu **Microsoft.Azure.Management.ResourceManager**, co jest najlepszą praktyką, niemniej obsługiwane są również [wersje zmienne](/nuget/consume-packages/package-references-in-project-files#floating-versions) NuGet.

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp1.1</TargetFramework>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.Azure.Management.ResourceManager" Version="1.4.0-preview" />
  </ItemGroup>

</Project>
```

> [!TIP]
> Poprzednie wersje platformy .NET Core używały plików project.json zamiast plików csproj.  Aby dowiedzieć się więcej o mapowaniu plików project.json na pliki csproj, [zobacz ten dokument](/dotnet/articles/core/tools/project-json-to-csproj).
