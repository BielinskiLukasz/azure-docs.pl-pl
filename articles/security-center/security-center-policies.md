#Azure セキュリティ センターでのセキュリティ ポリシーの設定
このドキュメントは、Azure セキュリティ センターでセキュリティ ポリシーを構成するために役立ちます。そのタスクを実行するために必要な手順について説明しています。

> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。

## Azure セキュリティ センターとは
Azure セキュリティ センターは、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。 これにより、サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

##セキュリティ ポリシーとは
セキュリティ ポリシーは、指定されたサブスクリプション内のリソースに推奨されるコントロールのセットを定義します。 Azure セキュリティ センターでは、セキュリティに関する会社のニーズ、および各サブスクリプションでのアプリケーションのタイプやデータの機密度に合わせて、Azure サブスクリプションのポリシーを定義できます。
 
たとえば、開発またはテストに使用されるリソースには、実稼働アプリケーションで使われるものと異なるセキュリティ要件があります。 同様に、PII (個人情報) のような規制されたデータを持つアプリケーションには、より高いレベルのセキュリティが必要です。 Azure セキュリティ センターで有効化されているセキュリティ ポリシーは、セキュリティに関する推奨事項と監視を促進して、潜在的な脆弱性を識別し、脅威を軽減します。

##セキュリティ ポリシーの設定

セキュリティ ポリシーは、サブスクリプションごとに構成されます。 セキュリティ ポリシーを変更するには、そのサブスクリプションの所有者または共同作業者である必要があります。 Azure セキュリティ センターでポリシーのセキュリティを構成するには、次の手順に従います。

1. をクリックして、 **セキュリティ ポリシー** Azure セキュリティ センターのダッシュ ボードに並べて表示します。
 
2.  **セキュリティ ポリシー - サブスクリプションごとのポリシー定義** ] ブレードが開いたら、右側に、[セキュリティ ポリシーを有効にサブスクリプション。

    ![データ収集の有効化](./media/security-center-policies/security-center-policies-fig0.png)

3.  **セキュリティ ポリシー**  以下に示すようなオプションのセットのサブスクリプションのブレードが開きます。

    ![データ収集の有効化](./media/security-center-policies/security-center-policies-fig1.png)

    
4. 確認 **仮想マシンからデータを収集** オプションは、 **に**します。 このオプションは、既存および新規のリソースに対する自動ログ収集を有効化します。 

5. ストレージ アカウントがまだ構成されていない場合も同様の警告が開いたときに次の図で示す参照してください可能性があります、 **セキュリティ ポリシー**:

    ![Storage の選択](./media/security-center-policies/security-center-policies-fig2.png) 

6. この警告が表示された場合は、このオプションをクリックして、次の図に示すようにリージョンを選択します。

    ![Storage の選択](./media/security-center-policies/security-center-policies-fig3.png) 

7. 仮想マシンが実行されている各リージョンに対し、これらの仮想マシンから収集されたデータが格納されるストレージ アカウントを選択します。 これにより、プライバシーとデータ主権目的で、同じ地域にデータを保持することが容易になります。 使用するリージョンを決めたら、そのリージョンを選択し、ストレージ アカウントを選択します。

8.  **ストレージ アカウントを選択** ブレードをクリックして **OK**します。

    > [AZURE.NOTE] を行う場合は、1 つの中央のストレージ アカウント内のさまざまな地域での仮想マシンからデータを集計できます。 参照してください、 [Azure セキュリティ センターに関する FAQ](security-center-faq.md) の詳細

9.  **セキュリティ ポリシー** ブレードをクリックして **に** をこのサブスクリプションで使用するセキュリティの推奨事項を有効にします。 参照として次の表を使用して、各オプションで何が実行されるかを理解します。  

    |ポリシー |状態 |
|----- |-----|
|システムの更新プログラム |Windows の仮想マシンにインストール先のサービスに応じて、Windows Update または WSUS から利用可能な更新の一覧がすべての 12 時間と不足している更新をお勧めするバーチャル マシンの構成を取得します。 |
|基準の規則 |攻撃を受けやすく、仮想マシンを行うことができる OS の構成のいずれかを識別するためにサポートされているすべての仮想マシン 12 時間を分析し、構成の変更にこれらの脆弱性に対処することをお勧めします。 参照してください、 [推奨される基準の一覧を](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335) 監視されている特定の構成についての詳細。 |
|マルウェア対策 |すべての Windows 仮想マシンを識別して、ウイルス、スパイウェア、およびその他の悪意のあるソフトウェアを削除することをお勧めのマルウェア対策をプロビジョニングします。 |
|エンドポイントのアクセス制御リスト |ことをお勧め、 [アクセス コントロール リスト](virtual-machines-set-up-endpoints.md) (ACL) クラシック仮想マシンのエンドポイントへのアクセスを制限するように構成します。 これは通常、企業ネットワークに接続されているユーザーのみが仮想マシンにアクセスできるようにするために使用されます。 |
|ネットワークのサブネットとネットワーク インターフェイス上のセキュリティ グループ |ことをお勧め [ネットワーク セキュリティ グループ](virtual-networks-nsg.md) (Nsg) のサブネットへの受信および送信のトラフィックを制御し、ネットワーク インターフェイス リソース マネージャーの仮想マシンのように構成します。 サブネットに構成された NSG は、他に指定しない限り、すべての仮想マシンのネットワーク インターフェイスによって継承されます。 NSG が構成されていることをチェックすることに加え、任意の受信トラフィックを許可するルールを識別するために受信セキュリティ ルールが評価されます。 |
|Web アプリケーション ファイアウォール |Web アプリケーション ファイアウォールは、リソース マネージャーの仮想マシンで提供することをお勧めとき: [インスタンス レベル パブリック IP](virtual-networks-instance-level-public-ip.md) (ILPIP) が使用され、関連付けられている NSG 受信のセキュリティ規則は、ポート 80 および 443 へのアクセスを許可するように構成されています。 負荷分散 IP (VIP) が使用され、関連付けられている負荷分散と受信 NAT ルールがポート 80 および 443 へのアクセスを許可するように構成されている (詳細については、次を参照してください [ロード バランサーの Azure リソース マネージャーによるサポート](load-balancer-arm.md)) |。
|SQL 監査 |検出と調査のための高度なコンプライアンスのための Azure SQL サーバーとデータベースへのアクセスの監査を有効にすることをお勧めします。 |
|SQL の透過的なデータ暗号化 |Rest で暗号化することをお勧め、Azure SQL データベース、関連付けられているバックアップとトランザクションのログ ファイルを保存、データが破られた場合でもは読み取れないように有効にします。 |

10. すべてのオプションの構成が完了したら、クリックして **保存** 変更をコミットします。

> [AZURE.NOTE] データ収集を有効にするには、監視エージェントがインストールされます。 この場所からのデータ収集を今すぐ有効にしない場合は、後からヘルス ビューと推奨事項のビューから有効にすることができます。 データ収集は、サブスクリプションや選択した VM だけに有効にすることもできます。 参照してください、 [Azure セキュリティ センターに関する FAQ](security-center-faq.md) サポートされている Vm についての詳細を知る。
