# 構成テーブルを使用してクロスプレミス仮想ネットワークを作成する

[AZURE.INCLUDE [learn-about-deployment-models-classic-include](../../includes/learn-about-deployment-models-classic-include.md)] リソース マネージャーの配置モデルです。

このトピックでは、以下の構成テーブルのセットで先に指定した設定を使用してクロスプレミス仮想ネットワークを作成する手順について説明します。

- テーブル V: クロスプレミス仮想ネットワークの構成
- テーブル S: 仮想ネットワークのサブネット
- テーブル D: オンプレミス DNS サーバー
- テーブル L: ローカル ネットワークのアドレス プレフィックス

これらのテーブルは通常、Azure 内の IT ワークロードの構成を説明するトピックで設定され、クロスプレミス仮想ネットワークを含みます。 参照してください [フェーズ 1: Azure を構成する](virtual-machines-workload-intranet-sharepoint-phase1.md) 例については、です。

次の手順では、これらのテーブルの情報を参考にして、仮想ネットワークの構成プロセスの手順を説明します。 場合は、まだ指定していない設定で別のトピックでこれらのテーブル間のクロスプレミス仮想ネットワークの構成を参照してください場合でも、 [を Azure の仮想ネットワーク間のクロスプレミス サイト対サイト接続を構成する](../vpn-gateway/vpn-gateway-site-to-site-create.md)です。

> [AZURE.NOTE] この手順では、サイト間 VPN 接続を使用して仮想ネットワークを作成します。 サイト間接続用に Azure ExpressRoute を使用する方法の詳細については、次を参照してください。 [ExpressRoute の技術概要](../expressroute/expressroute-introduction.md)します。

## 構成テーブルの設定を使用して新しいクロスプレミス Azure Virtual Network を作成する

1. [Azure クラシック ポータル] にサインインします。
2. タスク バーで、クリックして **新規 > ネットワーク サービス > 仮想ネットワーク > カスタム作成**します。
3.  **仮想ネットワークの詳細** ページ。
    -  **名**, 、表 V の項目 1 の名前を入力します。
    -  **場所**, 、テーブル v の項目 2 のリージョンを選択します。
4. 矢印をクリックして次へ進みます。
5.  **DNS サーバーおよび VPN 接続** ページ。
    -  **DNS サーバー**, 、テーブル D の各エントリは、フレンドリ名と内部設置型 DNS サーバーの IP アドレスを構成します。
    -  **サイト対サイト接続**, [ **サイト対サイト VPN の構成**します。
    - 既にを構成したローカル ネットワークを使用する場合でその名前を選択します。 **ローカル ネットワーク**します。 新しいローカル ネットワークを作成する場合は、選択 **新しいローカル ネットワークを指定して** で **ローカル ネットワーク**します。
    - サブスクリプションのローカル ネットワークを既に構成していない場合は表示されません、 **ローカル ネットワーク** フィールドです。
6. 矢印をクリックして次へ進みます。
7.  **サイト対サイト接続** ページ (選択した場合は **新しいローカル ネットワークを指定して** 手順 5. で)。
    -  **名**, 、表 V (ローカル ネットワーク名) に項目 3 から名前を入力します。
    -  **VPN デバイスの IP アドレス**, 、表 V のアイテム 4 からアドレスを入力します。
    -  **アドレス空間**, 、テーブル L の各エントリは、プレフィックスに関する組織ネットワークの IP アドレス空間を入力してください (で **開始 IP**) とプレフィックスの長さ (で **CIDR (アドレス数)**)。
8. 矢印をクリックして次へ進みます。
9.  **仮想ネットワーク アドレス空間** ページ。
    -      **アドレス空間**, 、仮想ネットワークのプライベート IP アドレス空間をテーブル v の項目 5 から、プレフィックスを入力 (で **開始 IP**) とプレフィックスの長さ (で **CIDR (アドレス数)**)。
    -  **サブネット**, 、テーブル S の各エントリに
        - 内のサブネットの名前を入力、 **サブネット** 列で、必要な場合は、既定の名前を上書きします。
        - サブネット プレフィックスのプライベート IP アドレス空間を入力 (で **開始 IP**) とプレフィックスの長さ (で **CIDR (アドレス数)**)。
    - クリックして **ゲートウェイ サブネットの追加**します。
10. チェック マークをクリックして、構成を完了します。
