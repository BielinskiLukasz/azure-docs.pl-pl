# StorSimple Snapshot Manager を使用したボリューム グループの作成と管理

## 概要

使用することができます、 **ボリューム グループ** 上のノード、 **スコープ** ボリュームをグループに割り当てるボリューム、ボリューム グループに関する情報の表示ウィンドウは、バックアップをスケジュールし、ボリューム グループを編集します。 

ボリューム グループは関連するボリュームのプールであり、バックアップのアプリケーション整合性を確保するために使用されます。 詳細については、次を参照してください。 [ボリュームとボリューム グループ](storsimple-what-is-snapshot-manager.md#volumes-and-volume-groups) と [Windows ボリューム シャドウ コピー サービスとの統合](storsimple-what-is-snapshot-manager.md#integration-with-windows-volume-shadow-copy-service)します。

>[AZURE.IMPORTANT] 
>
> * ボリューム グループ内のすべてのボリュームは、1 つのクラウド サービス プロバイダーから取得する必要があります。
> 
> * ボリューム グループを構成するときは、クラスター共有ボリューム (CSV) と非 CSV を同じボリューム グループに混在させないでください。 StorSimple Snapshot Manager では、同じスナップショット内に CSV と非 CSV が混在した環境はサポートされません。
 
![[ボリューム グループ] ノード](./media/storsimple-snapshot-manager-manage-volume-groups/HCS_SSM_Volume_groups.png)

**図 1: StorSimple Snapshot Manager の [ボリューム グループ] ノード** 

このチュートリアルでは、StorSimple Snapshot Manager を使用して次を行う方法を説明します。

- ボリューム グループに関する情報の表示 
- ボリューム グループの作成
- ボリューム グループのバックアップ
- ボリューム グループの編集
- ボリューム グループの削除

これらの操作でも使用可能な **アクション** ウィンドウです。
 
## ボリューム グループの表示

クリックすると、 **ボリューム グループ** ] ノード、 **結果** ペインは、各ボリューム グループに関する次の情報を表示、列の選択を行います。 (内の列、 **結果** ウィンドウは構成可能です。 右クリックし、 **ボリューム** ノードで、選択 **ビュー**, 、し、[ **列の追加/削除**.)

結果の列 | 説明 
:--------------|:------------ 
名前           |  **名前** 列には、ボリューム グループの名前が含まれています。
アプリケーション    |  **アプリケーション** 列は、Windows ホストで実行されている、現在インストールされている VSS ライターの数を表示します。
オン       |  **選択** 列は、ボリューム グループに含まれているボリュームの数を示します。 ゼロ (0) は、ボリューム グループ内のボリュームに関連付けられたアプリケーションが存在しないことを示します。
インポート済み       |  **インポート済み** 列にインポートされたボリュームの数が表示されます。 設定すると **True**, 、このコラムことを示し、ボリューム グループが Azure クラシック ポータルからインポートされた StorSimple Snapshot Manager では作成されませんでした。
 
>[AZURE.NOTE] StorSimple Snapshot Manager ボリューム グループに表示されます、 **バックアップ ポリシー** Azure クラシック ポータルでタブをクリックします。
 
## ボリューム グループの作成

ボリューム グループを作成するには、以下の手順を実行します。

#### ボリューム グループを作成するには

1. デスクトップ アイコンをクリックして、StorSimple Snapshot Manager を起動します。 

2.  **スコープ** ] ウィンドウを右クリックして **ボリューム グループ**, 、クリックして **ボリューム グループの作成**します。 

    ![ボリューム グループの作成](./media/storsimple-snapshot-manager-manage-volume-groups/HCS_SSM_Create_volume_group.png)
 
     **ボリューム グループの作成** ] ダイアログ ボックスが表示されます。 

    ![[ボリューム グループの作成] ダイアログ](./media/storsimple-snapshot-manager-manage-volume-groups/HCS_SSM_CreateVolumeGroup_dialog.png) 

3.  次の情報を入力します。 

    1.  **名前** ボックスに、新しいボリューム グループの一意の名前を入力します。 

    2.  **アプリケーション** ボックスで、ボリューム グループに追加するボリュームに関連付けられているアプリケーションを選択します。 

         **アプリケーション** Azure StorSimple ボリュームを使用し、VSS ライターがあるアプリケーションのみを有効にできるボックスに一覧表示します。 VSS ライターは、ライターが認識しているすべてのボリュームが Azure StorSimple ボリュームである場合にのみ有効です。 [アプリケーション] ボックスが空の場合は、Azure StorSimple ボリュームを使用し、サポートされる VSS ライターを持つアプリケーションはインストールされていません  (現在、Azure StorSimple は Microsoft Exchange と SQL Server をサポートしています)。VSS ライターの詳細については、次を参照してください。 [Windows ボリューム シャドウ コピー サービスとの統合](storsimple-what-is-snapshot-manager.md#integration-with-windows-volume-shadow-copy-service)します。

        アプリケーションを選択すると、それに関連付けられているすべてのボリュームが自動的に選択されます。 逆に、特定のアプリケーションに関連付けられているボリュームを選択する場合、アプリケーションが自動的に選択、 **アプリケーション** ボックス。 

    3.  **ボリューム** ボックスで、ボリューム グループに追加する Azure StorSimple ボリュームを選択します。 

      - 1 つまたは複数のパーティションを持つボリュームを含めることができます  (複数のパーティションを持つボリュームは、複数のパーティションを持つ動的ディスクまたは基本ディスクにすることができます)。複数のパーティションが含まれているボリュームは、1 つの単位として扱われます。 このため、ボリューム グループに 1 つのパーティションのみを追加する場合、その他のすべてのパーティションが、同時にそのボリューム グループに自動的に追加されます。 複数パーティション ボリュームをボリューム グループに追加した後も、その複数パーティション ボリュームは 1 つの単位として扱われます。

      - ボリュームを割り当てないようにして、空のボリューム グループを作成できます。 

      - クラスター共有ボリューム (CSV) と非 CSV を同じボリューム グループで混在させないでください。 StorSimple Snapshot Manager では、同じスナップショット内に CSV ボリュームと非 CSV ボリュームが混在した環境はサポートされません。 

4. クリックして **OK** ボリューム グループを保存します。

## ボリューム グループのバックアップ

ボリューム グループをバックアップするには、以下の手順を実行します。

#### ボリューム グループをバックアップするには

1. デスクトップ アイコンをクリックして、StorSimple Snapshot Manager を起動します。

2.  **スコープ** ] ウィンドウで、展開、 **ボリューム グループ** ノードでは、ボリューム グループ名を右クリックし、をクリックして **バックアップ**します。 

    ![ボリューム グループの即時バックアップ](./media/storsimple-snapshot-manager-manage-volume-groups/HCS_SSM_Take_backup.png)

3.  **バックアップ** ダイアログ ボックスで、 **ローカル スナップショット** または **クラウド スナップショット**, 、クリックして **作成**します。 

    ![[バックアップの作成] ダイアログ](./media/storsimple-snapshot-manager-manage-volume-groups/HCS_SSM_TakeBackup_dialog.png) 

4. バックアップが実行されていることを確認するには展開、 **ジョブ** ノード、およびクリック **を実行している**します。 バックアップが表示されます。

5. 完了したスナップショットを表示するには展開、 **バックアップ カタログ** ノードは、ボリューム グループ名を展開し、クリックして **ローカル スナップショット** または **クラウド スナップショット**します。 バックアップが正常に完了した場合は、バックアップが表示されます。 

## ボリューム グループの編集

ボリューム グループを編集するには、以下の手順を実行します。

#### ボリューム グループを編集するには

1. デスクトップ アイコンをクリックして、StorSimple Snapshot Manager を起動します。

2.  **スコープ** ] ウィンドウで、展開、 **ボリューム グループ** ノードでは、ボリューム グループ名を右クリックし、をクリックして **編集**します。 

3. **[ボリューム グループの作成]** ダイアログ ボックスが表示されます。 変更することができます、 **名前**, 、**アプリケーション**, 、および **ボリューム** エントリです。 

4. クリックして **OK** して変更を保存します。

## ボリューム グループの削除

ボリューム グループを削除するには、以下の手順を実行します。 

>[AZURE.WARNING] これは、ボリューム グループに関連付けられているすべてのバックアップも削除されます。

#### ボリューム グループを削除するには

1. デスクトップ アイコンをクリックして、StorSimple Snapshot Manager を起動します。 

2.  **スコープ** ] ウィンドウで、展開、 **ボリューム グループ** ノードでは、ボリューム グループ名を右クリックし、をクリックして **削除**します。 

3.  **ボリューム グループの削除** ] ダイアログ ボックスが表示されます。 型 **Confirm** テキスト ボックスをクリックし **OK**します。 

    削除されたボリューム グループは、リストから消去、 **結果** ウィンドウとそのボリューム グループに関連付けられているすべてのバックアップは、バックアップ カタログから削除されます。
