# Logic Apps を監視する

ロジック アプリを作成した後 (手順で [ロジック アプリの作成](app-service-logic-create-a-logic-app.md))、Azure ポータルで実行の完全な履歴を確認できます。 履歴を表示するには、次のように選択します。 **参照**, [ **Web + モバイル**, 、を選択し **Logic Apps**します。 サブスクリプション内のすべての Logic Apps の一覧が表示されます。 ロジック アプリを指定できます **有効** または **無効になっている**します。 **有効になっている** Logic Apps では、トリガーが、トリガー イベントに反応ロジック アプリを実行することを意味します。 **無効になっている** イベントへの応答でロジック アプリは実行しません。

![概要](./media/app-service-logic-monitor-your-logic-apps/overview.png)

表示されたロジック アプリのブレードには、役に立つセクションが 2 つあります。

- **概要**, 、最新の状態を示す、ロジック アプリを編集するためのエントリ ポイントです。
- **すべての実行**, 、このロジック アプリがいた実行内容の一覧を示します。

## アプリの実行の表示

![すべての実行](./media/app-service-logic-monitor-your-logic-apps/allruns.png)

この実行の一覧表示、 **開始時刻**, 、 **実行識別子** (はこれを使用して REST API を呼び出す)、および **期間** の特定の実行。 実行の詳細を表示する行を選択します。

詳細ブレードには、グラフが実行時間とその実行でのすべてのアクションの順序と共に表示されます。 その下に、実行されたすべてのアクションの完全な一覧が表示されます。  

![実行とアクション](./media/app-service-logic-monitor-your-logic-apps/runandaction.png)

最後に、特定のアクションに関してすれば、すべてのデータ、そのアクションに渡されたやアクションから受信した、 **入力** と **出力** セクションです。 すべてのコンテンツを表示するには、リンクを選択します (リンクをコピーして、コンテンツをダウンロードすることもできます)。 

もう 1 つの重要な情報は、 **追跡 ID**します。 この ID は、すべてのアクションの呼び出しのヘッダーで渡されます。 独自のサービス内でログを記録している場合、トレース ID を記録することをお勧めします。これによって、この ID を使用して独自のログと相互参照することができます。

## トリガー履歴の表示 

ポーリング トリガーはある間隔を置いて API を確認しますが、必ずしも実行が開始されるわけではなく、開始されるかどうかは応答によって決まります (たとえば、`200` は実行することを、`202` は実行しないことを意味します) 。 トリガー履歴には、発生したがロジック アプリが実行されていないすべての呼び出しが表示されます (`202` 応答)。  

![トリガー履歴](./media/app-service-logic-monitor-your-logic-apps/triggerhistory.png)

トリガーごとに確認できます、 **起動**, 実行しなかった場合、またはなんらかのエラーが発生した場合 (その **失敗**)。 理由を調べるため、トリガーが失敗した、選択、 **出力** リンクします。 で発生した場合は、選択、 **実行** 後に変更内容を表示するリンク。

注意してください *プッシュ* トリガー、操作を行う *いない* ここで、実行を開始する時刻を参照してください。 代わりに表示、 *コールバックの登録* 呼び出しでは、コールバックを取得するロジック アプリが登録した時点です。 Push トリガーが機能しない場合、登録に問題がある可能性がありますが ([出力] で確認できます)、それ以外の場合は、その API を特に調べる必要があります。

## バージョン管理の有効化

(近日対応予定) UI で現時点ではないが、使用して設定できるは追加的な機能がある、 [REST api](http://go.microsoft.com/fwlink/p/?LinkID=525617&clcid=0x409)します。 ロジック アプリの定義を更新すると、定義の前のバージョンが保存されます。 その理由は、実行が既に進行中の場合、その実行は、実行開始時のロジック アプリのバージョンを参照するためです。 実行の定義は進行中に変更できません。 バージョン履歴 REST API によって、この情報にアクセスできます。
