# Azure セキュリティ センターでのセキュリティ ポリシーの設定

このドキュメントは、Azure セキュリティ センターでセキュリティ ポリシーを構成するために役立ちます。そのタスクを実行するために必要な手順について説明しています。
> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。

## Azure セキュリティ センターとは

Azure セキュリティ センターは、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。 これにより、サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

## セキュリティ ポリシーとは

セキュリティ ポリシーは、指定されたサブスクリプション内のリソースに推奨されるコントロールのセットを定義します。 Azure セキュリティ センターでは、セキュリティに関する会社のニーズ、および各サブスクリプションでのアプリケーションのタイプやデータの機密度に合わせて、Azure サブスクリプションのポリシーを定義できます。

たとえば、開発またはテストに使用されるリソースには、実稼働アプリケーションで使われるものと異なるセキュリティ要件があります。 同様に、PII (個人情報) のような規制されたデータを持つアプリケーションには、より高いレベルのセキュリティが必要です。 Azure セキュリティ センターで有効化されているセキュリティ ポリシーは、セキュリティに関する推奨事項と監視を促進して、潜在的な脆弱性を識別し、脅威を軽減します。

## セキュリティ ポリシーの設定

セキュリティ ポリシーは、サブスクリプションごとに構成されます。 セキュリティ ポリシーを変更するには、そのサブスクリプションの所有者または共同作業者である必要があります。 Azure セキュリティ センターでポリシーのセキュリティを構成するには、次の手順に従います。

1. Azure セキュリティ センターのダッシュボードで **[セキュリティ ポリシー]** タイルをクリックします。

2. 右側に **[セキュリティ ポリシー - サブスクリプションごとにポリシーを定義]** ブレードが開いたら、セキュリティ ポリシーを有効にするサブスクリプションを選択します。

    ![データ収集の有効化](./media/security-center-policies/security-center-policies-fig0.png)

3. **セキュリティ ポリシー**  以下に示すようなオプションのセットのサブスクリプションのブレードが開きます。

    ![データ収集の有効化](./media/security-center-policies/security-center-policies-fig1.png)

4. **[仮想マシンからデータを収集]** オプションが **[オン]** になっていることを確認します。 このオプションは、既存および新規のリソースに対する自動ログ収集を有効化します。

5. ストレージ アカウントをまだ構成していない場合は、**セキュリティ ポリシー**を開いたときに、次の図と同様の警告が表示される場合があります。

    ![Storage の選択](./media/security-center-policies/security-center-policies-fig2.png)

6. この警告が表示された場合は、このオプションをクリックして、次の図に示すようにリージョンを選択します。

    ![Storage の選択](./media/security-center-policies/security-center-policies-fig3.png)

7. 仮想マシンが実行されている各リージョンに対し、これらの仮想マシンから収集されたデータが格納されるストレージ アカウントを選択します。 これにより、プライバシーとデータ主権目的で、同じ地域にデータを保持することが容易になります。 使用するリージョンを決めたら、そのリージョンを選択し、ストレージ アカウントを選択します。

8. **[ストレージ アカウントの選択]** ブレードで **[OK]** をクリックします。
    > [AZURE.NOTE] 必要に応じて、さまざまなリージョンの仮想マシンからのデータを 1 つの中央のストレージ アカウントに集計することができます。 参照してください、 [Azure セキュリティ センターに関する FAQ](security-center-faq.md) の詳細

9. **[セキュリティ ポリシー]** ブレードで **[オン]** をクリックして、このサブスクリプションで使用するセキュリティの推奨事項を有効にします。 参照として次の表を使用して、各オプションで何が実行されるかを理解します。

   | ポリシー| 状態オン|
|----- |-----|
| システムの更新| その仮想マシンに構成されているサービスに応じて、Windows Update または WSUS から利用可能な更新の一覧を 12 時間ごとに取得し、不足している更新を Windows の仮想マシンにインストールすることを推奨します。|
| 基準ルール| サポートされているすべての仮想マシンを 12 時間ごとに分析し、仮想マシンが攻撃を受けやすくなる可能性がある OS の構成を識別し、これらの脆弱性に対処するための構成の変更を推奨します。参照してください、 [推奨される基準の一覧を](https://gallery.technet.microsoft.com/Azure-Security-Center-a789e335) 監視されている特定の構成についての詳細。|
| マルウェア対策| ウイルス、スパイウェア、およびその他の悪意のあるソフトウェアを識別して削除するため、すべての Windows 仮想マシンにマルウェア対策をプロビジョニングすることをお勧めします。|
| エンドポイントの Access Control リスト| ことをお勧め、 [アクセス コントロール リスト](virtual-machines-set-up-endpoints.md) (ACL) クラシック仮想マシンのエンドポイントへのアクセスを制限するように構成します。これは通常、企業ネットワークに接続されているユーザーのみが仮想マシンにアクセスできるようにするために使用されます。|
| サブネットとネットワーク インターフェイス上のネットワーク セキュリティ グループ| ことをお勧め [ネットワーク セキュリティ グループ](virtual-networks-nsg.md) (Nsg) のサブネットへの受信および送信のトラフィックを制御し、ネットワーク インターフェイス リソース マネージャーの仮想マシンのように構成します。サブネットに構成された NSG は、他に指定しない限り、すべての仮想マシンのネットワーク インターフェイスによって継承されます。NSG が構成されていることをチェックすることに加え、任意の受信トラフィックを許可するルールを識別するために受信セキュリティ ルールが評価されます。|
| Web アプリケーション ファイアウォール| Web アプリケーション ファイアウォールは、リソース マネージャーの仮想マシンで提供することをお勧めとき: [インスタンス レベル パブリック IP](virtual-networks-instance-level-public-ip.md) (ILPIP) が使用され、関連付けられている NSG 受信のセキュリティ規則は、ポート 80 および 443 へのアクセスを許可するように構成されています。負荷分散 IP (VIP) が使用され、関連付けられている負荷分散と受信 NAT ルールがポート 80 および 443 へのアクセスを許可するように構成されている (詳細については、を参照してください [ロード バランサーの Azure リソース マネージャーによるサポート](load-balancer-arm.md))。|
| SQL 監査| コンプライアンス、高度な検出と調査のため、Azure SQL Server とデータベースへのアクセスの監査を有効にすることをお勧めします。|
| SQL の透過的なデータ暗号化| データがセキュリティ侵害を受けた場合でも読み取れないようするため、Azure SQL データベースと関連付けられたバックアップおよびトランザクション ログファイルに対し、REST での暗号化を有効にすることをお勧めします。|

10. すべてのオプションの構成が完了したら、**[保存]** をクリックしてこれらの変更を確定します。

> [AZURE.NOTE] データ収集を有効にすると、監視エージェントがインストールされます。 この場所からのデータ収集を今すぐ有効にしない場合は、後からヘルス ビューと推奨事項のビューから有効にすることができます。 データ収集は、サブスクリプションや選択した VM だけに有効にすることもできます。 参照してください、 [Azure セキュリティ センターに関する FAQ](security-center-faq.md) サポートされている Vm についての詳細を知る。
