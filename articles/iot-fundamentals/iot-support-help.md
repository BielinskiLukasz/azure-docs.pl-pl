# <a name="azure-iot-support-and-help-options"></a>Opcje wsparcia technicznego i pomocy usługi Azure IoT

Poniżej przedstawiono sugestie dotyczące tego, gdzie można uzyskać pomoc podczas opracowywania rozwiązań Azure IoT.

## <a name="create-an-azure-support-request"></a>Tworzenie wniosku o pomoc techniczną platformy Azure

<div class='icon is-large'>
    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

Zapoznaj się z zakresem [opcji pomocy technicznej platformy Azure i wybierz plan,](https://azure.microsoft.com/support/plans) który najlepiej pasuje, niezależnie od tego, czy jesteś programistą, który dopiero rozpoczyna podróż w chmurze, czy dużą organizacją wdrażającą aplikacje strategiczne o znaczeniu krytycznym dla firmy. Klienci platformy Azure mogą tworzyć żądania pomocy technicznej i zarządzać nimi w witrynie Azure portal.

* [Portal Azure](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [Portal Azure dla rządu Stanów Zjednoczonych](https://portal.azure.us)

## <a name="post-a-question-on-stack-overflow"></a>Zamieść pytanie na temat Przepełnienia stosu

<div class='icon is-large'>
    <img alt='Stack Overflow' src='https://docs.microsoft.com/media/logos/logo_stackoverflow.svg'>
</div>

Aby uzyskać natychmiastowe odpowiedzi od społeczności, poszukaj przepełnienia stosu. Przepełnienie stosu jest preferowanym kanałem **pytań związanych z rozwojem**. To właśnie w tym miejscu członkowie społeczności i członkowie zespołu firmy Microsoft są bezpośrednio zaangażowani w pomoc w rozwiązywaniu problemów.

Jeśli nie możesz znaleźć odpowiedzi na problem za pomocą wyszukiwania, prześlij nowe pytanie do Przepełnienie stosu. Podczas formułowanie pytania należy użyć jednego z następujących tagów:

 - [Azure Digital Twins](https://stackoverflow.com/questions/tagged/azure-digital-twins)
 - [Azure IoT Central](https://stackoverflow.com/questions/tagged/azure-iot-central)
 - [Azure IoT Edge](https://stackoverflow.com/questions/tagged/azure-iot-edge)
 - [Azure IoT Hub](https://stackoverflow.com/questions/tagged/azure-iot-hub)
 - [Mapy platformy Azure](https://stackoverflow.com/questions/tagged/azure-maps)
 - [Zestawy SDK usługi Azure IoT](https://stackoverflow.com/questions/tagged/azure-iot-sdk)
 - [Azure Time Series Insights](https://stackoverflow.com/questions/tagged/azure-timeseries-insights)

## <a name="start-a-conversation-on-the-msdn-forums"></a>Rozpoczynanie konwersacji na forach MSDN

Aby omówić technologie i usługi platformy Azure do tworzenia rozwiązań IoT. MSDN jest preferowanym kanałem **pytań związanych z rozwiązaniami.** To właśnie w tym miejscu członkowie społeczności i członkowie zespołu firmy Microsoft są bezpośrednio zaangażowani w pomoc w wyborze odpowiedniego rozwiązania dla twojego projektu.

Jeśli nie możesz znaleźć odpowiedzi na problem za pomocą wyszukiwania, prześlij nowe pytanie do MSDN na forum związanym z usługami:

- [Azure Digital Twins](https://social.msdn.microsoft.com/Forums/en-US/home?forum=azuredigitaltwins)
- [Azure IoT Hub](https://social.msdn.microsoft.com/Forums/en-US/home?forum=azureiothub)
- [Azure IoT Edge](https://social.msdn.microsoft.com/Forums/en-US/home?forum=iotedge)
- [Mapy platformy Azure](https://social.msdn.microsoft.com/Forums/en-US/home?forum=azurelbs)
- [Azure Time Series Insights](https://social.msdn.microsoft.com/Forums/en-US/home?forum=AzureTimeSeriesInsights)
