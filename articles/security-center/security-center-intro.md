# Azure Security Center 入門

Azure Security Center の主な機能とそのしくみについて説明します。

> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。 これは、サンプルのデプロイメントを使用してサービスを紹介するもので、

## Azure Security Center とは
Azure Security Center は、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。 これにより、Azure サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

##  主な機能
Azure Security Center は、脅威の回避、検出、対応を容易かつ効果的に行う Azure 組み込みの機能を提供します。 主な機能は次のとおりです。

| | |
|----- |-----|
| 回避 | Azure リソースのセキュリティの状態を監視します。 |
| | 会社のセキュリティ要件と、アプリケーションの種類やデータの機密性に基づいて、Azure サブスクリプションのポリシーを定義します。 |
| | ポリシーに基づいたセキュリティに関する推奨事項を使用して、サービスの所有者が必要な制御を実装するプロセスをサポートします。 |
| | Microsoft やパートナーからのセキュリティ サービスやアプライアンスを迅速にデプロイします。 |
| 検出 | Azure のリソース、ネットワーク、およびマルウェア対策とファイアウォールのようなパートナー ソリューションから、自動的にセキュリティ データを収集して分析します。 |
| | Microsoft 製品とサービス、デジタル犯罪やインシデント応答センター、および外部フィードからのグローバルな脅威情報を活用します。 |
| | 機械学習や動作分析などの高度な分析を適用します。 |
| 対応 | 優先順位付きのセキュリティ インシデントやアラートを提供します。 |
| | 攻撃元や影響を受けたリソースを分析します。 |
| | 現在の攻撃を阻止し、将来の攻撃を防止する方法を示します。 |

## 入門チュートリアル
Azure セキュリティ センターはからアクセス、 [Azure ポータル](http://azure.microsoft.com/features/azure-portal/)します。 アクセスに [、ポータルにサインイン](https://portal.azure.com), を選択 **参照**, 、までスクロールし、 **セキュリティ センター** オプションを選択するか、 **セキュリティ センター** 以前ポータルのダッシュ ボードに固定されているタイル。

![][1]

Azure Security Center からセキュリティ ポリシーの設定、セキュリティ構成の監視、セキュリティ アラートの表示を行えます。

### セキュリティ ポリシー

セキュリティに関する会社の要件、各サブスクリプションでのアプリケーションのタイプやデータの機密度に合わせて、Azure サブスクリプションのポリシーを定義できます。 たとえば、開発またはテストに使用されるリソースには、実稼働アプリケーションで使われるものと異なるセキュリティ要件があります。 同様に、PII のような規制されたデータを持つアプリケーションには、より高いレベルのセキュリティが必要です。

> [AZURE.NOTE] セキュリティ ポリシーを編集するには、所有者またはサブスクリプションの共同作業者があります。

クリックして、 **セキュリティ ポリシー** お客様のサブスクリプションの一覧についてはタイルし、ポリシーの詳細を表示するサブスクリプションを選択します。  

![][2]

**データの収集** (上記参照) はセキュリティ ポリシーについてのデータ コレクションを有効にします。 有効にすると、次のことが行われます。
- セキュリティの監視と推奨事項の仮想マシンをサポートするすべての日常的にスキャン
- 分析と脅威の検出のためのセキュリティ イベントの収集

**推奨事項を表示する:** (上記参照) を使用して、サブスクリプション内のリソースのセキュリティ ニーズに基づいて監視し、推奨するセキュリティ制御を選択します。

### セキュリティに関する推奨事項

Azure Security Center は Azure リソースのセキュリティの状態を分析して、潜在的なセキュリティ脆弱性を特定します。 推奨事項の一覧では、必要な制御を構成する手順を説明します。 たとえば、次のようになります。

- 悪意のあるソフトウェアを識別して削除するためのマルウェア対策をプロビジョニングする
- 仮想マシンへのトラフィックを制御するためにネットワーク セキュリティ グループと規則を構成する
- Web アプリケーションを対象とする攻撃から保護するための Web アプリケーション ファイアウォールをプロビジョニングする
- 不足しているシステムの更新をデプロイする
- 推奨基準と一致しない OS 構成に対処する

をクリックして、 **の推奨事項** の推奨事項の一覧については並べて表示します。 各推奨事項をクリックすることで、追加の情報を表示したり、問題を解決するアクションを実行したりできます。

![][3]

### リソース ヘルス

 **リソース正常性** タイル リソースの種類の仮想マシン、web アプリケーションおよびその他のリソースで、環境の全体的なセキュリティ対策を示しています。   

リソースの種類を選択して、 **リソース正常性** 識別されている潜在的なセキュリティの脆弱性の一覧を含む詳細 (次の例での仮想マシン) を表示するタイルです。

![][4]

### セキュリティのアラート

Azure Security Center は、Azure のリソース、ネットワーク、およびマルウェア対策とファイアウォールのようなパートナー ソリューションから、自動的にログ データを収集して分析し、統合します。 脅威が検出されると、セキュリティのアラートが作成されます。 たとえば、次のような項目が検出されます。

- 既知の悪意のある IP アドレスと通信する、セキュリティ侵害された仮想マシン
- Windows エラー報告を使用して検出された高度なマルウェア
- 仮想マシンに対するブルート フォース攻撃
- 統合されたマルウェア対策とファイアウォールからのセキュリティのアラート

クリックすると、 **セキュリティの警告** タイルには、優先順位のアラートの一覧が表示されます。

![][5]

アラートを選択すると、攻撃や修復方法に関する提案について詳しく表示されます。

![][6]

## 作業開始
Azure Security Center を使用するには、Microsoft Azure のサブスクリプションが必要です。 Azure Security Center は、Azure サブスクリプションがある場合に有効です。 サインアップすることができます、サブスクリプションを持っていない場合、 [無料評価版](https://azure.microsoft.com/pricing/free-trial)します。

Azure セキュリティ センターはからアクセス、 [Azure ポータル](http://azure.microsoft.com/features/azure-portal/)します。 参照してください [ポータルのドキュメント](https://azure.microsoft.com/documentation/services/azure-portal/) の詳細。
