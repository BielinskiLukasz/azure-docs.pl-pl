# Azure ストレージ アカウントの一覧 #

Azure ストレージ アカウントは、JDK、アプリケーション サーバー、および任意のコンポーネントのためだけでなく、キャッシュを使用する際に、状態を格納するために使用されるダウンロードの場所を提供します。 Eclipse では、Eclipse ワークスペース内のプロジェクトで使用可能な既知のストレージ アカウントの一覧が保持されます。 開くには、 **ストレージ アカウント** eclipse で、一覧の管理に使用されるダイアログでクリックして **ウィンドウ**, 、] をクリックして **の基本設定**, 、展開 **Azure**, 、] をクリックし、 **ストレージ アカウント**します。

次に示す、 **ストレージ アカウント** ダイアログ。

![][ic719496]

このダイアログ ボックスを開くことも、 **アカウント** 次などのストレージ アカウントを使用するダイアログ ボックスにリンクします。

*  **JDK** のタブ、 **サーバー構成** ダイアログ。
*  **Server** のタブ、 **サーバー構成** ダイアログ。
*  **コンポーネントの追加** ダイアログ。
*  **キャッシュ** プロパティ] ダイアログ ボックス。

## 発行設定ファイルを使用して、ストレージ アカウントをインポートするには ##

1. 内で、 **ストレージ アカウント** ダイアログ ボックスで、をクリックして **発行設定ファイルからインポート**します。
2. (ローカル コンピューターへ既に発行設定ファイルを保存している場合、この手順はスキップしてください。) **Import Subscription Information** ダイアログ ボックスで、をクリックして **発行設定ファイルのダウンロード**します。 まだ Azure アカウントにログインしていない場合は、ログインするように求められます。 その後、Azure 発行設定ファイルを保存するよう求めるメッセージが表示されます。 (結果としてログオン ページに表示される手順は無視することができます。それらは、Azure ポータルによって提供される、Visual Studio ユーザー向けの手順です。)このファイルは、ローカル コンピューターに保存してください。
3. 引き続き、 **サブスクリプション情報のインポート** ダイアログ ボックスで、をクリックして、 **参照** ボタンをクリックし、ローカルに保存した以前は、発行設定ファイルを選択し、をクリックして **開く**します。
4. をクリックして **OK** を閉じる、 **サブスクリプション情報のインポート** ダイアログ。

## 新しいストレージ アカウントを作成するには ##

1. 内で、 **ストレージ アカウント** ダイアログ ボックスで、をクリックして **追加**します。
2. 内で、 **ストレージ アカウントの追加** ダイアログ ボックスで、をクリックして **新規**します。
3. 内で、 **新しいストレージ アカウント** ] ダイアログ ボックスで、次の値を指定します。
    * ストレージ アカウント名。
    * ストレージ アカウントの場所。
    * ストレージ アカウントの説明。
    * ストレージ アカウントが属しているサブスクリプション。
4. をクリックして **OK** を閉じる、 **新しいストレージ アカウント** ダイアログ。

ストレージ アカウントの作成には数分かかる場合があります。 作成すると、クリックして **[ok]** を閉じる、 **ストレージ アカウントの追加** ダイアログ、および、新しいストレージ アカウントは、使用可能なストレージ アカウントの一覧に追加されます。

## 既存のストレージ アカウントを一覧に追加するには ##

1. Azure ストレージ アカウントがあるない場合は作成に示された手順に従って、 **新しいストレージ アカウント] セクションを作成する** 上です。 (またはで新しいストレージ アカウントを作成することができます、 [Azure 管理ポータル][].)
2. 内で、 **ストレージ アカウント** ダイアログ ボックスで、をクリックして **追加**します。
3. 内で、 **ストレージ アカウントの追加** ] ダイアログ ボックスで値を入力 **名** と **アクセス キー**します。 アカウント名とアクセス キーは、既存の Azure ストレージ アカウントのものである必要があります。 使用して、 **ストレージ** のセクションで、 [Azure 管理ポータル][] 、ストレージ アカウント名とキーを表示します。  **ストレージ アカウントの追加** ダイアログは、次のようになります。

    ![][ic719497]

4. をクリックして **OK** を閉じる、 **ストレージ アカウントの追加** ダイアログ。

## 新しいアクセス キーを使用するためにストレージ アカウントを変更するには ##

1. 内で、 **ストレージ アカウント** ダイアログ ボックスで、ストレージ アカウントを編集する] をクリックし、 **編集**します。
2. 内で、 **ストレージ アカウント アクセス キーの編集** ダイアログ ボックスで、変更、 **アクセス キー** 値。
3. をクリックして **OK** を閉じる、 **ストレージ アカウント アクセス キーの編集** ダイアログ。

## Eclipse で保持されている一覧からストレージ アカウントを削除するには ##

1. 内で、 **ストレージ アカウント** ダイアログ ボックスで、ストレージ アカウントを編集する] をクリックし、 **削除**します。
2. クリックして **OK** ストレージ アカウントを削除するように求められたらします。

>[AZURE.NOTE] ストレージ アカウントを削除して、 **ストレージ アカウント** ダイアログからのみ削除 Eclipse 内で表示できるストレージ アカウントの一覧です。 ストレージ アカウントは、Azure サブスクリプションからは削除されません。 また、ストレージ アカウントは、Eclipse でサブスクリプションの詳細を再読み込みした後に、再度表示される場合があります。
