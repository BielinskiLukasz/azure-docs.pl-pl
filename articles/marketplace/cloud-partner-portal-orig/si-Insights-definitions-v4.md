<a name="seller-insights-definitions"></a>Definicje Insights sprzedawcy
=======================

Poniższa tabela zawiera definicje dla wielu terminów używanych w sprzedawcy szczegółowych informacji.

|  **Termin**                 |  **Definicja**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Typ licencji platformy Azure**                               | Typ umowy licencyjnej, używany przez klientów do zakupienia usług Azure.  Znany także jako kanał.                                                  |
| **Typ licencji platformy Azure: Cloud Solution Provider**      | Odbiorcy końcowego pozyskuje platformy Azure i oferty w portalu Marketplace za pośrednictwem ich dostawcy rozwiązań w chmurze, który działa jako swojego odsprzedawcy.                 |
| **Typ licencji platformy Azure: Enterprise**                   | Odbiorcy końcowego pozyskuje platformy Azure i oferty w portalu Marketplace za pośrednictwem umowy Enterprise Agreement podpisaną bezpośrednio od firmy Microsoft.                  |
| **Typ licencji platformy Azure: Przedsiębiorstwa za pośrednictwem odsprzedawcy**  | Odbiorcy końcowego pozyskuje platformy Azure i oferty w portalu Marketplace za pośrednictwem odsprzedawcy, który ułatwia tworzenie umów Enterprise Agreement z firmą Microsoft.     |
| **Typ licencji platformy Azure: Płatność zgodnie z rzeczywistym użyciem**                | Odbiorcy końcowego pozyskuje platformy Azure i oferty w portalu Marketplace za pośrednictwem "Płatność zgodnie z rzeczywistym użyciem" podpisanej bezpośrednio od firmy Microsoft.                |
| **Kwota opłaty (DW)**                              | Kwota obciążany opłatą za klienta, w *walutę* (DW) do rozliczeń.                                                                 |
| **Kwota opłaty (PC)**                               | Kwota obciążany opłatą za klienta, w *waluty informacje dotyczące wypłat* (PC).                                                                      |
| **Data obciążenia**                                      | Data opłat klienta obliczane (zazwyczaj od razu po okresie użytkowania).                                             |
| **Nazwa wystąpienia chmury**                              | Microsoft Cloud wystąpił podczas wdrażania maszyny Wirtualnej.                                   |
| **Nazwa wystąpienia chmury: Azure Global**                | Globalne Microsoft chmury publicznej.                           |
| **Nazwa wystąpienia chmury: Platforma Azure Government**        | Chmury firmy Microsoft przeznaczonych dla instytucji rządowych dla jednego z następujących instytucje rządowe: Chiny (Niemcy) i Stanów Zjednoczonych.                           |
| **Stan kolekcji**  | Określone opłatę, najnowszy stan cyklu życia Pomoc dotycząca rozliczeń i kolekcji.  Na przykład stan może być zbieranie w toku, zebrane lub refundacji.                  |
| **Rozmiar podstawowej**  | Liczba rdzeni wirtualnych przedstawiony przez maszynę wirtualną, którą jest zasilany z tej oferty.               |
| **Miasto klienta**  | Nazwa miasta, udostępniane przez klienta. Może to być inny niż miasta w subskrypcji platformy Azure klienta.  |
| **Język komunikacji klienta**        | Język preferowany przez klienta do komunikacji.                                  |
| **Nazwa firmy klienta**                  | Nazwa firmy, udostępniane przez klienta. Może to być inny niż miasta w subskrypcji platformy Azure klienta.                                  |
| **Kraju klienta**                       | Nazwa kraju/regionu, udostępniane przez klienta.  Może to być inny niż kraj/region, w przypadku subskrypcji Azure klienta.                               |
| **Waluta odbiorcy (DW)**                 | Waluta preferowane przez klienta na temat cen i rozliczeń.                            |
| **Klient poczty E-mail**                         | Podany adres e-mail przez klienta końcowego.  Może to być inny niż adres e-mail w przypadku subskrypcji Azure klienta.                   |
| **Imię klienta**                              | Imię udostępniane przez klienta.  Może to być inna niż nazwa podana w subskrypcji platformy Azure klienta.                    |
| **Nazwisko klienta**                               | Ostatnia nazwa podana przez klienta.  Może to być inna niż nazwa podana w subskrypcji platformy Azure klienta.                    |
| **Identyfikator klienta**                               | Unikatowy identyfikator przypisany do klienta.  Klient może mieć zero lub więcej subskrypcji portalu Marketplace platformy Azure.                    |
| **Typ płatności odbiorcy**                            | Typ instrumentu płatniczego, używany przez klienta.  Na przykład to być karty, faktury lub wyboru elektronicznych.                             |
| **Kod pocztowy klienta**                             | Kod pocztowy, udostępniane przez klienta.  Może to być inny niż kod pocztowy w subskrypcji platformy Azure klienta.                                                                 |
| **Stan klienta**                                   | Stan (adres), udostępniane przez klienta.  Może to być inny niż ostatni nazwa podana w subskrypcji platformy Azure klienta.                                                                 |
| **Data przyjęcia**                                    | Data pierwszego klienta zakupić dowolnej oferty, które zostały opublikowane przez Ciebie.                 |
| **Data utraty**                                        | Data ostatniego klienta anulowane ostatniego wszystkich ofert z wcześniej kupił.                                                         |
| **Szacowana opłata rozszerzone (DW)**                   | Szacowane rozszerzone opłaty za ilość jednostek użycia dla danej jednostki SKU (w walucie klienta). Ta wartość nie może być dokładne z powodu błędów zaokrąglania lub obcięcie.                            |
| **Szacowana opłata rozszerzone (PC)**                   | Szacowany rozszerzone opłaty za ilość jednostek użycia dla danej jednostki SKU oparte na konwersji walutowych użycia Data jest obliczana (w walucie wydawcy). Ta wartość nie może być dokładne z powodu błędów zaokrąglania lub obcięcie.                  |
| **Informacje dotyczące wypłat szacowany (PC)**                            | Szacowana opłata za ilość jednostek użycia dla danej jednostki SKU oparte na konwersji walutowych w dniu, w których użycie jest obliczane (w walucie wydawcy). Ta wartość nie może być dokładne z powodu błędów zaokrąglania lub obcięcie.                        |
| **Szacowana cena (PC)**                             | Szacowana cena jednostki użycia dla danej jednostki SKU oparte na konwersji walutowych użycia Data jest obliczana (w walucie wydawcy). Ta wartość nie może być dokładne z powodu błędów zaokrąglania lub obcięcie.                                             |
| **To nowy klient**                                  | Wartość zostanie wyświetlone powiadomienie, jeśli jest nowego klienta używania lub zakupu jeden lub więcej z portalu Marketplace oferuje po raz pierwszy (lub nie).  Wartość będzie "Yes", jeśli jest tym samym miesiącu kalendarzowym dla "Data nabyte".  Wartość będzie "No", jeśli klient kupił znajdujących się w Twoich ofert przed miesiąca kalendarzowego zgłaszane.                                                       |
| **Jest jednostki SKU w wersji zapoznawczej**                                  | Wartość zostanie wyświetlone powiadomienie, jeśli oznaczyła jednostkę SKU jako "wersja zapoznawcza". Wartość będzie "Tak", jeśli zostały odpowiednio oznakowane jednostki SKU, a tylko subskrypcje autoryzowanych przez użytkownika można wdrożyć i użyć tego obrazu.  Wartość będzie "No", jeśli jednostka SKU nie została zidentyfikowana jako "wersja zapoznawcza".                                          |
| **Jest kontakt promocyjne zgodzić się na**                   | Wartość zostanie wyświetlone powiadomienie, jeśli klient aktywnie zgoda promocyjne kontaktu z wydawcy. W tej chwili jest nie wyświetlana opcja dla klientów, więc wskazują "No", w tablicy. Po wdrożeniu tej funkcji, firma Microsoft rozpocznie się aktualizowanie odpowiednio.                                          |
| **Typ licencji w portalu Marketplace**                         | Metoda rozliczeń oferty w portalu Marketplace.                             |
| **Typ licencji w portalu Marketplace: Rozliczane za pośrednictwem platformy Azure**   | Firma Microsoft jest agenta dla tej oferty w portalu Marketplace oraz rozlicza klientów w Twoim imieniu. (PAYG karty kredytowej lub faktury Enterprise)       |
| **Typ licencji w portalu Marketplace: Model dostarczania własnej licencji** | Maszyna wirtualna wymaga klucza licencji dostarczonych przez klienta do wdrożenia. Firma Microsoft rozlicza klientów za użycie ofert w witrynie Marketplace. |
| **Typ licencji w portalu Marketplace: Bezpłatne**                   | Oferty w portalu Marketplace jest skonfigurowane jako bezpłatna dla wszystkich użytkowników. Firma Microsoft rozlicza klientów do ich użycia tej oferty w portalu Marketplace.    |
| **Typ licencji w portalu Marketplace: Odsprzedawcy firmy Microsoft**  | Firma Microsoft jest odsprzedawcy dla tej oferty w portalu Marketplace.                               |
| **Nazwa firmy subskrypcji portalu Marketplace**  | Nazwa firmy, udostępniane przez klienta w ramach swojej subskrypcji platformy Azure.                           |
| **Identyfikator subskrypcji portalu Marketplace**            | Unikatowy identyfikator skojarzonego z subskrypcją Azure klienta wykorzystać na zakup oferty w portalu Marketplace.  Znajdowało się identyfikator GUID subskrypcji platformy Azure.  W niektórych przypadkach, takich jak kolejność SAAS subskrypcja nie jest wymagane.           |
| **Opłata za firmy Microsoft (DW)**                               | Opłata Microsoft transakcji w walucie klienta.                              |
| **Nazwa oferty**                                | Nazwa oferty w portalu Marketplace.                                                             |
| **Typ oferty**                                | Typ oferty Microsoft Marketplace.  Na przykład może być aplikacja zarządzana, licencji maszyny wirtualnej lub kontenera.                           |
| **Data anulowania zamówienia**                         | Data zamówienia witryny Marketplace zostało anulowane.                                                       |
| **Identyfikator zamówienia**                                  | Unikatowy identyfikator zamówienia klienta dla usługi witryny Marketplace.  Oferty opartej na użyciu maszyny wirtualnej nie są skojarzone z zamówieniem.                 |
| **Data zakupu zamówienia**                       | Data utworzenia zamówienie witryny Marketplace.                                                        |
| **Ilość zamówienia**                       | Liczba zamówień w ramach tej samej jednostki SKU skojarzony z subskrypcją platformy Azure.  Dla zamówienia SAAS, czyli przedstawia liczbę stanowisk skojarzony z tym wystąpieniem zamówienie.                                            |
| **Stan zlecenia**                              | Stan zamówienia witryny Marketplace, w czasie ostatniego odświeżania danych.  Na przykład kolejność może być aktywne lub anulowane.             |
| **Stan zamówienia: Aktywne**                           | Klient kupił zamówienie i nie zostało anulowane ich kolejność.   |
| **Stan zamówienia: Anulowane**                        | Klient wcześniej kupił zamówienie i następnie anulować ich kolejność.        |
| **Kwoty wypłata (PC)**                             | Kwoty wypłacane, w preferowanym informacje dotyczące wypłat waluty (komputer).                                       |
| **Informacje dotyczące wypłat waluty (PC)**                           | Waluta używana dla Twojego wypłaty.                                                                |
| **Informacje dotyczące wypłat daty**                                    | Data płatności żądanie zostało wysłane przez firmę Microsoft do wybranej instytucji finansowych.                                                                         |
| **Informacje dotyczące wypłat stanu**                                  | Stan informacje dotyczące wypłat w czasie ostatniego odświeżania danych.                                   |
| **Informacje dotyczące wypłat stanu: W toku**               | Transakcja nie jest gotowy, aby uzyskać informacje dotyczące wypłat. (Aby uzyskać więcej informacji, zobacz stan kolekcji)                             |
| **Informacje dotyczące wypłat stanu: Wypłacane**                          | Transakcja została uwzględniona w ciągu ostatnich obliczeniach informacje dotyczące wypłat. Płatne wartości dodatnich i wartości ujemne są kompensowane łączna kwota. |
| **Informacje dotyczące wypłat stanu: Nadchodzące wypłaty**                   | Transakcja jest gotowy, aby uzyskać informacje dotyczące wypłat i zostaną uwzględnione w następnym obliczeń dostępne informacje dotyczące wypłat.                      |
| **Cena (DW)**                                   | Cena jednostki użycia dla danej jednostki SKU (w walucie klienta).                                                                  |
| **Dostawcy poczty E-mail**                                 | Adres e-mail dostawcy uczestniczących w relacji między firmami Microsoft i odbiorcy końcowego. Jeśli klient jest w przedsiębiorstwie za pośrednictwem odsprzedawcy, będzie odsprzedawcy.  Jeśli uczestniczy Cloud Solution Provider (CSP), będzie ona dostawcy usług Kryptograficznych.                                                                 |
| **Nazwa dostawcy**                                 | Nazwa dostawcy uczestniczących w relacji między klienta firmy Microsoft i zakończenia. Jeśli klient jest w przedsiębiorstwie za pośrednictwem odsprzedawcy, będzie odsprzedawcy.  Jeśli uczestniczy Cloud Solution Provider (CSP), będzie ona dostawcy usług Kryptograficznych.                                                                 |
| **Identyfikator URI zasobu**                                  | Ścieżka wdrożonym artefaktów wdrożenia.                                            |
| **SKU**                                           | Nazwa jednostki SKU, zgodnie z definicją podczas publikowania. Oferta może mieć wiele jednostek SKU, ale jednostki SKU może być skojarzony tylko z jednej oferty.              |
| **Typ rozliczeń jednostki SKU**                                 | Metody rozliczeń jednostki SKU.                                                                                                             |
| **Typ rozliczeń jednostki SKU: Model dostarczania własnej licencji**         | Maszyna wirtualna lub usługa wymaga klucza licencji dostarczonych przez klienta do wdrożenia. Firma Microsoft rozlicza klientów do ich użycia tych ofert portalu Marketplace.   |
| **Typ rozliczeń jednostki SKU: Microsoft As Reseller**          | Firma Microsoft jest odsprzedawcy w ramach tej jednostki SKU.   |
| **Typ rozliczeń jednostki SKU: Bezpłatne**                           | Jednostka SKU jest skonfigurowane jako bezpłatna dla wszystkich użytkowników. Firma Microsoft rozlicza klientów do ich użycia tej jednostki SKU.                           |
| **Typ rozliczeń jednostki SKU: Płatne**                           | Firma Microsoft jest agenta dla tej jednostki SKU i rozlicza klientów w Twoim imieniu. (PAYG karty kredytowej lub faktury Enterprise)                   |
| **Typ rozliczeń jednostki SKU: Trial**                          | Klient znajduje się w ich okresu próbnego i zostanie przekonwertowana na płatną, jeśli nie anulować lub usunąć.                                         |
| **Kwota podatku (DW)**                                  | Kwota podatku stosowana do rozliczenia za klienta w walucie klienta (DW).                                                               |
| **Data transakcji**                                 | Data transakcji są rejestrowane w informacje dotyczące wypłat raportowania.                                                                           |
| **Typ transakcji**                               | Typ transakcji, który jest zgłaszany.  Na przykład to być opłaty, zwrotu lub informacje dotyczące wypłat dopasowania.                                                                                        |
| **Data końcowa wersja próbna**                                 | Data to zamówienie okres próbny zakończy się lub została zakończona.                                                                           |
| **Data wykorzystania**                                       | Użycie klienta Data wystąpił.                                                                                                             |
| **Ilość użycia**                                       | Użycie klienta zgłoszone jednostki SKU. W przypadku obrazów maszyn wirtualnych rekordów użycia reprezentują użycia zgłoszonych okres tego rozmiaru maszyny Wirtualnej i jednostki SKU.                                               |
| **Odwołanie do użycia**                                       | Identyfikator dla jednego lub więcej dni użycia przez klientów dla danej jednostki SKU, skojarzone z wpisem w raporcie informacje dotyczące wypłat.                                               |
| **Typ użycia**                                       | Opis użycia mierzony. (Użycie znormalizowanych lub nieprzetworzonych użycia)                                                              |
| **Typ użycia: Użycie znormalizowanych**                | Godziny użycia znormalizowane do konta dla liczby rdzeni maszyn wirtualnych, związane z użycia: [rdzeni liczbę maszyn wirtualnych] x [godzin z pierwotnych Usage]. Maszyny wirtualne, oznaczone jako "SHAREDCORE" Użyj 1/6 (lub 0.1666) mnożnik [rdzeni liczbę maszyn wirtualnych].                                        |
| **Typ użycia: Nieprzetworzone obciążenie**                        | Godziny użycia dla oferty w portalu Marketplace.  Są one przedstawiane w godzinach "Kalendarz".  Jest działały liczby maszyn wirtualnych w czasie.                           |
| **Użycie jednostek**                                   | Jednostka miary dla określonego użycia. Maszyny wirtualne są zawsze mierzona z jednostki godzinowe miary.                               |
| **Rozmiar maszyny Wirtualnej**                                       | Reprezentuje rozmiar sprzętu maszyny wirtualnej, które są wyrównane z ofertą platformy Azure. Przykłady obejmują `Basic_A0`, `Standard_A11`, `Standard_D12`, i `Standard_G4`.   |
|  |  |
