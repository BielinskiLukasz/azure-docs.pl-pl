# <a name="understanding-azure-virtual-machine-usage"></a>Opis użycia maszyny wirtualnej platformy Azure
Analizując dane użycia platformy Azure, można uzyskać zaawansowane statystyki zużycia — szczegółowe informacje, które umożliwiają lepsze zarządzanie kosztami i alokację w całej organizacji. Ten dokument zawiera szczegółowe informacje o zużycie danych dotyczących obliczeń usługi Azure. Aby uzyskać więcej informacji na temat ogólnego użycia platformy Azure, przejdź do [pozycji Opis rachunku](https://docs.microsoft.com/azure/billing/billing-understand-your-bill).

## <a name="image-type"></a>Typ obrazu
W przypadku niektórych obrazów w galerii platformy Azure typ obrazu jest wypełniany w polu Informacje dodatkowe. Dzięki temu użytkownicy mogą zrozumieć i śledzić, co wdrożyli na swojej maszynie wirtualnej. Następujące wartości, które są wypełniane w tym polu na podstawie wdrożonego obrazu:
  - Skała BitRock 
  - Canonical 
  - FreeBSD 
  - Otwórz logikę 
  - Oracle 
  - SLES dla SAP 
  - Sql Server 14 Preview w systemie Windows Server 2012 R2 Preview 
  - SUSE
  - SUSE Premium
  - Urządzenie w chmurze StorSimple 
  - Red Hat
  - Czerwony kapelusz dla aplikacji biznesowych SAP     
  - Czerwony kapelusz dla SAP HANA 
  - Klient systemu Windows BYOL 
  - Windows Server BYOL 
  - Wersja zapoznawcza systemu Windows Server 

## <a name="service-type"></a>Typ usługi
Pole typu usługi w polu Informacje dodatkowe odpowiada dokładnie wdrożonemu rozmiarowi maszyny Wirtualnej. Maszyny wirtualne magazynu w stanie Premium (oparte na dyskach SSD) i maszyny wirtualne magazynu niesadunkowego (oparte na dyskach twardych) są wyceniane tak samo. W przypadku wdrożenia rozmiaru opartego na\_ssd, takiego jak\_Standard DS2 v2,\_w\_polu Informacje dodatkowe w polu Informacje dodatkowe w polu Informacje dodatkowe zostanie\_wyświetlony\_rozmiar nienawiązki do serwera SSD ("Standardowa maszyna wirtualna D2 v2") oraz rozmiar SSD ("Standardowy DS2 v2".

## <a name="virtual-machine-usage-faq"></a>Często zadawane pytania dotyczące użycia maszyny wirtualnej
### <a name="what-resources-are-charged-when-deploying-a-vm"></a>Jakie zasoby są naliczane podczas wdrażania maszyny Wirtualnej?    
Maszyny wirtualne pozyskują koszty dla samej maszyny Wirtualnej, dowolnego oprogramowania premium uruchomionego na maszynie wirtualnej, konta magazynu\zarządzanego dysku skojarzonego z maszyną wirtualną oraz transferów przepustowości sieci z maszyny Wirtualnej.
### <a name="how-can-i-tell-if-a-vm-is-using-azure-hybrid-benefit-in-the-usage-csv"></a>Jak sprawdzić, czy maszyna wirtualna korzysta z korzyści hybrydowych platformy Azure w pliku CSV użycia?
Jeśli wdrożysz przy użyciu [korzyści hybrydowej platformy Azure,](https://azure.microsoft.com/pricing/hybrid-benefit/)naliczana jest opłata za maszynę wirtualną z systemem innych niż Windows, ponieważ wprowadzasz własną licencję do chmury. Na rachunku można odróżnić maszyny wirtualne Menedżera zasobów, które korzystają z\_korzyści hybrydowych platformy\_Azure, ponieważ mają one "Windows Server BYOL" lub "Windows Client BYOL" w kolumnie ImageType.
### <a name="how-are-basic-vs-standard-vm-types-differentiated-in-the-usage-csv"></a>W jaki sposób podstawowe i standardowe typy maszyn wirtualnych są zróżnicowane w pliku CSV użycia?
Oferowane są zarówno maszyny wirtualne klasy podstawowej, jak i standardowej serii A. Jeśli wdrożysz podstawową maszynę wirtualną, w podkategorii miernika ma ciąg "Basic". Jeśli wdrożysz standardową maszynę wirtualną z serii A, rozmiar maszyny Wirtualnej jest wyświetlany jako "Maszyna wirtualna A1", ponieważ standard jest wartością domyślną. Aby dowiedzieć się więcej o różnicach między standardem podstawowym a standardowym, zobacz [stronę cennika](https://azure.microsoft.com/pricing/details/virtual-machines/).
### <a name="what-are-extrasmall-small-medium-large-and-extralarge-sizes"></a>Co to są rozmiary ExtraSmall, Small, Medium, Large i ExtraLarge?
ExtraSmall - ExtraLarge to starsze\_nazwy standardu A0 – Standard\_A4. W klasycznych rekordach użycia maszyn wirtualnych może zostać wyświetlona ta konwencja używana, jeśli te rozmiary zostały wdrożone.
### <a name="what-is-the-difference-between-meter-region-and-resource-location"></a>Jaka jest różnica między regionem licznika a lokalizacją zasobów?
Region miernika jest skojarzony z miernikiem. W przypadku niektórych usług platformy Azure, które używają jednej ceny dla wszystkich regionów, pole Region miernika może być puste. Jednak ponieważ maszyny wirtualne mają dedykowane ceny za region dla maszyn wirtualnych, to pole jest wypełniane. Podobnie lokalizacja zasobów dla maszyn wirtualnych jest lokalizacją, w której jest wdrażana maszyna wirtualna. Regiony platformy Azure w obu polach są takie same, chociaż mogą mieć inną konwencję ciągu dla nazwy regionu.
### <a name="why-is-the-imagetype-value-blank-in-the-additional-info-field"></a>Dlaczego wartość ImageType jest pusta w polu Informacje dodatkowe?
Pole ImageType jest wypełniane tylko dla podzbioru obrazów. Jeśli nie wdrożono jednego z powyższych obrazów, typ obrazu jest pusty.
### <a name="why-is-the-vmname-blank-in-the-additional-info"></a>Dlaczego nazwa VMName jest pusta w informacjach dodatkowych?
Nazwa VMname jest wypełniona tylko w polu Informacje dodatkowe dla maszyn wirtualnych w zestawie skalowania. Pole InstanceID zawiera nazwę maszyny wirtualnej dla maszyn wirtualnych zestawu nieskażkowego.
### <a name="what-does-computehr-mean-in-the-usagetype-field-in-the-additional-info"></a>Co oznacza computeHR w polu UsageType w polu Dodatkowe informacje?
ComputeHR oznacza Compute Hour, który reprezentuje zdarzenie użycia dla podstawowych kosztów infrastruktury. Jeśli UsageType jest ComputeHR\_SW, zdarzenie użycia reprezentuje opłaty za oprogramowanie premium dla maszyny Wirtualnej.
### <a name="how-do-i-know-if-i-am-charged-for-premium-software"></a>Skąd mam wiedzieć, czy za oprogramowanie premium naliczane są opłaty?
Podczas eksplorowania obrazu maszyny Wirtualnej najlepiej odpowiada Twoim potrzebom, zapoznaj się z [witryną Azure Marketplace.](https://azuremarketplace.microsoft.com/marketplace/apps/category/compute) Obraz ma szybkość planu oprogramowania. Jeśli widzisz "Free" dla stawki, nie ma żadnych dodatkowych kosztów dla oprogramowania. 
### <a name="what-is-the-difference-between-microsoftclassiccompute-and-microsoftcompute-in-the-consumed-service"></a>Jaka jest różnica między programem Microsoft.ClassicCompute a microsoft.compute w usłudze zużytej?
Microsoft.ClassicCompute reprezentuje zasoby klasyczne wdrożone za pośrednictwem usługi Azure Service Manager. Jeśli wdrożysz za pośrednictwem Menedżera zasobów, microsoft.Compute jest wypełniany w zużytej usłudze. Dowiedz się więcej o [modelach wdrażania platformy Azure](https://docs.microsoft.com/azure/azure-resource-manager/resource-manager-deployment-model).
### <a name="why-is-the-instanceid-field-blank-for-my-virtual-machine-usage"></a>Dlaczego pole InstanceID jest puste dla użycia maszyny wirtualnej?
Jeśli wdrożysz za pośrednictwem klasycznego modelu wdrażania, ciąg InstanceID nie jest dostępny.
### <a name="why-are-the-tags-for-my-vms-not-flowing-to-the-usage-details"></a>Dlaczego tagi dla moich maszyn wirtualnych nie są przesyłane do szczegółów użycia?
Tagi tylko przepływ do ciebie użycia CSV dla maszyn wirtualnych Menedżera zasobów. Klasyczne tagi zasobów nie są dostępne w szczegółach użycia.
### <a name="how-can-the-consumed-quantity-be-more-than-24-hours-one-day"></a>W jaki sposób zużyta ilość może być większa niż 24 godziny dziennie?
W modelu klasycznym rozliczenia za zasoby są agregowane na poziomie usługi w chmurze. Jeśli masz więcej niż jedną maszynę wirtualną w usłudze w chmurze, która używa tego samego miernika rozliczeń, użycie jest agregowane razem. Maszyny wirtualne wdrożone za pośrednictwem Menedżera zasobów są rozliczane na poziomie maszyny Wirtualnej, więc ta agregacja nie będzie miała zastosowania.
### <a name="why-is-pricing-not-available-for-dsfsgsls-sizes-on-the-pricing-page"></a>Dlaczego ceny nie są dostępne dla rozmiarów DS/FS/GS/LS na stronie cennika?
Maszyny wirtualne obsługujące magazyn w stanie premium są rozliczane z taką samą stawką jak maszyny wirtualne z magazynu niewłaskowej. Różnią się tylko koszty magazynowania. Odwiedź [stronę z cenami magazynu, aby](https://azure.microsoft.com/pricing/details/storage/unmanaged-disks/) uzyskać więcej informacji.
