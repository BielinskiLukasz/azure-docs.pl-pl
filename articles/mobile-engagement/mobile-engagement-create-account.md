# Azure Mobile Engagement アカウントを作成してアプリの管理を開始する

この記事では、**簡易作成**法を使用して新しい **Azure Mobile Engagement** アカウントを作成する方法について説明します。 また、アプリの監視と管理を開始するために **Mobile Engagement** ポータルに移動する方法についても説明します。

アプリのデータを収集してプッシュ通知を送信できるようにするには、最小限のセットである "基本的な統合" を追加する必要があります。 完全な統合のドキュメントについては記載されて、 [モバイル エンゲージメント統合](mobile-engagement-windows-store-integrate-engagement.md)します。
> [AZURE.IMPORTANT] Azure Mobile Engagement のチュートリアルを完了するには、アクティブな Azure アカウントが必要です。 アカウントがない場合は、無料試用版アカウントを数分で作成することができます。 詳細については、「 <a href="http://azure.microsoft.com/pricing/free-trial/?WT.mc_id=A0E0E5C02&amp;returnurl=http%3A%2F%2Fwww.windowsazure.com%2Fen-us%2Fdevelop%2Fmobile%2Ftutorials%2Fget-started%2F" target="_blank">Azure 無料試用版</a>します。

## Mobile Engagement ポータルに移動します

アプリの監視と管理を開始するには、下部にある **[エンゲージ]** ボタンをクリックして、Mobile Engagement ポータルに移動します。

![](../../includes/media/mobile-engagement-connect-app-with-monitor/engage-button.png)

Analiyze をすることができますが表示されたら、モバイル エンゲージメント ポータルで、作成し、セグメントを管理、ユーザーなどに接続します。。

- [アプリケーションに関するデータをリアルタイムを監視します。](mobile-engagement-user-interface-monitor.md)
- [アプリケーションに関する履歴データを分析します。](mobile-engagement-user-interface-analytics.md)
- [作成し、使用パターンを識別するためにユーザーのセグメントを管理](mobile-engagement-user-interface-segments.md)
- [プッシュ通知を使用してアプリケーションのユーザーにアクセスします。](mobile-engagement-user-interface-reach.md)
