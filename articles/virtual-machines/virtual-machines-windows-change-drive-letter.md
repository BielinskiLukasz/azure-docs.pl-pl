# Windows 仮想マシンのデータ ドライブとしての D ドライブの使用

[AZURE.INCLUDE [learn-about-deployment-models](../../includes/learn-about-deployment-models-classic-include.md)] リソース マネージャーのモデルです。


データの格納に D ドライブを使用する必要がある場合は、一時ディスク用に別のドライブ文字を使用する次の手順に従います。 維持する必要があるデータは、一時ディスクに格納しないでください。

## データ ディスクのアタッチ

最初に、仮想マシンにデータ ディスクを接続する必要があります。 新しいディスクを接続するには、次を参照してください。 [Windows 仮想マシンにデータ ディスクをアタッチする方法 ][attach]します。

既存のデータ ディスクを使用する場合は、ストレージ アカウントに VHD をアップロードしたことを確認します。 手順については、ステップ 3 と 4 を参照してください [の作成とアップロード Azure ][vhd]します。


## Pagefile.sys を一時的に C ドライブに移動します。

1. 仮想マシンへの接続

2. **[スタート]** メニューを右クリックし、**[システム]** を選択します。

3. 左側のメニューで、**[システムの詳細設定]** を選択します。

4. **[パフォーマンス]** セクションで、**[設定]** を選択します。

5. **[詳細]** タブを選択します。

5. **[仮想メモリ]** セクションで、**[変更]** を選択します。

6. **C** ドライブを選択し、**[システム管理サイズ]** をクリックして、**[設定]** をクリックします。

7. 選択、 **D** ドライブをクリックして **ページング ファイルなし** ] をクリックし、 **設定**します。

8. [適用] をクリックします。 変更を有効にするにはコンピューターの再起動が必要であることを示す警告が表示されます。

9. 仮想マシンを再起動します。




## ドライブ文字を変更します。

1. VM が再起動した後、VM に再びログオンします。

2. **[スタート]** メニューをクリックし、「**diskmgmt.msc**」と入力して、Enter キーを押します。 [ディスクの管理] が開始します。

3. 一時ストレージ ドライブの **D** を右クリックし、**[ドライブ文字とパスの変更]** を選択します。

4. ドライブ文字として **G** を選択し、**[OK]** をクリックします。

5. データ ディスクを右クリックして、**[ドライブ文字とパスの変更]** を選択します。

6. ドライブ文字の下にあるドライブを選択して **D** ] をクリックし、 **OK**します。

7. 右クリックして **G**, 、一時ストレージ ドライブ、および選択 **変更ドライブ文字とパスの**します。

8. ドライブ文字の下にあるドライブを選択して **E** ] をクリックし、 **OK**します。

> [AZURE.NOTE] VM に他のディスクまたはドライブがある場合は、同じ方法を使用して他のディスクおよびドライブのドライブ文字を割り当てなおします。 ディスクの構成ができます。  
>-C: OS ディスク  
>-D: データ ディスク  
>-E: 一時ディスク



## pagefile.sys を一時ストレージ ドライブに戻します。

1. 右クリックし、 **開始** メニューをクリック **システム**

2. 左側のメニューで、**[システムの詳細設定]** を選択します。

3. **[パフォーマンス]** セクションで、**[設定]** を選択します。

4. **[詳細]** タブを選択します。

5. **[仮想メモリ]** セクションで、**[変更]** を選択します。

6. OS ドライブを選択して **C** ] をクリック **ページング ファイルなし** ] をクリックし、 **設定**します。

7. 一時ストレージ ドライブを選択して **E** ] をクリックし、 **システム管理サイズ** 順にクリック **設定**します。

8. **[Apply]** をクリックします。 変更を有効にするにはコンピューターの再起動が必要であることを示す警告が表示されます。

9. 仮想マシンを再起動します。
