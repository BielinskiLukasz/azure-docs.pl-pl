# Azure Active Directory の認証プロトコル

Azure Active Directory (Azure AD) では、最も広く使用されている認証および承認のプロトコルを複数サポートしています。 このセクションの各トピックでは、Azure AD でサポートされているプロトコルとその実装について説明します。 トピックでは、サポートされているクレームの種類を使用するフェデレーション メタデータの概要の確認を含めると、OAuth 2.0 を詳しく説明します。 SAML 2.0 プロトコル リファレンス ドキュメント、およびトラブルシューティングのセクション。

## 認証プロトコルの記事とリファレンス

- [重要な情報署名キー ロール オーバーに関する Azure AD で](https://msdn.microsoft.com/library/azure/dn641920.aspx) – Azure AD の署名キー ロール オーバーの周期、キーを自動的に更新することができの変更および最も一般的なアプリケーション シナリオを更新する方法について説明します。


- [サポートされているトークンとクレームの種類](active-directory-token-and-claims.md) -Azure AD が発行するトークン内の要求について説明します。


- [フェデレーション メタデータ](https://msdn.microsoft.com/library/azure/dn195592.aspx) の検索および Azure AD が生成するメタデータ ドキュメントを解釈する方法について説明します。


- [Azure AD での OAuth 2.0](https://msdn.microsoft.com/library/azure/dn645545.aspx) -Azure AD での OAuth 2.0 の実装について説明します。


- [OpenID Connect 1.0](https://msdn.microsoft.com/library/azure/dn645541.aspx) -認証に OAuth 2.0 では、承認のプロトコルを使用する方法について説明します。


- [SAML プロトコル リファレンス](https://msdn.microsoft.com/library/azure/dn195591.aspx) -Azure AD のシングル サインオンおよびシングル サインアウト SAML プロファイルについて説明します。


- [Ws-federation 1.2](https://msdn.microsoft.com/library/azure/dn903702.aspx) -Azure AD の Ws-federation 1.2 について説明します。


- [認証プロトコルのトラブルシューティング](https://msdn.microsoft.com/library/azure/dn195584.aspx) の問題を回避し、解釈し、Azure AD を使用する場合は、エラーを解決する方法について説明します。
