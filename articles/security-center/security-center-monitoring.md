# Azure セキュリティ センターでのセキュリティ ヘルスの監視

このドキュメントは、Azure セキュリティ センターで監視機能を使用してポリシーのコンプライアンスを監視するために役立ちます。
> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。

## Azure セキュリティ センターとは

Azure セキュリティ センターは、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。 これにより、サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

## セキュリティのヘルスの監視とは

私たちは監視について、状況に対処できるように、イベントが発生するのを監視して待機することだと考えがちです。 セキュリティの監視は、組織の標準やベスト プラクティスを満たしていないシステムを識別するために、リソースを監査するプロアクティブな戦略を持つことを意味します。

## セキュリティ ヘルスの監視

有効にした後 [セキュリティ ポリシー](security-center-policies.md) サブスクリプションのリソースの Azure セキュリティ センターは、潜在的な脆弱性を識別するために、リソースのセキュリティを分析します。 ネットワーク設定に関する情報はすぐに利用可能になりますが、セキュリティの更新ステータスや OS の構成などの仮想マシンの構成に関する情報は、利用可能になるまで 1 時間以上かかる場合があります。 [リソースのセキュリティ ヘルス] ブレードで、リソースのセキュリティの状態と、問題がある場合はその問題を確認することができます。 これらの問題の一覧は、[推奨事項] ブレードでも確認できます。

推奨設定を適用する方法の詳細については、「 [Azure セキュリティ センターのセキュリティに関する推奨事項を実装する](security-center-recommendations.md)します。

**[リソース ヘルス]** タイルで、リソースのセキュリティの状態を監視することができます。 次に示すように、注意を必要とする高レベルと中レベルの重大度を持つ問題の数が表示されます。

![リソース ヘルス](./media/security-center-monitoring/security-center-monitoring-fig1.png)

有効になっているセキュリティ ポリシーが、ここで監視されているコントロールのタイプに影響します。 Azure セキュリティ センターなどのセキュリティ更新プログラムまたはなしのサブネットが不足している VM への対処する必要がある脆弱性を識別する場合、 [ネットワーク セキュリティ グループ](virtual-networks-nsg.md), 、ここで識別されます。

### 仮想マシン

**[リソース ヘルス]** タイルで **[Virtual Machines]** をクリックすると、**[Virtual Machines]** ブレードが開き、次のようにオンボードおよび防止の手順に関する詳細と、Azure セキュリティ センターで監視されるすべての VM の一覧が表示されます。

![VM ごとの欠落しているシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig2.png)

このブレードが開くと、3 つのセクションが表示されます。 各セクションでは、その問題に対処するために推奨される手順に関する詳細を表示する個別のオプションを選択できます。

たとえば、**[防止手順]** の下で **[不足しているシステムの更新]** をクリックすると、**[不足しているシステムの更新]** ブレードが開き、次のようにパッチが欠落している VM とそれぞれの重要度の一覧が表示されます。

![システムの更新の欠落防止](./media/security-center-monitoring/security-center-monitoring-fig3.png)

推奨される更新プログラムの詳細を表示するには、VM の名前をクリックします。 その VM の新しいブレードが開き、不足している更新プログラムの一覧が表示されます。

![VM ごとの推奨されるシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig4.png)
> [AZURE.NOTE] ここのセキュリティの推奨事項は、[推奨事項] ブレードに表示されているものと同じです。 記事を参照して [Azure セキュリティ センターのセキュリティに関する推奨事項を実装する](security-center-recommendations.md) の推奨事項の解決の詳細についてです。 これは VM だけでなく、[リソース ヘルス] タイルで利用できるすべてのリソースに適用できます。

### ネットワーク

ネットワークの防止ステータスは、Azure セキュリティ センターによって監視されている仮想ネットワークを示します。 [リソース ヘルス] タイルで **[ネットワーク]** をクリックすると、**[ネットワーク]** ブレードが開き、次のような詳細が表示されます。

![VM ごとの推奨されるシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig5.png)

仮想マシン リソースのヘルス情報と同様に、このブレードでは上部に問題をまとめた一覧と下部に監視対象のネットワークの一覧が表示されます。

ネットワークの状態の内訳のセクションで防止のように手順 [エンドポイントの Acl](virtual-machines-set-up-endpoints.md) が有効でない [ネットワーク セキュリティ グループ](virtual-networks-nsg.md) 無効になっており、正常なサブネットが一覧表示します。 このいずれかをクリックして詳細を表示し、問題を解決するためのさらなるアクションを実行できます。

この一例が、**サブネット上の NSG が有効になっていない**警告になります。 この警告をクリックすると、**[サブネットに欠落しているネットワーク セキュリティ グループを構成]** ブレードが開きます。 ここに、有効になっているネットワーク セキュリティ グループがないサブネットの説明と、このサブネットに属するネットワーク インターフェイス カードの一覧が表示されます。 セキュリティ グループを適用するサブネットを選択し、**[NSG の構成]** オプションをクリックします。 次のように **[ネットワーク セキュリティ グループの選択]** ブレードが開きます。

![VM ごとの推奨されるシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig6.png)

### SQL

**[リソース ヘルス]** タイル内の **[SQL]** をクリックすると、**[SQL]** ブレードが開き、監査が有効になっていない、透過的なデータ暗号化が有効になっていない、データベースの全般のヘルス状態などの問題に関する推奨事項が示されます。 ブレードの下部には、次のように、SQL Server の内訳と各サーバーに属しているデータベースが示されます。

![VM ごとの推奨されるシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig7.png)

このいずれかの推奨事項をクリックして詳細を表示し、問題を解決するためのさらなるアクションを実行できます。 次に示す例は、SQL server で監査を有効にする拡張を示しています。

![VM ごとの推奨されるシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig8.png)

### アプリケーション

Azure ワークロードに内に置かれたアプリケーション [リソース マネージャーの Vm](resource-manager-deployment-model.md) 公開された web ポート (TCP ポート 80 および 443) を Azure セキュリティ センターを監視できます、潜在的なセキュリティ問題を特定し、修復手順をお勧めします。 **[アプリケーション]** タイル内をクリックすると、**[アプリケーション]** ブレードが開き、防止手順のセクションに一連の推奨事項が示され、さらに次のように、ホスト/仮想 IP ごとのアプリケーションの内訳も示されます。

![VM ごとの推奨されるシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig9.png)

前のリソースの他の推奨事項と同様に、推奨事項をクリックすると、問題と修復方法の詳細を確認できます。 次の図に示す例は、**セキュリティ保護されていない Web アプリケーション**として識別されたアプリケーションです。 安全でないと見なされたアプリケーションを選択すると、使用可能なオプション (この場合は、Web アプリケーション ファイアウォールを有効化するためのオプション) が示された別のブレードが開きます。

![VM ごとの推奨されるシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig10.png)
