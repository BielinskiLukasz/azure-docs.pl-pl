# <a name="azure-synapse-analytics-release-notes"></a>Informacje o wersji usługi Azure Synapse Analytics

W tym artykule podsumowano nowe funkcje i ulepszenia w ostatnich wersjach [synapse SQL](sql-data-warehouse-overview-what-is.md) w usłudze Azure Synapse. W artykule wymieniono również znaczące aktualizacje zawartości, które nie są bezpośrednio związane z wydaniem, ale opublikowane w tym samym przedziale czasowym. Aby uzyskać ulepszenia innych usług platformy Azure, zobacz [aktualizacje usługi](https://azure.microsoft.com/updates).

## <a name="check-your-azure-synapse-version"></a>Sprawdzanie wersji usługi Azure Synapse

Po wdrożeniu nowych funkcji we wszystkich regionach sprawdź wersję wdrożoną w wystąpieniu i najnowsze informacje o wersji, aby uzyskać dostępność funkcji. Aby sprawdzić wersję, połącz się z pulą SQL za pośrednictwem `SELECT @@VERSION;` programu SQL Server Management Studio (SSMS) i uruchom, aby zwrócić bieżącą wersję.

Użyj zidentyfikowanej wersji, aby potwierdzić, która wersja została zastosowana do puli SQL. Data w danych wyjściowych określa miesiąc dla wydania zastosowanego do puli SQL.

> [!NOTE]
> Nazwa produktu zwrócona@VERSION przez select @ zmieni się z Usługi Microsoft Azure SQL Data Warehouse na Azure Synapse Analytics. Wyślemy powiadomienie z wyprzedzeniem przed dokonaniem zmiany. Ta zmiana jest istotna dla klientów, którzy analizują@VERSION nazwę produktu z wyniku SELECT @ w kodzie aplikacji. Aby uniknąć zmian kodu aplikacji z powodu rebrandingu produktu, użyj tych poleceń, aby zbadać serverproperty dla nazwy produktu bazy danych i wersji: Aby zwrócić numer wersji XX. X.XXXXX. X (bez nazwy produktu) użyj tego polecenia: 
> ```
> SELECT SERVERPROPERTY('ProductVersion')
> ```
> Aby przywrócić wersję aparatu, użyj tego polecenia, które zwraca 6 dla usługi Azure Synapse Analytics (dawniej SQL Data Warehouse): 
> ```
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Dzięki temu kolejne wykonania kwerendy, aby uzyskać wyniki bezpośrednio z utrwalonych pamięci podręcznej, więc ponowneuwkomczenie nie jest potrzebne. Buforowanie zestawu wyników zwiększa wydajność zapytań i zmniejsza użycie zasobów obliczeniowych. Ponadto kwerendy przy użyciu zestawu wyników buforowanych nie używają żadnych gniazd współbieżności i w związku z tym nie są wliczane do istniejących limitów współbieżności. Dla zabezpieczeń użytkownicy mogą uzyskiwać dostęp do wyników w pamięci podręcznej tylko wtedy, gdy mają takie same uprawnienia dostępu do danych, jak użytkownicy tworzący wyniki w pamięci podręcznej. Aby uzyskać więcej informacji, zobacz [Dostrajanie wydajności z buforowaniem zestawu wyników](/azure/sql-data-warehouse/performance-tuning-result-set-caching). Dotyczy wersji 10.0.10783.0 lub nowszej.|

Tempdb — zrozumieć, kiedy skalować i skonfigurować klasy zasobów w celu zmniejszenia rywalizacji tempdb.<br/><br/>Istnieje głębsza integracja metryk magazynu danych z [usługą Azure Monitor,](https://azure.microsoft.com/blog/enhanced-capabilities-to-monitor-manage-and-integrate-sql-data-warehouse-in-the-azure-portal/) w tym ulepszony, dostosowywany wykres monitorowania dla metryk w czasie zbliżonym do rzeczywistego w bloku przeglądu. Nie musisz już opuszczać bloku przeglądu magazynu danych, aby uzyskać dostęp do metryk usługi Azure Monitor podczas monitorowania użycia lub sprawdzania poprawności i stosowania zaleceń dotyczących magazynu danych. Ponadto dostępne są nowe metryki, takie jak tempdb i adaptacyjne wykorzystanie pamięci podręcznej w celu uzupełnienia zaleceń dotyczących wydajności.|

Za pomocą jednego kliknięcia przycisku można teraz skonfigurować dzienniki diagnostyczne pod kątem historycznych możliwości rozwiązywania problemów z wydajnością zapytań przy użyciu [usługi Log Analytics.](https://docs.microsoft.com/azure/log-analytics/log-analytics-queries) Dzienniki diagnostyczne usługi Azure Monitor obsługują dostosowywane okresy przechowywania, zapisując dzienniki na koncie magazynu do celów inspekcji, możliwość przesyłania strumieniowego dzienników do centrów zdarzeń w pobliżu szczegółowych informacji w czasie rzeczywistym oraz możliwość analizowania dzienników przy użyciu usługi Log Analytics za pomocą zapytań dziennika. Dzienniki diagnostyczne składają się z widoków telemetrycznych magazynu danych równoważnych najczęściej używanym rejestratorom dmv rozwiązywania problemów z wydajnością.
