# <a name="what-tools-are-included-on-the-azure-data-science-virtual-machine"></a>Jakie narzędzia znajdują się na maszynie wirtualnej usługi Azure Data Science?

Maszyna wirtualna do nauki o danych to najprostszy sposób eksplorowania danych i uczenia maszynowego w chmurze. Maszyny wirtualne do nauki o danych są wstępnie skonfigurowane z pełnym systemem operacyjnym, poprawkami zabezpieczeń, sterownikami oraz popularnym oprogramowaniem do nauki o danych i tworzenia danych. Można wybrać środowisko sprzętowe, od tańszych maszyn zorientowanych na procesor CPU do bardzo wydajnych maszyn z wieloma procesorami graficznymi, pamięcią masową NVMe i dużą ilością pamięci. W przypadku komputerów z procesorami GPU są zainstalowane wszystkie sterowniki, wszystkie struktury uczenia maszynowego są dopasowane do wersji dla zgodności z procesorem GPU, a akceleracja jest włączona we wszystkich aplikacjach obsługujących procesory GPU.

Maszyna wirtualna do nauki o danych jest wyposażona w najbardziej przydatne narzędzia do nauki o danych, które są fabrycznie zainstalowane:

## <a name="how-to-choosecondaenvironment"></a>Jak wybrać środowisko conda  

> [!NOTE]
> Istnieją przełomowe zmiany podczas przełączania do nowej wersji maszyny wirtualnej nauki o danych ze starej wersji. 

**Przełączanie z systemu Windows Server 2016 do systemu Windows Server 2019** </br> 

* Środowisko conda jest ustawione na python37_default. Nie obsługujemy python2. Jeśli używasz tensorflow lub pytorch w środowisku python conda użyj odpowiednio py37_tensorflow lub py37_pytorch. Jeśli używasz tylko usługi AzureML używać azureml_py36_automl </br>

* Jeśli używasz tensorflow lub pytorch w środowisku azureml conda użyj azureml_py36_tensorflow lub azureml_py36_pytorch odpowiednio. </br>

**Zmiana z Ubuntu 16.04 na Ubuntu 18.04** </br>

* Jeśli używasz tensorflow lub pytorch w azureml_py36 środowiska conda używać azureml_py36_tensorflow lub azureml_py36_pytorch odpowiednio. </br>

* Jeśli używasz tensorflow lub pytorch w środowisku conda py37 używać odpowiednio py37_tensorflow lub py37_pytorch. </br>

* W przeciwnym razie użyj python37_default lub jeśli używasz tylko azureml_py36 używać azureml_py36_automl </br>
