# Windows 仮想マシンのデータ ドライブとしての D ドライブの使用 

[AZURE.INCLUDE [learn-about-deployment-models](../../includes/learn-about-deployment-models-classic-include.md)] リソース マネージャーのモデルです。


データの格納に D ドライブを使用する必要がある場合は、一時ディスク用に別のドライブ文字を使用する次の手順に従います。 維持する必要があるデータは、一時ディスクに格納しないでください。

## データ ディスクのアタッチ

最初に、仮想マシンにデータ ディスクを接続する必要があります。 新しいディスクをアタッチするには、[Windows の仮想マシンにデータ ディスクを接続する方法] を参照してください。 [アタッチ] です。 

既存のデータ ディスクを使用する場合は、ストレージ アカウントに VHD をアップロードしたことを確認します。 手順については、ステップ 3 と 4 を参照してください [の作成とアップロード Azure への Windows Server VHD][VHD]します。 


## Pagefile.sys を一時的に C ドライブに移動します。

1. 仮想マシンへの接続 

2. 右クリックし、 **開始** メニューをクリック **システム**します。

3. 左側のメニューでは、次のように選択します。 **システムの詳細設定**します。

4.  **パフォーマンス** ] セクションの [ **設定**します。

5. 選択、 **詳細** ] タブをクリックします。

5.  **仮想メモリ** ] セクションの [ **変更**します。

6. 選択、 **C** ドライブをクリックして **システム管理サイズ** ] をクリックし、 **設定**します。

7. 選択、 **D** ドライブをクリックして **ページング ファイルなし** ] をクリックし、 **設定**します。

8. [適用] をクリックします。 変更を有効にするにはコンピューターの再起動が必要であることを示す警告が表示されます。

9. 仮想マシンを再起動します。




## ドライブ文字を変更します。 

1. VM が再起動した後、VM に再びログオンします。

2. をクリックして、 **開始** メニューと種類 **diskmgmt.msc** し、enter キーを押します。 [ディスクの管理] が開始します。

3. 右クリックして **D**, 、一時ストレージ ドライブ、および選択 **変更ドライブ文字とパスの**です。

4. ドライブ文字の下にあるドライブを選択して **G** ] をクリックし、 **OK**します。 

5. データ ディスクを右クリックして **変更ドライブ文字とパス**します。

6. ドライブ文字の下にあるドライブを選択して **D** ] をクリックし、 **OK**します。 

7. 右クリックして **G**, 、一時ストレージ ドライブ、および選択 **変更ドライブ文字とパスの**です。

8. ドライブ文字の下にあるドライブを選択して **E** ] をクリックし、 **OK**します。 

> [AZURE.NOTE] VM に他のディスクまたはドライブがある場合は、他のディスクおよびドライブのドライブ文字を再割り当てする同じメソッドを使用します。 ディスクの構成ができます。  
>- C: OS ディスク  
>- D: データ ディスク  
>- E: 一時ディスク



## pagefile.sys を一時ストレージ ドライブに戻します。 

1. 右クリックし、 **開始** メニューをクリック **システム**

2. 左側のメニューでは、次のように選択します。 **システムの詳細設定**します。

3.  **パフォーマンス** ] セクションの [ **設定**します。

4. 選択、 **詳細** ] タブをクリックします。

5.  **仮想メモリ** ] セクションの [ **変更**します。

6. OS ドライブを選択して **C** ] をクリック **ページング ファイルなし** ] をクリックし、 **設定**します。

7. 一時ストレージ ドライブを選択して **E** ] をクリックし、 **システム管理サイズ** 順にクリック **設定**します。

8. クリックして **適用**します。 変更を有効にするにはコンピューターの再起動が必要であることを示す警告が表示されます。

9. 仮想マシンを再起動します。
