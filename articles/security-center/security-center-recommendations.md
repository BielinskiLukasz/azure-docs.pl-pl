# Azure セキュリティ センターでのセキュリティに関する推奨事項の管理

このドキュメントでは、Azure セキュリティ センターでの推奨事項に従ってご使用の Azure のリソースを保護する方法について説明します。
> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。 これは、サンプルのデプロイメントを使用してサービスを紹介するもので、 ステップ バイ ステップ ガイドではありません。

## Azure セキュリティ センターとは

Azure セキュリティ センターは、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。 これにより、サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

## セキュリティに関する推奨事項とは

Azure セキュリティ センターは、Azure リソースのセキュリティの状態を定期的に分析します。 潜在的なセキュリティの脆弱性が識別されると、推奨事項が作成されます。 推奨事項では、必要なコントロールを構成する手順を説明します。

## セキュリティに関する推奨事項の実装

### 推奨事項の設定

[Azure セキュリティ センターのセキュリティ ポリシーの設定](security-center-policies.md) する方法について説明します。

- セキュリティ ポリシーの構成
- データ取集の有効化
- セキュリティ ポリシーの一環として見なす推奨事項の選択

システムの更新プログラム、基準の規則、マルウェア対策を現在のポリシーの推奨事項の中心 [エンドポイントの Acl](virtual-machines-set-up-endpoints.md), 、[ネットワーク セキュリティ グループ](virtual-networks-nsg.md) サブネットとネットワーク インターフェイス、SQL Database の監査、SQL データベースの透過的なデータ暗号化、および web アプリケーション ファイアウォール上です。 [セキュリティ ポリシーの設定](security-center-policies.md) 推奨設定の各オプションの説明を提供します。

### 監視に関する推奨事項

セキュリティ ポリシーを設定すると、潜在的な脆弱性を識別するために、Azure セキュリティ センターがリソースのセキュリティの状態を分析します。 **[セキュリティ センター]** ブレードの **[推奨事項]** タイルでは、Azure セキュリティ センターで識別された推奨事項の総数を把握することができます。

![][2]

各推奨事項の詳細を表示するには、以下の手順に従います。

1. **[セキュリティ センター]** ブレードの **[推奨事項]** タイルをクリックします。 **[推奨事項]** ブレードが開きます。
2. 表示された推奨事項を、状態と重要度によってフィルター処理することができます。 **[推奨事項]** ブレードで **[フィルター]** をクリックします。 [フィルター] ブレードが開きを確認したい重要度と状態の値を選択します。
![][3]

3. 推奨事項が適用できないと判断した場合、その推奨事項を無視し、ビューから除外することができます。 推奨事項を無視するには 2 つの方法があります。 項目を右クリックして **[無視]** を選択するか、項目の上にマウスを合わせ、右側に表示されている 3 つの点をクリックして、**[無視]** を選択します。 完了済みの推奨事項を表示] をクリックして **フィルター** を選択すると、 **Dismissed**します。
![][4]

推奨事項は表形式で表示されます。表の行はそれぞれ特定の推奨事項を表します。 この表には次の列があります。

- **説明**: 推奨事項およびそれに対応するために必要な処理の説明
- **リソース**: この推奨事項が適用されるリソース
- **状態**: 推奨事の現在の状態
  - **オープン**: 未対応
  - **処理中**: 推奨事項は現在、これらのリソースに適用されており、ユーザーのアクションは不要
  - **解決済み**: 推奨事項が既に完了済み (状態が解決済みになると、行の色が灰色表示になる)
- **重要度**: 特定の推奨事項の重要度の説明
  - **高**: 重要なリソース (アプリケーション、VM、ネットワーク セキュリティ グループ) に脆弱性が存在しており、注意が必要
  - **中**: プロセスを完了または脆弱性を排除するために重大ではないまたは追加の手順が必要
  - **低**: 対処すべきだが、直ちにアクションが必要ではない脆弱性 (既定では、低い推奨事項が用意されていませんがそれらを表示する場合、低い推奨事項にフィルターできます)。

次の表を参照として使用し、使用可能な推奨事項と、適用した場合のそれぞれの結果について理解します。

| 推奨| 説明|
|----- |-----|
| サブスクリプションまたは仮想マシンのデータ収集の有効化| 選択した VM に対し、セキュリティ ポリシーでデータ収集を有効にすることをお勧めします。|
| 基準ルールの不一致の解決| たとえばパスワードの保存を許可しないなど、OS の構成を推奨される基準と一致させることをお勧めします。|
| システムの更新の適用| 不足しているシステムの更新を VM (Windows VM のみ) にデプロイすることをお勧めします。|
| エンドポイントへの ACL の構成| VM (クラシック VM のみ) への着信アクセスを制限するアクセス制御リストを構成することをお勧めします。|
| Web アプリケーション ファイアウォールの追加| Web エンドポイントに Web アプリケーション ファイアウォール (WAF) をデプロイすることをお勧めします (リソース マネージャー VM のみ)。|
| Web アプリケーション ファイアウォールのセットアップの完了| WAF の構成を完了するには、WAF アプライアンスにトラフィックを再ルーティングする必要があります。この推奨事項を適用すると、必要なセットアップの変更が完了します。|
| マルウェア対策の有効化| マルウェア対策を VM (Windows Vm のみ) にプロビジョニングすることをお勧めします。|
| サブネットとネットワーク インターフェイス上でネットワーク セキュリティ グループを有効化| サブネットとネットワーク インターフェイス上でネットワーク セキュリティ グループ (NSG) を有効にすることをお勧めします (リソース マネージャー VM のみ)。|
| パブリックの外部エンドポイントを通じたアクセスを制限| NSG に着信トラフィックのルールを構成することをお勧めします。|
| サーバーの SQL 監査の有効化| Azure SQL サーバー (Azure SQL サービスのみ、仮想マシンで実行されている SQL を除く) に対して監査を有効にすることをお勧めします。|
| データベースの SQL 監査の有効化| Azure SQL データベース (Azure SQL サービスのみ、仮想マシンで実行されている SQL を除く) に対して監査を有効にすることをお勧めします。|
| SQL データベースでの透過的なデータ暗号化の有効化| SQL データベース (Azure SQL のサービスのみ) に対して暗号化を有効にすることをお勧めします。|
| VM エージェントのデプロイ| VM エージェントを必要とする VM を確認できます。パッチのスキャン、基準のスキャン、およびマルウェア対策をプロビジョニングするには、VM 上に VM エージェントをインストールする必要があります。既定では、Azure Marketplace からデプロイされた VM には VM エージェントがインストールされます。記事 [VM エージェントと拡張機能 – パート 2](http://azure.microsoft.com/blog/2014/04/15/vm-agent-and-extensions-part-2/) VM エージェントをインストールする方法について説明します。|

### 推奨事項の適用

すべての推奨事項を確認したら、最初に適用する推奨事項を決めることができます。 重要度の評価を主なパラメーターとして使用して、最初に適用すべき推奨事項を評価することをお勧めします。
マルウェア対策の推奨事項を使用して、推奨事項の適用方法の例を紹介しましょう。

1. **の推奨事項** ブレードで、 **マルウェア対策を有効にする**します。
![][5]

2. **[マルウェア対策のインストール]** ブレードでマルウェア対策が有効になっていない仮想マシンの一覧から選択し、**[マルウェア対策のインストール]** をクリックします。
3. **[新しいリソース]** ブレードが開き、使用するマルウェア対策ソリューションを選択できます。 **[Microsoft マルウェア対策]** を選択します。
4. マルウェア対策ソリューションに関する追加情報が表示されます。 **[作成]** を選択します。
5. 必要な構成設定を入力して、 **拡張機能の追加** ブレードで、および選択 **[ok]**します。
![][6]

[Microsoft マルウェア対策](azure-security-antimalware.md) は選択したバーチャル マシンでアクティブになります。

### 推奨されるパートナー ソリューションのデプロイ

推奨事項は、Microsoft パートナーからの統合セキュリティ ソリューションをデプロイすることです。 このやり方の例を紹介しましょう。

1. **[推奨事項]** ブレードに戻ります。
2.  推奨事項 **[Web アプリケーション ファイアウォールを使用した安全な Web アプリケーション]** を選択します。 開き、 **Web アプリケーションを保護されていない** ブレードです。
![][7]
3. Web アプリケーションを選択すると、**[Web アプリケーション ファイアウォールの追加]** ブレードが開きます。
4. **[Barracuda Web アプリケーション ファイアウォール]** を選択します。 **Barracuda Web アプリケーション ファイアウォール**に関する情報を提供するブレードが開きます。
5. [情報] ブレードで **[作成]** クリックします。 **[新しい Web アプリケーション ファイアウォール]** ブレードが開きます。ここで **VM 構成**手順を実行して、**WAF 情報**を提供できます。
6. **[VM 構成]** を選択します。  **VM 構成** ブレード WAF を実行する仮想マシンの起動に必要な情報を入力します。
![][8]
7. **[新しい Web アプリケーション ファイアウォール]** ブレードに戻り、**[WAF 情報]** を選択します。 **[WAF 情報]** ブレードで WAF そのものを構成します。 手順 6 では、WAF が実行される仮想マシンを構成できます。手順 7 では、WAF そのものをプロビジョニングできます。

8. **[推奨事項]** ブレードに戻ります。 WAF の作成後に、新しいエントリ **[Web アプリケーション ファイアウォールのセットアップの完了]** が生成されました。 これにより、実際には、Azure の仮想ネットワーク内の WAF を配線いるため、アプリケーションを保護できますプロセスを完了する必要があるかがわかります。
![][9]

9. **[Web アプリケーション ファイアウォールのセットアップの完了]** を選択すると、新しいブレードが開きます。 トラフィックを再ルーティングする必要がある Web アプリケーションがあることが確認できます。
10. Web アプリケーションを選択すると、Web アプリケーション ファイアウォールのセットアップを完了するための手順が表示されたブレードが開きます。 手順の完了] をクリックして **トラフィックを制限する**, 、Azure セキュリティ センターは配線アップするとします。
![][10]

その WAF からのログは完全に統合されます。 Azure セキュリティ センターは、重要なセキュリティの警告を表示するため、自動的にログの収集と分析を開始することができます。
