# Android SDK for Azure Mobile Engagement

Azure Mobile Engagement を Android アプリに統合する方法の詳細について紹介します。 それをまず試して場合を作成するには、 [15 分間チュートリアル](mobile-engagement-android-get-started.md)します。

クリックすると表示、 [SDK コンテンツ](mobile-engagement-android-sdk-content.md)します。

## 統合手順

1. ここから開始: [Android アプリでモバイル エンゲージメントを統合する方法](mobile-engagement-android-integrate-engagement.md)

2. 通知: [Android アプリでリーチ (通知) を統合する方法](mobile-engagement-android-integrate-engagement-reach.md)
    1. Google の Cloud は Messaging (GCM): [GCM を Mobile Engagement に統合する方法](mobile-engagement-android-gcm-integrate.md)
    2. Amazon デバイス メッセージング] (ADM): [ADM を Mobile Engagement に統合する方法](mobile-engagement-android-adm-integrate.md)

3. タグ付けプランの実装: [タグ付け API を Android アプリで高度なモバイル エンゲージメントを使用する方法](mobile-engagement-android-use-engagement-api.md)


## リリース ノート

## 4.1.3 (12/9/2015)

- 安定性が向上しました。

## 4.1.0 (2015 年 8 月 25 日)

- Android M の新しいアクセス許可モデルを処理します。
- 使用せずに実行時に場所の機能を構成できるよう  `AndroidManifest.xml`します。
- アクセス許可のバグの修正: を使用する場合 `ACCESS_FINE_LOCATION`, 、し `ACCESS_COARSE_LOCATION` は必要ありません。
- 安定性が向上しました。

以前のバージョンを参照してください、 [リリース ノートを完了](mobile-engagement-android-release-notes.md)します。

## アップグレードの手順

既にある古いバージョンの SDK を統合、アプリケーションを参照してください [アップグレード手順](mobile-engagement-android-upgrade-procedure.md)します。
