# Azure Privileged Identity Management: セキュリティ通知の構成方法
## セキュリティ通知の概要
Azure Privileged Identity Management では以下の通知が提供されており、これらは構成できます。 セキュリティ通知は、PIM ダッシュボードの [アラート] セクションで見ることができます。

| アラート: | トリガー |
| ------------- | ------------- |
| **永続的アクティブ化攻撃の可能性があります** | 管理者が、PIM の外部で一時的なロールをアクティブ化しました。 |
| **特権付きロールのアクティブ化の疑わしい更新です** | 設定で許可された時間に行われた同じロールの再アクティブ化が多すぎます。 |
| **ハニー トークン グローバル管理者ユーザーの疑わしい使用です** | "ハニー ポット" ユーザーの使用が検出されました。|
| **ロールのアクティブ化に弱い認証が構成されています** | MFA が有効ではないロールが設定に存在します。 |
| **冗長な管理者により攻撃対象領域が増えています** | 設定されている日数内にロールをアクティブ化していない一時管理者がいます。 |
| **多すぎるグローバル管理者により攻撃対象領域が増えています** | 設定で許可されている数より多くのグローバル管理者がいます。 |

## セキュリティ通知の構成

### "特権付きロールのアクティブ化の疑わしい更新です" 通知を構成する
1.  **アクティビティ** [ダッシュ ボードのセクション **セキュリティの警告**します。  **のアクティブなセキュリティの警告** ブレードが表示されます。
2. クリックして **設定**します。
3. 設定、 **ライセンス認証の更新期間** スライダーを調整するか、テキスト フィールドに分単位の数を入力しています。 指定できる最大の値は 100 です。
4. 設定、 **アクティブ化の更新の数** でスライダーを調整するか、テキスト フィールドに更新の数を入力してライセンス認証更新時間帯。  更新の最大回数は 100 です。
5. クリックして **保存**します。

### "冗長な管理者により攻撃対象領域が増えています" 通知を構成する
1.  **アクティビティ** [ダッシュ ボードのセクション **セキュリティの警告**します。   **のアクティブなセキュリティの警告** ブレードが表示されます。
2. クリックして **設定**します。
3. スライダーを調整するか、テキスト フィールドに日数を入力して、ロールをアクティブ化しなくても許される日数を選択します。
4. クリックして **保存**

### "多すぎるグローバル管理者により攻撃対象領域が増えています" 通知を構成する

この通知には、通知をトリガーする可能性がある設定が 2 つあります。  グローバル管理者の最小数は、許可される数より多くの管理者がいる場合に通知をトリガーします。  すべての種類の管理者の総数に対するグローバル管理者の割合が設定されている値より大きい場合、通知がトリガーされます。

1.  **アクティビティ** [ダッシュ ボードのセクション **セキュリティの警告**します。   **のアクティブなセキュリティの警告** ブレードが表示されます。
2. クリックして **設定**します。
3. 設定、 **グローバル管理者の最小数** スライダーを調整またはテキスト フィールドに番号を入力します。
4. 設定、 **割合場合は、グローバル管理者** スライダーを調整することで、テキスト フィールドに番号を入力します。
