# Azure Privileged Identity Management セキュリティ ウィザード

Azure Privileged Identity Management を初めて実行すると、ウィザードが表示されます。 このウィザードでは、特権 ID のセキュリティ リスクと、Privileged Identity Management を使用してリスクを軽減する方法がわかります。

確認するには 3 つのセクションがある **、ADMINS 可能性がありますを配置する危険、管理、管理者の攻撃対象領域**, 、および **一時的管理者設定の定義**します。 各セクションでは、概念の概要と、実行する操作の説明が示されています。

初期状態では、グローバル管理者はすべて永続的です。 クリックすると **、ADMINS 可能性がありますを配置する危険**, 、グローバル管理者ロールの数を使用して現在の一覧が表示されます。

クリックすると **管理の管理者の攻撃対象領域**, は一時管理者を変更するには、その永続的なままにして営業案件が表示またはロールからまとめて削除します。

**一時的な管理者の設定を定義する** 一時的などのくらいの時間を決定することができます特権を持っている、通知を有効にして多要素認証を要求する管理者です。

## グローバル管理者ロールを一時的または永続的に変更する

グローバル管理者の期間の変更に関しては 3 つのオプションがあります。

1.  クリックして、 **一時的なものすべて** ] ボタンをクリックするとすべてのグローバル管理者が一時的なものです。

2.  クリックして、 **すべて永続的** 、すべてのグローバル管理者を固定化] ボタンをクリックします。

3.  選択 **保持 Perm**, 、**、Temp**, 、または **ロールから削除** 管理者ごとにグローバルです。

4.  Click **OK**.

5.  クリックして **送信**します。

## グローバル管理者ロールのアクティブ化期間を変更します。

1.  移動、 **のライセンス認証期間** スライダーを左または右のライセンス認証期間を増減します。 アクティブ化期間は最長 72 時間まで指定できます。

2.  時間数を入力して、 **時間** つまみの右側にフィールドです。

## 通知を有効にする

管理者は、ロールがアクティブに加えられたときにメールを受信できるをクリックして通知を有効にする、 **を有効にする** ] ボタンをクリックします。 この機能を無効にすることもできます。

## 多要素認証を要求する

MFA を使用して、自分のアカウントにログインし、自分のロールの延長をリクエストすることを管理者にする場合をクリックして MFA を有効にする、 **を有効にする** ] ボタンをクリックします。 この機能を無効にすることもできます。
