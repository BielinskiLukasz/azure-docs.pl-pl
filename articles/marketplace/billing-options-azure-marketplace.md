# <a name="billing-options"></a>Opcje rozliczeń  

## <a name="commercial-considerations-on-the-marketplace"></a>Zagadnienia komercyjnych w witrynie marketplace  
Portal marketplace nie udostępnia przychód w przypadku następujących typów listy.  
*   List  
*   Wersja próbna  
*   Przeprowadzaj transakcje za pomocą modelu rozliczeniowego Bring Your Own License (BYOL)

Nie są rozliczane dodatkowe opłaty za udział w sklepów w witrynie marketplace.  
*   Aby uzyskać więcej informacji, odwiedź witrynę Azure Marketplace — zasady udziału strony znajduje się w [azure.microsoft.com/support/legal/marketplace/participation-policies](https://azure.microsoft.com/support/legal/marketplace/participation-policies).  

## <a name="pay-as-you-go-and-bring-your-own-license-billing-options"></a>Płatność za rzeczywiste użycie i przenieść własną licencję opcje rozliczeń  
Gdy używasz zgodnie z rzeczywistym użyciem model rozliczeń jako opcja publikowania jest przychód licencjonowania oprogramowania na podstawie użycia udostępnione 80/20, między użytkownikiem a firmą Microsoft. Jednej oferty mogą są wyceniane za pomocą zarówno w płatność za rzeczywiste użycie, jak i w ramach opcji BYOL modelami rozliczeń. Dwa modele rozliczeniowe współistnieć na tym poziomie ofertę jako osobne jednostki SKU. Należy skonfigurować modelami rozliczeń w Twoją ofertę w portalu Cloud Partner. 

Należy wziąć pod uwagę następujące przykłady.  
*   Jeśli włączona jest opcja płatności, masz następujące wyniki.  
    | Koszt licencji | $1.00 za godzinę |  
    |:--- |:--- |  
    | Koszt użycia platformy Azure (D1/1-rdzeniowe) | 0,14 USD za godzinę |  
    | *Rozliczenie przez firmę Microsoft* | *$1.14 za godzinę* |  

    W tym scenariuszu firma Microsoft rozlicza 1.14 $ na godzinę w przypadku użycia opublikowanego obrazu maszyny Wirtualnej.  
    | W ramach firmy Microsoft | $1.14 za godzinę |  
    |:--- |:--- |  
    | Microsoft płaci 80% kosztów licencji | $0.80 za godzinę |  
    | Firma Microsoft zachowuje 20% przychodów kosztami licencji | 0,20 USD za godzinę |  
    | Firma Microsoft zachowuje koszty użycia platformy Azure | 0,14 USD za godzinę |  

*   Po włączeniu opcji BYOL, masz następujące wyniki.  
    | Koszt licencji | Opłata za licencję negocjowane i rozliczane przez użytkownika |  
    |:--- |:--- |  
    | Koszt użycia platformy Azure (D1/1-rdzeniowe) | 0,14 USD za godzinę |  
    | *Rozliczenie przez firmę Microsoft* | *0,14 USD za godzinę* |  

    W tym scenariuszu firma Microsoft rozlicza 0,14 USD za godzinę do użycia opublikowanego obrazu maszyny Wirtualnej.  
    | W ramach firmy Microsoft | 0,14 USD za godzinę |  
    |:--- |:--- |  
    | Firma Microsoft zachowuje koszty użycia platformy Azure | 0,14 USD za godzinę |  
    | Firma Microsoft zachowuje 0% kosztów licencji | 0,00 USD za godzinę |  

## <a name="single-billing-and-payment-methods"></a>Jednym rozliczeń oraz metod płatności  
Ważną zaletą używania Transact listę opcji jest możliwość swoje koszty licencjonowania i użycie platformy Azure rozliczana pojedynczego bezpośrednio do klienta.  

W tym scenariuszu firma Microsoft rozlicza i zbiera w Twoim imieniu. Rozliczenia Microsoft eliminuje konieczność utworzenia relacji zamówienia z klientem. Rozliczenia pojedynczego mogą zaoszczędzić czas i zasoby. Rozliczenia pojedynczego mogą pomóc Ci skoncentrować się na docelowa sprzedaży, zamiast zbieranie rachunku. 

## <a name="enterprise-agreement"></a>Umowa Enterprise  
Jeśli korzystasz z umową Enterprise Agreement (EA) z firmą Microsoft, może używać Twojej umowy EA, płacisz z produktów firmy Microsoft. Może zostać naliczona produktów, w tym użycie platformy Azure. Płatność za pomocą umowy EA jest przeznaczona dla organizacji, które mają licencje na oprogramowanie i usługi w chmurze dla co najmniej trzy lata. Można rozłożyć płatności zamiast jednego dokonywania płatności z góry. Jeśli używasz płatność za rzeczywiste użycie transact typ oferty, następnie naliczania opłat za koszty licencjonowania oprogramowania następuje co kwartał nadwyżkowe cyklu rozliczeniowym umowy EA.  

### <a name="monetary-commitment"></a>Zobowiązanie pieniężne  
Jeśli jesteś klientem z umową EA, Azure może dodać do Umowy. Azure należy dodać do Umowy, podejmując zobowiązanie pieniężne na platformie Azure. Zobowiązania pieniężnego jest wykorzystywane przez cały rok. Zobowiązanie obejmuje dowolną kombinację użycia szerokiej gamy usług na platformie Azure.
