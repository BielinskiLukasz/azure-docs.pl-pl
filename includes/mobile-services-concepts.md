## <a name="what-is"></a>Mobile Services とは

Azure Mobile Services は拡張性の高いモバイル アプリケーション開発プラットフォームです。Azure を使用するとモバイル デバイスのアプリケーションに強化された機能を追加できます。 

Mobile Services を使用すると次ができるようになります。 

+ **ネイティブやクロス プラットフォーム アプリ作成** -接続、iOS、Android、Windows、またはクロスプラット フォーム Xamarin や Cordova (Phonegap) アプリケーションをネイティブ Sdk を使用して、バックエンドのモバイル サービスです。  
+ **ユーザーにプッシュ通知を送信** -アプリのユーザーにプッシュ通知を送信します。
+ **ユーザー認証を行う** -してアプリケーションのユーザーを認証するには、Facebook、Twitter などの一般的な id プロバイダーを利用します。
+ **クラウドにデータを格納** -(既定)、SQL データベースにユーザー データを保存または Mongo DB、DocumentDB、Azure テーブル、または Azure Blob です。 
+ **同期のオフライン対応アプリの構築** -アプリがオフラインで作業して、バック グラウンドでデータを同期するモバイル サービスを使用してください。
+ **監視し、アプリをスケール** : アプリの使用状況を監視し、需要の増加に伴い、バックエンドを拡張します。 

## <a name="concepts"> </a>Mobile Services の概念

Mobile Services の主要な機能と概念は次のとおりです:

+ **アプリケーション キー:** 任意のクライアントから、モバイル サービスにアクセスを制限するために使用する一意の値「キー」がセキュリティ トークンではありませんし、アプリのユーザーを認証するのには使用されません。    
+ **バックエンド:** アプリケーションをサポートするモバイル サービス インスタンス。 モバイル サービスが実装されている ASP.NET Web API プロジェクトとして (*.NET バックエンド* ) や Node.js プロジェクト (*JavaScript バックエンド*)。
+ **Id プロバイダー:** 外部のサービスがモバイル サービスによって信頼されるアプリケーションのユーザーを認証します。 サポートされているプロバイダーには Facebook、Twitter、Google、Microsoft Account、 Azure Active Directory が含まれます。 
+ **プッシュ通知:** 登録されたデバイスまたは Azure Notification Hubs を使用してユーザーに送信されるサービスが発信するメッセージ。
+ **小数点以下桁数:** を追加する機能、追加のコスト、複数の処理能力、パフォーマンス、およびストレージのように、アプリケーションが広く普及しました。
+ **スケジュールされたジョブ:** あらかじめ決められたスケジュールまたはオンデマンドで実行されるカスタム コードです。
