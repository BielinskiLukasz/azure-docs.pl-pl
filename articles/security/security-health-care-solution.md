# <a name="a-practical-guide-to-designing-secure-health-care-solutions-in-azure"></a>Praktyczny przewodnik dotyczący projektowania bezpiecznych rozwiązań opieki zdrowotnej na platformie Azure
Startupy branży ochrony zdrowia, integratorzy systemów (SIs), niezależnym dostawcom oprogramowania (ISV) i organizacji opieki zdrowotnej, biorąc pod uwagę przeniesienie do systemu Azure szukasz wskazówek, która pomoże mu dołączyć kontroli zabezpieczeń zobowiązań zgodności.

[Praktyczny przewodnik dotyczący projektowania bezpiecznych rozwiązań opieki zdrowotnej w systemie Microsoft Azure](https://aka.ms/azureindustrysecurity) pomaga zrozumieć, jak poprawić zabezpieczenia dla rozwiązań przy użyciu usług platformy Azure i funkcje, które można skonfigurować zgodnie z wymaganiami.
Zawartość jest podzielony na trzy główne sekcje:

1. Zagadnienia dotyczące wskazówki dotyczące korzystania z technologii chmury, w tym zarządzania ryzykiem udostępnione odpowiedzialności, ustanawiania systemu zarządzania zabezpieczeniami informacji, poznawania branżowych i lokalnych przepisów prawa oraz ustanawiania standardowych procedur działania.
2. Zasad bezpieczeństwa, które są wyrównane do standardowego standardem bezpieczeństwa informacji zarządzania, np. ISO 27001 oraz tworzenia w standardzie procesy, takie jak cykl projektowania zabezpieczeń (SDL) firmy Microsoft.
3. Stosowanie zasad kluczy z przypadkami użycia poprzez zademonstrowanie wyrównania z punktu widzenia Architekt rozwiązań, gdy wymagania dotyczące rozwiązania są wyrównane do standardowego zarządzania bezpieczeństwem informacji.

Mamy nadzieję, że możesz znaleźć [praktyczny przewodnik dotyczący projektowania bezpiecznych rozwiązań opieki zdrowotnej](https://aka.ms/azureindustrysecurity) przydatne i Daj nam znać, jeśli masz pytania lub sugestie, pozostawiając komentarz poniżej.
