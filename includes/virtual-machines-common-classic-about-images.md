

Obrazy są używane na platformie Azure w celu zapewnienia nowej maszyny wirtualnej z systemem operacyjnym. Obraz może również mieć co najmniej jeden dysk danych. Obrazy są dostępne z wielu źródeł:

* Platforma Azure udostępnia obrazy w [Marketplace](https://azure.microsoft.com/gallery/virtual-machines/). Dostępne są nowe wersje systemu Windows Server i dystrybucje systemu operacyjnego Linux. Niektóre obrazy zawierają również aplikacji, takich jak SQL Server. Subskrybenci MSDN, płatności i korzyści MSDN mają dostęp do dodatkowych obrazów.
* Społeczności "open source" oferuje obrazy za pomocą [VM Depot](http://vmdepot.msopentech.com/List/Index).
* Można również przechowywać i używać własnych obrazów na platformie Azure, przez Przechwytywanie istniejącej maszyny wirtualnej platformy Azure do użycia jako obraz lub przekazanie obrazu.

## <a name="about-vm-images-and-os-images"></a>Dotyczące obrazów maszyn wirtualnych i obrazy systemu operacyjnego
Dwa typy obrazów może służyć w systemie Azure: *obrazu maszyny Wirtualnej* i *obrazu systemu operacyjnego*. Obraz maszyny Wirtualnej zawiera system operacyjny i wszystkie dyski dołączone do maszyny wirtualnej, po utworzeniu obrazu. Obraz maszyny Wirtualnej jest nowszy typ obrazu. Przed wprowadzeniem obrazów maszyn wirtualnych, obraz na platformie Azure może mieć tylko uogólnionego systemu operacyjnego i żadne dodatkowe dyski. Obraz maszyny Wirtualnej, który zawiera tylko uogólnionego systemu operacyjnego jest zasadniczo taki sam jak oryginalny typ obrazu, obraz systemu operacyjnego.

Możesz utworzyć własne obrazy, na podstawie maszyny wirtualnej na platformie Azure lub maszyna wirtualna uruchomiona w innym miejscu, kopiowanie i przekazywanie. Jeśli chcesz użyć obrazu, aby utworzyć więcej niż jedną maszynę wirtualną, należy przygotować go do użycia jako obraz, korzystając z funkcji uogólniania go. Aby utworzyć obraz systemu Windows Server, należy uruchomić polecenie Sysprep na serwerze, aby uogólnić go, aby przekazać plik VHD. Aby uzyskać szczegółowe informacje o narzędziu Sysprep, zobacz [sposobu użycia programu Sysprep: wprowadzenie](http://go.microsoft.com/fwlink/p/?LinkId=392030) i [Obsługa narzędzia Sysprep dla ról serwera](https://msdn.microsoft.com/windows/hardware/commercialize/manufacture/desktop/sysprep-support-for-server-roles). Tworzenie kopii zapasowej maszyny Wirtualnej przed uruchomieniem programu Sysprep. Tworzenie obrazu systemu Linux jest zależna od dystrybucji. Zwykle musisz uruchomić zestaw poleceń, które są specyficzne dla dystrybucji i uruchom agenta systemu Linux dla platformy Azure.
