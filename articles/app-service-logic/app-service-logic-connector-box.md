# Box コネクタの使用開始とロジック アプリへの追加 
アップロード、削除を Box に接続します。 その他のファイルへ コネクタは、"ワークフロー" の一部として Logic Apps で使用されます。 

状況によっては、ファイアウォールの外側にいる人ともデータを安全に共有できる Box を操作しなければならない場合があります。 ロジック アプリはさまざまなデータ ソースを基にトリガーでき、フローの一環としてデータの取得と処理のためのコネクタを提供します。


## トリガーとアクション
Box のギャラリー アプリには、Box を操作する仕組みとしてアクションが用意されています。

**アクション**: アクションでは、ロジック アプリに構成済みの Box アカウントで定義済みのアクションを実行することができます。 Box コネクタを使用して Box アカウントで実行できるアクションを次に示します。

a. *ファイルの一覧:* この操作は、フォルダー内のすべてのファイルの情報を返します。 アクションの必要なパラメーターの一覧:  

パラメーター名 | 説明 | 必須
--- | --- | ---
フォルダー パス | 一覧表示するフォルダーのパス。 | あり

> [AZURE.NOTE] ファイル コンテンツは返しません。

b. *Get ファイル:* この操作は、コンテンツとプロパティを含むファイルを取得します。 アクションに必要なパラメーターの一覧:

パラメーター名 | 説明 | 必須
--- | --- | ---
ファイル パス | ファイルが存在するフォルダーのパス。 | あり
ファイルの種類 | ファイルがテキストかバイナリかを指定します。 | いいえ

> [AZURE.NOTE] この操作は、読み取り後に、ファイルを削除されません。


c. *ファイルをアップロード*: 名前からわかるように、アクションには、Box アカウントにファイルをアップロードします。 ファイルが既に存在する場合、上書きはされず、エラーがスローされます。 アクションに必要なパラメーターの一覧:

パラメーター名 | 説明 | 必須
--- | --- | ---
ファイル パス | ファイルへのパス。 | あり
ファイル コンテンツ | アップロードされるファイルのコンテンツ。 | あり
コンテンツ転送エンコード | コンテンツのエンコードの種類。[Base64] または [なし]。 | 

d. *ファイルを削除*: フォルダーから、指定したファイルを削除します。 ファイルまたはフォルダーが見つからない場合は、例外がスローされます。 アクションに必要なパラメーターの一覧:

パラメーター名 | 説明 | 必須
--- | --- | ---
ファイル パス | フォルダーを含む完全なファイル パス。 | あり


## ロジック アプリの Box コネクタの作成

コネクタは、ロジック アプリ内で作成することも、Azure Marketplace から直接作成することもできます。 Marketplace からコネクタを作成するには、次の操作を実行します。  

1. Azure のスタート画面では、次のように選択します。 **Marketplace**します。
2. 「Box コネクタ」を検索して選択し、選択 **作成**します。
3. 名前、App Service プラン、その他のプロパティを入力します。  
    ![][1]
4. 選択 **作成**します。


## ロジック アプリでの Box コネクタの使用

API アプリを作成したら、ロジック アプリのアクションとして Box コネクタを使用できます。 これを行うには、次の手順を実行します。

1. ロジック アプリで開く **トリガーとアクション** Logic Apps デザイナーを開き、フローを構成します。 Box コネクタがギャラリーの一覧に表示されます。 この Box コネクタを選択すると、自動的にロジック アプリ デザイナーに追加されます。

    > [AZURE.NOTE] Box コネクタをロジック アプリの開始時に選択した場合は、トリガーのように機能します。 それ以外の場合は、コネクタを使用する Box アカウントに対するアクションとして動作します。 この記事の執筆時点では、Box コネクタにトリガーはありません。

2. ユーザーの代わりに操作を実行するロジック アプリの認証および承認を行います。 選択 **承認** Box コネクタで。  
    ![][2]

3. 操作を実行する Box アカウントの詳細に、記号を入力します。  
    ![][3]

4. あなたの代わりに操作を実行するアカウントにロジック アプリのアクセスを付与します。  
    ![][4]

5. アクションの一覧が表示され、適切な操作を実行することができます。  
    ![][5]

## コネクタでできること
コネクタが作成されたため、ロジック アプリを使用してコネクタをビジネス ワークフローに追加できます。 参照してください [Logic Apps とは?](app-service-logic-what-are-logic-apps.md)します。

>[AZURE.NOTE] Azure Logic Apps と Azure アカウントにサインアップする前に開始する場合に、 [ロジック アプリを実行してください](https://tryappservice.azure.com/?appservice=logic), 、App Service で有効期間の短いスターター ロジック アプリをすぐに作成する場所です。 このサービスの利用にあたり、クレジット カードは必要ありません。契約も必要ありません。

表示、Swagger の REST API リファレンス [コネクタと API Apps のリファレンス](http://go.microsoft.com/fwlink/p/?LinkId=529766)します。

パフォーマンス統計をレビューし、コネクタに対するセキュリティを制御することもできます。 参照してください [の管理し、監視、組み込みの API Apps とコネクタ](app-service-logic-monitor-your-connectors.md)します。
