#カスタム仮想マシンの作成方法

A *カスタム* 仮想マシンが使用して作成する仮想マシンには、 **ギャラリーから** メソッドより多くのオプションを構成することができるので、 **簡易作成** メソッドです。 設定できるオプションは次のとおりです。

- 仮想マシン (VM) の作成に使用するイメージのより豊富な選択肢
- 仮想ネットワークと VM との接続
- 既存のクラウド サービスへの VM の追加
- 可用性セットへの VM の追加

> [AZURE.IMPORTANT] 仮想マシンのホスト名またはクロスプレミス接続の設定した直接接続できますが、仮想ネットワークを使用する場合は、仮想マシンを作成するときに、仮想ネットワークを指定することを確認します。 仮想マシンが仮想ネットワークに接続するように設定できるのは、仮想マシンの作成時に限られています。 仮想ネットワークの詳細については、次を参照してください。 [仮想ネットワークの概要](http://go.microsoft.com/fwlink/p/?LinkID=294063)します。

1. [Azure ポータル](http://manage.windowsazure.com)にサインインします。

2. コマンド バー] をクリックして **新規**します。

3. クリックして **コンピューティング**, 、] をクリックして **仮想マシン**, 、順にクリック **ギャラリーから**します。

4. 使用するイメージを選択し、矢印をクリックして続行します。

5. 複数のバージョンのイメージがあるかで **バージョンのリリース日**, 、使用するバージョンを選択します。

6.  **仮想マシン名**, 、仮想マシンに使用する名前を入力します。

7. 使用 **層** と **サイズ** 、仮想マシンの適切なサイズを選択します。 選択したサイズは、仮想マシンの最大構成だけでなく、料金にも影響を及ぼします。 構成の詳細については、「 [仮想マシンと Azure のクラウド サービスのサイズ](http://go.microsoft.com/fwlink/p/?LinkID=389844)します。

8.  **新しいユーザー名**, 、サーバーの管理に使用する管理者アカウントの名前を入力します。

9.  **新しいパスワード**, 、管理アカウントの強力なパスワードを入力します。  **パスワードの確認**, 、同じパスワードを再度入力します。

10. 矢印をクリックして続行します。

11.  **クラウド サービス**, 、次のいずれかを実行します。

    - クラウド サービスの最初または唯一の仮想マシンの場合は、選択 **新しいクラウド サービスを作成する**です。 その後、[ **クラウド サービス DNS 名**, 、3 ~ 24 文字で、小文字と数字の間で使用する名前を入力します。 この名前は、クラウド サービスを介して仮想マシンにアクセスするときに使用する URI の一部になります。
    - この仮想マシンをクラウド サービスに追加する場合は、一覧からそのクラウド サービスを選択します。

    > [AZURE.NOTE] 同じクラウド サービスにバーチャル マシンの配置の詳細については、次を参照してください。 [クラウド サービス内の仮想マシンを接続する方法](http://www.windowsazure.com/manage/windows/how-to-guides/connect-to-a-cloud-service/)します。

12.  **リージョン/アフィニティ グループ/仮想ネットワーク**, 、リージョン、アフィニティ グループ、または仮想マシンに使用する仮想ネットワークを選択します。 アフィニティ グループについての詳細については、次を参照してください。 [仮想ネットワークのアフィニティ グループについて](../virtual-network/virtual-networks-migrate-to-regional-vnet.md)します。

13.  **ストレージ アカウント**, を VHD ファイルの既存のストレージ アカウントを選択するか自動的に生成されたストレージ アカウントを使用します。 自動的に作成されるストレージ アカウントはリージョンあたり 1 つだけです。 この設定で作成する他のすべての仮想マシンがこのストレージ アカウントに配置されます。 ストレージ アカウントは 20 個に制限されています。

14. 仮想マシンの可用性セットに属しているようにする場合 **可用性セット**, [ **可用性セットの作成** か、既存の可用性セットに追加します。

    **注**: 可用性セットに仮想マシンを異なるフォールト ドメインに配置されます。 可用性セットに複数の仮想マシンがあると、ネットワークやローカル ハード ディスクで問題が発生した場合、または予定されたシステム停止時にもアプリケーションを使用できます。

15.   **エンドポイント**, 、バーチャル マシンなどをリモート デスクトップを使用または Secure Shell (SSH) クライアントへの接続を許可するように作成される新しいエンドポイントを確認します。 ここでエンドポイントを追加したり、後でエンドポイントを作成することもできます。 後で作成する方法の詳細については、次を参照してください。 [仮想マシンに対してエンドポイントを設定する方法](../articles/virtual-machines/virtual-machines-set-up-endpoints.md)します。

16.   **VM エージェント**, 、VM エージェントをインストールするかどうかを決定します。 このエージェントには、仮想マシンの操作に役立つ拡張機能をインストールするための環境が用意されています。 詳細については、「 [拡張機能の管理](http://go.microsoft.com/FWLink/p/?LinkID=390493)します。

17. 矢印をクリックして、仮想マシンを作成します。

    ![カスタム仮想マシンの作成に成功](./media/howto-custom-create-vm/VMSuccessWindows.png)

##次のステップ##
仮想マシンは、作成し終わると、自動的に開始されます。 ポータルのステータスが実行中であれば、仮想マシンにログインできます。 手順については、次のいずれかの記事を参照してください。

- [Linux を実行する仮想マシンにログオンする方法](../articles/virtual-machines/virtual-machines-linux-how-to-log-on.md)
- [Windows Server が実行されている仮想マシンにログオンする方法](../articles/virtual-machines/virtual-machines-log-on-windows-server.md)
