## <a name="azure-expressroute-support"></a>Pomoc techniczna usługi Azure ExpressRoute

Można wywrzeć zapas danych za pomocą usługi Azure ExpressRoute za pomocą publicznej komunikacji równorzędnej (dostępnej dla starych obwodów) i komunikacji równorzędnej firmy Microsoft. Tworzenie kopii zapasowych za pomocą prywatnej komunikacji równorzędnej nie jest obsługiwane.

Komunikacja równorzędna publiczna: zapewnij dostęp do następujących domen/adresów:

* `http://www.msftncsi.com/ncsi.txt`
* `microsoft.com`
* `.WindowsAzure.com`
* `.microsoftonline.com`
* `.windows.net`

W komunikacji równorzędnej firmy Microsoft wybierz następujące usługi/regiony i odpowiednie wartości społeczności:

* Usługa Azure Active Directory (12076:5060)
* Region Platformy Microsoft Azure (w zależności od lokalizacji magazynu usług odzyskiwania)
* Usługa Azure Storage (w zależności od lokalizacji magazynu usług odzyskiwania)

Aby uzyskać więcej informacji, zobacz [wymagania dotyczące routingu usługi ExpressRoute](https://docs.microsoft.com/azure/expressroute/expressroute-routing).

>[!NOTE]
>Publiczne peering jest przestarzałe dla nowych obwodów.

## <a name="cluster-support"></a>Obsługa klastrów

Usługa Azure Backup Server może chronić dane w następujących aplikacjach klastrowanych:

* Serwery plików

* SQL Server

* Funkcja Hyper-V — jeśli chronisz klaster funkcji Hyper-V przy użyciu skalowanego w poziomie agenta ochrony MABS, nie można dodać dodatkowej ochrony chronionych obciążeń funkcji Hyper-V.

    Jeśli program Hyper-V jest uruchamiany w systemie Windows Server 2008 R2, należy zainstalować aktualizację opisaną w kb [975354](https://support.microsoft.com/kb/975354).
    Jeśli program Hyper-V jest uruchamiany w systemie Windows Server 2008 R2 w konfiguracji klastra, upewnij się, że zainstalowano dodatek SP2 i KB [971394](https://support.microsoft.com/kb/971394).

* Exchange Server — serwer kopii zapasowej platformy Azure może chronić klastry dysków nieuwzwolonych dla obsługiwanych wersji programu Exchange Server (replikacja ciągła klastra), a także chronić program Exchange Server skonfigurowany do lokalnej replikacji ciągłej.

* SQL Server — usługa Azure Backup Server nie obsługuje tworzenia kopii zapasowych baz danych programu SQL Server hostowanych na woluminach udostępnionych przez klaster (CSV).

Usługa Azure Backup Server może chronić obciążenia klastra, które znajdują się w tej samej domenie co serwer MABS oraz w domenie podrzędnej lub zaufanej. Jeśli chcesz chronić źródła danych w niezaufanych domenach lub grupach roboczych, użyj uwierzytelniania NTLM lub certyfikatu dla pojedynczego serwera lub uwierzytelniania certyfikatów tylko dla klastra.
