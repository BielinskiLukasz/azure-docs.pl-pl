# <a name="understanding-azure-virtual-machine-usage"></a>Zrozumienie wykorzystania maszyn wirtualnych platformy Azure

## <a name="image-type"></a>Typ obrazu
Dla niektórych obrazów w galerii systemu Azure typ obrazu jest wypełniana w polu dodatkowe informacje. Dzięki temu użytkownicy do zrozumienia i śledzić, co wdrożonych na swojej maszynie wirtualnej. Następujące wartości, które są wypełnione w tym polu, na podstawie obrazu wdrożono:
  - BitRock 
  - Canonical 
  - FreeBSD 
  - Otwórz logiki 
  - Oracle 
  - SLES dla rozwiązania SAP 
  - Wersję zapoznawczą programu SQL Server 14 w systemie Windows Server 2012 R2 Preview 
  - SUSE
  - SUSE Premium
  - Urządzenie StorSimple w chmurze 
  - Red Hat
  - Red Hat for SAP Business Applications     
  - Red Hat dla platformy SAP HANA 
  - Klient Windows BYOL 
  - System Windows Server w ramach opcji BYOL 
  - Windows Server w wersji zapoznawczej 

## <a name="service-type"></a>Typ usługi
Pole Typ usługi, w polu informacje dodatkowe odpowiada określić dokładny rozmiar maszyny Wirtualnej, wdrożone. Magazyn w warstwie Premium (oparte na SSD) maszyn wirtualnych są wyceniane taki sam jak inne niż premium storage maszyn wirtualnych (opartych na dysk twardy). W przypadku wdrożenia rozmiar oparte na dyskach SSD, takich jak standardowy\_DS2\_v2, zostanie wyświetlony rozmiar innym niż SSD ("standardowe\_D2\_maszyny Wirtualnej v2") w kolumnie Podkategoria licznika, a rozmiar SSD ("standardowe\_DS2\_ v2 ") w polu dodatkowe informacje.

## <a name="virtual-machine-usage-faq"></a>Użycie maszyny wirtualnej — często zadawane pytania
### <a name="what-resources-are-charged-when-deploying-a-vm"></a>Jakie zasoby są naliczane w przypadku wdrażania maszyny Wirtualnej?    
Maszyn wirtualnych są określane kosztów samej maszyny wirtualnej, premium oprogramowanie uruchomione na maszynie Wirtualnej, skojarzony z maszyną Wirtualną, dysk account\managed magazynu i przepustowości sieci przesyła z maszyny Wirtualnej.
### <a name="how-can-i-tell-if-a-vm-is-using-azure-hybrid-benefit-in-the-usage-csv"></a>Jak sprawdzić, czy maszyny Wirtualnej korzysta z korzyści użycia hybrydowego platformy Azure w woluminie CSV użycia?
Jeśli wdrażasz za pomocą [korzyść użycia hybrydowego platformy Azure](https://azure.microsoft.com/pricing/hybrid-benefit/), opłaty są naliczane inne niż Windows na maszynie Wirtualnej stanowiąca, ponieważ to użycie własnej licencji do chmury. Na rachunku, można odróżnić, które maszyny wirtualne usługi Resource Manager są uruchomione korzyść użycia hybrydowego platformy Azure, ponieważ mają one "Windows\_serwera BYOL" lub "Windows\_klienta BYOL" w kolumnie ImageType.
### <a name="how-are-basic-vs-standard-vm-types-differentiated-in-the-usage-csv"></a>Jak są podstawowe vs. Standardowe typy maszyn wirtualnych ze zróżnicowanych w woluminie CSV użycia?
Podstawowa i standardowa maszyny wirtualne z serii A są oferowane. W przypadku wdrożenia podstawowej maszyny Wirtualnej w Podkategoria licznika ma ciąg "Podstawowy". W przypadku wdrożenia maszyny Wirtualnej z systemem standardowa — seria A rozmiar maszyny Wirtualnej pojawi się jako "Maszyna wirtualna A1", ponieważ Standard jest ustawieniem domyślnym. Aby dowiedzieć się więcej na temat różnic między podstawowa i standardowa, zobacz [strona cennika](https://azure.microsoft.com/pricing/details/virtual-machines/).
### <a name="what-are-extrasmall-small-medium-large-and-extralarge-sizes"></a>Co to są rozmiary ExtraSmall, małe, średnie, duże i ExtraLarge?
ExtraSmall — ExtraLarge są starsze nazwy dla warstwy standardowej\_A0 – Standard\_A4. Rekordy użycia klasycznej maszyny Wirtualnej może zawierać tę Konwencję używane, jeśli wdrożono tych rozmiarów.
### <a name="what-is-the-difference-between-meter-region-and-resource-location"></a>Jaka jest różnica między Region licznika i lokalizacja zasobu?
Region miernika jest skojarzony z licznika. Dla niektórych usług platformy Azure, którzy korzystają z jednej cenie we wszystkich regionach pole Region licznika może być pusta. Jednakże ponieważ maszyny wirtualne są wyposażone w dedykowane ceny na region w przypadku maszyn wirtualnych, to pole jest wypełniane. Podobnie lokalizacja zasobów dla maszyn wirtualnych jest lokalizacja, w którym wdrożono maszynę Wirtualną. Regiony platformy Azure w obu polach są takie same, jednak mogą pojawić się innym ciągiem Konwencja nazwa regionu.
### <a name="why-is-the-imagetype-value-blank-in-the-additional-info-field"></a>Dlaczego jest wartość ImageType puste pola dodatkowe informacje?
Tylko wypełnianie pola ImageType dla podzbioru obrazów. Jeśli nie zostały wdrożone, jeden z obrazów powyżej, ImageType jest pusta.
### <a name="why-is-the-vmname-blank-in-the-additional-info"></a>Nazwy jest puste w dodatkowe informacje
Nazwy tylko zapełnieniu pola dodatkowe informacje dla maszyn wirtualnych w zestawie skalowania. Pole Identyfikator wystąpienia zawiera nazwę maszyny Wirtualnej dla maszyn wirtualnych w zestawie skalowania nie.
### <a name="what-does-computehr-mean-in-the-usagetype-field-in-the-additional-info"></a>Co oznacza ComputeHR w polu UsageType w dodatkowe informacje?
Koszt godzin obliczeniowych, który reprezentuje zdarzenia użycia podstawowej cenie infrasturcture oznacza ComputeHR. W przypadku UsageType ComputeHR\_SW reprezentuje zdarzenia użycia oprogramowania premium opłata za maszynę Wirtualną.
### <a name="how-do-i-know-if-i-am-charged-for-premium-software"></a>Jak sprawdzić, jeśli I jest naliczana za oprogramowanie premium?
Podczas eksplorowania obrazu maszyny Wirtualnej, która najlepiej spełnia Twoje potrzeby, należy koniecznie zapoznaj się z [portalu Azure Marketplace](https://azuremarketplace.microsoft.com/marketplace/apps/category/compute). Obraz, który ma stawkę plan oprogramowania. Jeśli widzisz "Bezpłatny" szybkość istnieje bez dodatkowych kosztów oprogramowania. 
### <a name="what-is-the-difference-between-microsoftclassiccompute-and-microsoftcompute-in-the-consumed-service"></a>Jaka jest różnica między Microsoft.ClassicCompute i Microsoft.Compute w usłudze zużyto?
Obszar Microsoft.ClassicCompute reprezentuje klasyczne zasoby wdrożone za pośrednictwem usługi Azure Service Manager. W przypadku wdrożenia przy użyciu usługi Resource Manager w użyta usługa została wpisana Microsoft.Compute. Dowiedz się więcej o [modele wdrażania Azure](https://docs.microsoft.com/azure/azure-resource-manager/resource-manager-deployment-model).
### <a name="why-is-the-instanceid-field-blank-for-my-virtual-machine-usage"></a>Pole Identyfikator wystąpienia jest puste w przypadku użycia maszyny wirtualnej
W przypadku wdrożenia przy użyciu klasycznego modelu wdrażania, ciąg InstanceID nie jest dostępna.
### <a name="why-are-the-tags-for-my-vms-not-flowing-to-the-usage-details"></a>Dlaczego są znaczniki dla moich maszyn wirtualnych nie przesyła danych, aby uzyskać szczegółowe informacje o?
Tagi przepływać tylko dla Ciebie CSV użycia dla maszyn wirtualnych usługi Resource Manager tylko. Tagi zasobów klasycznych nie są dostępne w obszarze szczegółów użycia.
### <a name="how-can-the-consumed-quantity-be-more-than-24-hours-one-day"></a>Jak zużyta ilość może być dłuższy niż 24 godziny na jeden dzień
W modelu klasycznym opłaty za zasoby są agregowane na poziomie usługi w chmurze. Jeśli masz więcej niż jedną maszynę Wirtualną w usłudze w chmurze, która używa tego samego wpływu na rozliczenia, użycie jest agregowane ze sobą. Maszyny wirtualne wdrożone przy użyciu usługi Resource Manager są naliczane na poziomie maszyny Wirtualnej, dzięki czemu nie będzie stosowana ta agregacja.
### <a name="why-is-pricing-not-available-for-dsfsgsls-sizes-on-the-pricing-page"></a>Dlaczego jest nie dostępne ceny dla rozmiarów DS/FS/GS/LS na stronie cennika?
Usługa Premium storage zdolne do maszyn wirtualnych są rozliczane według stawki takie same, jak inne niż premium storage zdolne do maszyn wirtualnych. Różnią się tylko koszty magazynowania. Odwiedź stronę [magazynu stronę z cennikiem](https://azure.microsoft.com/pricing/details/storage/unmanaged-disks/) Aby uzyskać więcej informacji.
