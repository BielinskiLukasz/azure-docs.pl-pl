# Azure Data Catalog の新機能

更新プログラムを **Azure Data Catalog** を定期的にリリースします。 すべてのリリースにユーザー向けの新機能が含まれているわけではなく、一部のリリースはバックエンド サービス機能向けです。 このページに追加ユーザー向けの新機能を強調表示、 **Azure Data Catalog** サービスです。

## 2015 年 11 月 20 日の週のリリースの新機能

2015 年 11 月 20 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- 表示し、内から接続文字列をコピーするには、機能、 **Azure Data Catalog** (Azure SQL データベースを含む)、SQL Server と Oracle データ ソースのポータルです。 SQL Server または Oracle のテーブル、ビュー、またはデータベースの接続情報に含まれる [接続文字列の表示] リンクをクリックすることで、データ ソースに接続するために使用される接続文字列を確認できます。 SQL Server データ ソースには、ADO.NET、ODBC、OLEDB、および JDBC 接続文字列があります。 Oracle データ ソースには、ODBC および OLEDB 接続文字列があります。
- Teradata のテーブルとビューの登録時にデータ プロファイルを含めることのサポート。
- SQL Server (Azure SQL DB と Azure SQL Data Warehouse を含む)、Server Analysis Services、Azure Storage、および HDFS ソースに対する [Power BI Desktop で開く] のサポート。  

> [AZURE.NOTE] 「Power BI Desktop で開いた状態」では、現在のバージョンの Power BI Desktop のアプリケーションをインストールする必要があります。 問題や、この機能を使用してエラーが発生した場合は、Power BI Desktop からの最新バージョンがあることを確認してください。 [PowerBI.com](https://powerbi.com)します。

## 2015 年 11 月 13 日の週のリリースの新機能

2015 年 11 月 13 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- Teradata データ ソース用の LDAP 認証のサポート。 Teradata のテーブルとビューの登録時に、TD2 認証ではなく LDAP 認証を使用して Teradata に接続することを選択できます。
- Teradata データ ソースに対する [Excel で開く] のサポート。
- 最近使用した検索用語のサポート、 **Azure Data Catalog** ポータルです。 ポータルで検索するときに、最近使用した検索語句から選択することで、検索時間を短縮できます。

## 2015 年 11 月 6 日の週のリリースの新機能

2015 年 11 月 6 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- Teradata データ ソースのプレビューのサポート。 Teradata のテーブルとビューを登録するとき、ユーザーはデータ ソースから抽出されたメタデータのスナップショット レコードを含めることができます。
- Azure SQL Data Warehouse データ ソースに対する [Excel で開く] のサポート。
- 手動で登録されたデータ資産に対する列レベルのスキーマの定義と編集のサポート。 使用したデータ資産を手動で作成した後、 **Azure Data Catalog** ポータルで、ユーザーできる列の定義の追加データ資産のプロパティです。
- サポートの「には、」クエリ検索するときに **Azure Data Catalog**, 、固有のメタデータを保持している登録されているデータ資産の検出を有効にします。 **Azure Data Catalog** クエリ構文になりました。

| クエリ構文            | 目的 |
|-------------------------|---------|
| has:previews             | プレビューを含むデータ資産を検索します。  |
| has:documentation       | ドキュメントが提供されているデータ資産を検索します。 |
| has:tableDataProfiles   | テーブル レベルのデータ プロファイル情報でデータ資産を検索します。 |
| has:columnsDataProfiles | 列レベルのデータ プロファイル情報でデータ資産を検索します。 |


## 2015 年 10 月 30 日の週のリリースの新機能

2015 年 10 月 30 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- 登録済みのデータ ソースデータに関するデータ プレビューおよびデータ プロファイルの保存時に暗号化をサポートします。 **Azure Data Catalog** プレビュー レコードを透過的に暗号化し、プロファイル、カタログ管理者キーの管理することがなく、サービスに登録されているデータ ソースのデータです。

## 2015 年 10 月 23 日の週のリリースの新機能

2015 年 10 月 23 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- Teradata データ ソースのサポート。 Teradata のテーブルとビューの登録と検出を実行できるようになりました。

> [AZURE.NOTE] 現在のリリースでは、Teradata TD2 認証のみがサポートされます。 今後、他の認証メカニズムもサポートされる予定です。

## 2015 年 10 月 16 日の週のリリースの新機能

2015 年 10 月 16 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- オンプレミス Hive データ ソースのサポート。 ユーザーは、Hadoop オンプレミス データ ソースで Apache Hive 用の Hive テーブルを登録し、検出できるようになりました。
- [保存された検索のサポート、 **Azure Data Catalog** ポータルです。 ユーザーは検索語句とフィルター選択項目を保存することで、簡単に以前の検索を繰り返し、カタログのコンテンツの便利なビューを定義することができます。 また、ユーザーは既定の検索として保存した検索をマークすることもできます。 ユーザーがから、「拡大鏡」検索アイコンをクリックすると、 **Azure Data Catalog** ポータルのホーム ページまたは「はじめに」ページから、ユーザーが既定値としてフラグが設定された保存された検索を直接なされるようにします。


## 2015 年 10 月 9 日の週のリリースの新機能

2015 年 10 月 9 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- Azure Data Catalog ポータルで、登録済みのデータ資産とコンテナーに関するリッチ テキスト ドキュメントがサポートされるようになりました。 ユーザーは、タグや説明が不十分なシナリオで、データ資産 (テーブル、ビュー、レポートなど) とコンテナー (データベース、モデルなど) に関するドキュメントを提供できます。

## 2015 年 10 月 2 日の週のリリースの新機能

2015 年 10 月 2 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- 既知のデータ ソースの種類を手動で登録できるようになりました。 ユーザーは、データ ソース情報を使用して手動で入力できます、 **Azure Data Catalog** でサポートされているすべてのデータ ソースの種類のポータル **Azure Data Catalog**します。
- Azure Active Directory セキュリティ グループを承認できるようになりました。 カタログ管理者には、カタログ グループへのアクセスのセキュリティも、ユーザー アカウントかへのアクセスを管理しやすくなりますが有効にすることができます **Azure Data Catalog**します。
- 使用して Excel で Hive データ ソースを開くためのサポート、 **Azure Data Catalog** ポータルです。

> [AZURE.NOTE] Hive データ ソースの"Excel で開いている"機能を使用するには、ユーザー インストールする必要がありますが、ODBC ドライバー Hive 用です。

## 2015 年 9 月 25 日の週のリリースの新機能

2015 年 9 月 25 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

## 2015 年 9 月 18 日の週のリリースの新機能

2015 年 9 月 18 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- 新しい"getting started"のデータ ソースの検出エクスペリエンス、 **Azure Data Catalog** ポータルです。 ユーザーが、「検出」のページを入力すると、  **Azure Data Catalog** ポータルで、検索語句を入力しなくてが表示されます、最もよく使用されるタグとデータ ソースの種類とオブジェクトの種類の専門家を含むカタログの内容の概要を説明します。
- Azure SQL Data Warehouse のオブジェクトおよびデータベースを登録して検出するためのサポート。 追加の詳細については、Azure SQL Data Warehouse では、次を参照してください。 [SQL Data Warehouse](http://azure.microsoft.com/services/sql-data-warehouse/)します。
- SQL Server Analysis Services モデルおよび SQL Server Reporting Services サーバーをコンテナーとして登録して検出するためのサポート。 SSAS や SSRS のオブジェクトを登録するときに **Azure Data Catalog** SSAS モデルおよびレポートやその他のオブジェクトの場合と同様に、SSRS サーバーのエントリが作成されます。 コンテナーが検出できるしを使用して注釈が付けられた、 **Azure Data Catalog** ポータルです。 カタログのコンテンツを検索し、フィルター処理するだけでなく、モデルまたはサーバーのコンテンツを検索し、フィルター処理することもできます。

> [AZURE.NOTE] 9 月 18 日リリースする前に登録済みの SSAS や SSRS のオブジェクトは、モデルまたはサーバーのエントリは、カタログに追加する前に、データ ソース登録ツールを使用して再登録する必要があります。 データ ソースを再登録するには影響しません内のユーザーが追加されている任意の注釈、 **Azure Data Catalog** ポータルです。

## 2015 年 9 月 11 日の週のリリースの新機能

2015 年 9 月 11 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- HTTP/HTTPS を介した SQL Server Analysis Services オブジェクトの登録および検出をサポートします。 ユーザーは、サーバー名ではなく (https://servername/olap/msmdpump.dll) などの URL を使用して SSAS サーバーに接続できるようになりましたと基本認証と Windows 認証の他の匿名接続を使用することができます。 SSAS への HTTP または HTTPS 接続の詳細については、次を参照してください。 [Analysis Services への HTTP アクセスの構成](https://msdn.microsoft.com/library/gg492140.aspx)します。
- HDInsight で Hive データ ソースをサポートします。 ユーザーは、HDInsight データ ソースにおいて、Hadoop の Apache Hive に対する Hive テーブルを登録および検出できるようになりました。 HDInsight での hive の詳細については、次を参照してください。、 [HDInsight ドキュメント センター](../hdinsight-use-hive/)します。
- Oracle データベースと HDFS クラスターをコンテナーとして登録し、検出できるようになりました。 Oracle テーブルとビュー、または、hdfs 上に登録するときに **Azure Data Catalog** テーブルとビューの場合と同様のデータベースのエントリが作成されます。 データベースが検出できるし、を使用して注釈が付けられた、 **Azure Data Catalog** ポータルです。 カタログのコンテンツを検索し、フィルター処理するだけでなく、データベースまたはクラスターのコンテンツを検索し、フィルター処理することもできます。


> [AZURE.NOTE] Oracle テーブルとビューと HDFS のファイルとディレクトリ 11 年 9 月のリリースする前に登録済みの再登録するデータベースまたはクラスターのエントリは、カタログに追加する前に、データ ソース登録ツールを使用します。 データ ソースを再登録するには影響しません内のユーザーが追加されている任意の注釈、 **Azure Data Catalog** ポータルです。

## 2015 年 9 月 4 日の週のリリースの新機能

2015 年 9 月 4 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- 不明なデータ ソースを手動で登録できるようになりました。 ユーザーは、データ ソース情報を使用して手動で入力できます、 **Azure Data Catalog** ポータルで、データ ソース登録ツールで明示的にサポートされているデータ ソースの注釈が付けられたして検出できるようにします。
- SQL Server データベースをコンテナーとして登録し、検出できるようになりました。 SQL Server のテーブルとビューを登録するときに **Azure Data Catalog** テーブルとビューの場合と同様のデータベースのエントリが作成されます。 データベースが検出できるし、を使用して注釈が付けられた、 **Azure Data Catalog** ポータルです。 カタログのコンテンツを検索し、フィルター処理するだけでなく、データベースのコンテンツを検索し、フィルター処理することもできます。

> [AZURE.NOTE] SQL Server テーブルとビューが 4 年 9 月のリリースする前に登録された再登録するデータベースのエントリは、カタログに追加する前に、データ ソース登録ツールを使用します。 データ ソースを再登録するには影響しません内のユーザーが追加されている任意の注釈、 **Azure Data Catalog** ポータルです。

## 2015 年 8 月 28 日の週のリリースの新機能

2015 年 8 月 28 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- SQL Server と Oracle のデータ ソースのデータプロファイルのサポート。 SQL Server と Oracle のテーブルとビューを登録するときに、登録するオブジェクトのデータ プロファイル情報を含めることを選択できます。 データ プロファイルには、オブジェクト レベルと列レベルの統計情報が含まれます。
- Hadoop HDFS データ ソースのサポート。 HDFS のファイルとディレクトリを登録および検出できるようになりました。

## 2015 年 8 月 21 日の週のリリースの新機能

2015 年 8 月 21 日の週の時点で、次の機能に加えられた **Azure Data Catalog**:

- 登録されているデータ ソースに対するアクセス要求情報の提供のサポート。 任意の登録済みデータ資産について、ユーザーは電子メール リンクや URL などのアクセス要求方法を提供し、既存のツールおよびプロセスと簡単に統合できます。
- 登録されているデータ資産に対してどのようなユーザーがどのようなメタデータを提供したのかを簡単に検出できるようにするための、タグとエキスパートに関するヒント。
- 上部のナビゲーション バーに新しい [ユーザー] ボタンとメニューを追加しました。 この] メニューで、ユーザーへのログオンに使用するアカウントを参照してください **Azure Data Catalog**, 、必要な場合は、サインアウトするとします。 このメニューでは、使用している開発者にとって有用なカタログの名前も表示されます、 **Azure Data Catalog** REST API です。
- Standard Edition のみ: は、データ資産に所有者を追加するときに **Azure Data Catalog** ユーザー アカウントとセキュリティ グループを所有者としての両方がサポートされました。 選択したデータ資産の所有者としてセキュリティ グループを追加する場合、グループの表示名またはグループの UPN 電子メール アドレス (ある場合) のいずれかを入力できます。
- Azure BLOB Storage データ ソースのサポート。 ユーザーは、Azure Storage の BLOB およびディレクトリを登録および検出できるようになりました。
