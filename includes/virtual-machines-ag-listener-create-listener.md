この手順では、フェールオーバー クラスター マネージャーおよび SQL Server Management Studio (SSMS) で可用性グループ リスナーを手動で作成します。

1. プライマリ レプリカをホストしているノードからフェールオーバー クラスター マネージャーを開きます。

1. 選択、 **ネットワーク** ノードを展開し、クラスター ネットワーク名を書き留めます。 この名前は、PowerShell スクリプトで $ClusterNetworkName 変数に使用されます。

1. クラスター名を展開し、クリックして **ロール**します。

1.  **ロール** ] ウィンドウで、可用性グループ名を右クリックし、[ **リソースの追加** > **クライアント アクセス ポイント**します。

    ![可用性グループのクライアント アクセス ポイントを追加する](./media/virtual-machines-sql-server-configure-alwayson-availability-group-listener/IC678769.gif)

1.  **名前** ボックスに、この新しいリスナーの名前を作成、クリックして **次** を 2 回順にクリック **完了**します。 この時点では、リスナーまたはリソースをオンラインにしないでください。

1. クリックして、 **リソース** ] タブの順に作成したクライアント アクセス ポイントを展開します。 表示されます、 **IP アドレス** のクラスターのクラスター ネットワークごとのリソースです。 これが Azure 専用ソリューションである場合は、1 つの IP アドレス リソースだけが表示されます。

1. ハイブリッド ソリューションを構成している場合は、この手順を続行します。 Azure 専用ソリューションを構成している場合は、次の手順に進みます。 
     - 内部設置型サブネットに対応する IP アドレス リソースを右クリックし [ **プロパティ**します。 IP アドレス名とネットワーク名をメモします。
     - 選択 **静的 IP アドレス**, 、未使用の IP アドレスを割り当てるし、順にクリックして **OK**します。

1. ご使用の Azure サブネットに対応する IP アドレス リソースを右クリックし、[プロパティ] を選択します。
    >[AZURE.NOTE] 後で、リスナーは、DHCP によって選択された競合する IP アドレスであるためオンラインに失敗した場合、このプロパティ ウィンドウで有効な静的 IP アドレスを構成できます。

1. 同じ **IP アドレス** プロパティ] ウィンドウで変更、 **IP アドレス名**します。 この IP アドレス名は次のために使用される、 **$IPResourceName** 、PowerShell スクリプトの変数です。 ソリューションが複数の Azure VNet にまたがる場合は、IP リソースごとにこの手順を繰り返します。
