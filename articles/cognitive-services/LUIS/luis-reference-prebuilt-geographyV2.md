# <a name="geographyv2-prebuilt-entity-for-a-luis-app"></a>Wstępnie skompilowana jednostka regionu GeographyV2 dla aplikacji usługi LUIS
Wstępnie skompilowana jednostka geografiiV2 wykrywa miejsca. Ponieważ ta jednostka jest już przeszkolony, nie trzeba dodawać wypowiedzi przykład zawierający GeographyV2 do intencji aplikacji. Jednostka GeographyV2 jest obsługiwana w [kulturze angielskiej](luis-reference-prebuilt-entities.md).

## <a name="subtypes"></a>Podtypów
Lokalizacje geograficzne mają podtypy:

|Podtypu|Przeznaczenie|
|--|--|
|`poi`|punkt zainteresowania|
|`city`|nazwa miasta|
|`countryRegion`|nazwa kraju lub regionu|
|`continent`|nazwa kontynentu|
|`state`|nazwa stanu lub prowincji|


## <a name="resolution-for-geographyv2-entity"></a>Rozdzielczość dla jednostki GeographyV2

Następujące obiekty encji są zwracane dla kwerendy:

`Carol is visiting the sphinx in gizah egypt in africa before heading to texas.`

#### <a name="v3-response"></a>[Odpowiedź V3](#tab/V3)

Następujący JSON jest `verbose` z parametrem ustawionym na: `false`

```json
"entities": {
    "geographyV2": [
        {
            "value": "the sphinx",
            "type": "poi"
        },
        {
            "value": "gizah",
            "type": "city"
        },
        {
            "value": "egypt",
            "type": "countryRegion"
        },
        {
            "value": "africa",
            "type": "continent"
        },
        {
            "value": "texas",
            "type": "state"
        }
    ]
}
```

W poprzednim JSON `poi` jest skrótem od **Point of Interest**.
