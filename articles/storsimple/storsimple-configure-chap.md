# StorSimple デバイスの CHAP の構成

このチュートリアルでは、StorSimple デバイスの CHAP を構成する方法について説明します。 CHAP は、チャレンジ ハンドシェイク認証プロトコルを意味します。 これは、サーバーがリモート クライアントの ID を検証する際に使用する認証スキームです。 この検証は、共有パスワードまたはシークレットに基づいて行われます。 

CHAP には、単一方向 (一方向 CHAP) と双方向 (相互 CHAP) とがあります。 一方向 CHAP は、ターゲットがイニシエーターを認証する際に使用されます。 一方、相互 CHAP やリバース CHAP では、ターゲットがイニシエーターを認証した後、イニシエーターがターゲットを認証します。 イニシエーターの認証は、ターゲットの認証なしに実装できますが、 ターゲットの認証は、イニシエーターも実装されている場合にのみ実装できます。

ベスト プラクティスとして、CHAP を使用して iSCSI セキュリティを強化することをお勧めします。

>[AZURE.NOTE] IPSEC が現在は StorSimple デバイスでサポートされないことに留意してください。

StorSimple デバイスの CHAP 設定は、次の方法で構成できます。

- 一方向認証または単一方向認証

- 双方向認証または相互認証、リバース認証

いずれの場合も、デバイスのポータルとサーバーの iSCSI イニシエーター ソフトウェアを構成する必要があります。 この構成の詳細な手順は、次のチュートリアルで説明します。

## 一方向認証または単一方向認証

一方向認証では、ターゲットがイニシエーターを認証します。 この認証では、StorSimple デバイスで CHAP イニシエーター設定を構成し、ホストで iSCSI イニシエーター ソフトウェアを構成することが必要になります。 StorSimple デバイスおよび Windows ホスト用の詳細な手順を次に説明します。

#### 一方向認証用にデバイスを構成するには

1. Azure クラシック ポータルでの **デバイス** ] ページで、をクリックして、 **構成** ] タブをクリックします。

    ![CHAP Initiator](./media/storsimple-configure-chap/IC740943.png)

2. 下へスクロールし、このページで、 **CHAP イニシエーター** セクション。
                                                    
    1. CHAP イニシエーターのユーザー名を指定します。

    2. CHAP イニシエーターのパスワードを指定します。

    > [AZURE.IMPORTANT] CHAP のユーザー名より少ない 233 文字を含める必要があります。 CHAP のパスワードは 12 ～ 16 文字にする必要があります。 ユーザー名やパスワードをそれより長くすると、Windows ホストで認証エラーとなります。

#### Windows ホスト サーバーで一方向認証を構成するには

1. Windows ホスト サーバーで、iSCSI イニシエーターを開始します。

2.  **ISCSI イニシエーターのプロパティ** ウィンドウで、次の手順を実行します。
                                                    
    1. クリックして、 **検出** ] タブをクリックします。

        ![iSCSI イニシエーターのプロパティ](./media/storsimple-configure-chap/IC740944.png)

    2. クリックして **ポータルの探索**します。

3.  **ターゲット ポータルの探索** ] ダイアログ ボックス。
                                                    
    1. デバイスの IP アドレスを指定します。

    3. クリックして **高度な**です。

        ![Discover target portal](./media/storsimple-configure-chap/IC740945.png)

4.  **の詳細設定** ] ダイアログ ボックス。
                                                    
    1. 選択、 **CHAP を有効にするログオンを** チェック ボックスをオンします。

    2.  **名前** フィールドに、クラシック ポータルで CHAP イニシエーターに対して指定したユーザー名を入力します。

    3.  **ターゲット シークレット** フィールドに、クラシック ポータルで CHAP イニシエーターに対して指定したパスワードを入力します。

    4. Click **OK**.

        ![Advanceds settings general](./media/storsimple-configure-chap/IC740946.png)

5.  **ターゲット** のタブ、 **iSCSI イニシエーターのプロパティ** ウィンドウで、デバイスの状態として表示されます **接続**します。

    > [AZURE.IMPORTANT] ISCSI の名前を変更する場合は、新しい名前を使用して新しい iSCSI のセッション。 新しい設定は、ログオフしてもう一度ログオンするまで、既存のセッションに使用されません。

Windows ホスト サーバーで CHAP を構成する方法の詳細についてを参照してください [追加の考慮事項](#additional-considerations)します。


## 双方向認証または相互認証

双方向認証では、ターゲットがイニシエーターを認証して、イニシエーターがターゲットを認証します。 これには、ユーザーがデバイスで CHAP イニシエーター設定とリバース CHAP 設定を構成し、ホストで iSCSI イニシエーター ソフトウェアを構成する必要があります。 デバイスと Windows ホストで相互認証を構成する手順を次に説明します。

#### 相互認証用にデバイスを構成するには

1. Azure クラシック ポータルでの **デバイス** ] ページで、をクリックして、 **構成** ] タブをクリックします。

    ![CHAP target](./media/storsimple-configure-chap/IC740948.png)

2. 下へスクロールし、このページで、 **CHAP ターゲット** セクション。
                                                    
    1. 提供、 **リバース CHAP ユーザー名** デバイス用です。

    2. 指定、 **リバース CHAP パスワード** デバイス用です。

    3. パスワードを確認入力します。

3.  **CHAP イニシエーター** セクション。
                                                
    1. 提供、 **ユーザー名** デバイス用です。

    1. 提供、 **パスワード** デバイス用です。

    3. パスワードを確認入力します。

4. クリックして **保存**します。 確認メッセージが表示されます。 クリックして **OK** 変更を保存します。

#### Windows ホスト サーバーで双方向認証を構成するには

1. Windows ホスト サーバーで、iSCSI イニシエーターを開始します。

2.  **ISCSI イニシエーターのプロパティ** ウィンドウで、をクリックして、 **構成** ] タブをクリックします。

3. クリックして **CHAP**します。

4.  **ISCSI イニシエーター相互 CHAP シークレット** ] ダイアログ ボックス。
                                                    
    1. 型、 **リバース CHAP パスワード** Azure クラシック ポータルで構成します。

    2. Click **OK**.

        ![iSCSI initiator mutual CHAP secret](./media/storsimple-configure-chap/IC740949.png)

5. クリックして、 **ターゲット** ] タブをクリックします。

6. クリックして、 **接続** ] ボタンをクリックします。

7.  **ターゲットへの接続** ダイアログ ボックスで、をクリックして **詳細**します。

8.  **プロパティの詳細** ] ダイアログ ボックス。
                                                    
    1. 選択、 **CHAP を有効にするログオンを** チェック ボックスをオンします。

    2.  **名前** フィールドに、クラシック ポータルで CHAP イニシエーターに対して指定したユーザー名を入力します。

    3.  **ターゲット シークレット** フィールドに、クラシック ポータルで CHAP イニシエーターに対して指定したパスワードを入力します。

    4. 選択、 **相互認証の実行** チェック ボックスをオンします。

        ![Advanceds settings mutual authentication](./media/storsimple-configure-chap/IC740950.png)

    5. をクリックして **OK** CHAP 構成を完了します。

Windows ホスト サーバーで CHAP を構成する方法の詳細についてを参照してください [追加の考慮事項](#additional-considerations)します。

## 追加の考慮事項

 **クイック接続** 機能では、CHAP を有効になっている接続はサポートされません。 CHAP を有効にすると、使用して確認、 **接続** で使用可能なボタン、 **ターゲット** をターゲットに接続] タブをクリックします。

![Connect to target](./media/storsimple-configure-chap/IC740947.png)

 **ターゲットへの接続** ダイアログ ボックスが表示されたら、選択、 **この接続をお気に入りのターゲットの一覧に追加する** チェック ボックスをオンします。 これにより、コンピューターを再起動するたびに、iSCSI のお気に入りのターゲットへの接続の復元が試行されます。

## 構成中のエラー

CHAP の構成が正しくないかどうかは、よく見られますが、 **認証の失敗** エラー メッセージ。

## CHAP の構成の確認

次の手順を実行することで、CHAP が使用されていることを確認します。

#### CHAP の構成を確認するには

1. クリックして **お気に入りのターゲット**します。

2. 認証を有効にしたターゲットを選択します。

3. クリックして **詳細**します。

    ![iSCSI initiator properties favorite targets](./media/storsimple-configure-chap/IC740951.png)

4.  **お気に入りのターゲットの詳細** ] ダイアログ ボックスで、エントリを確認、 **認証** フィールドです。 構成が成功したと **CHAP**します。

    ![Favorite target details](./media/storsimple-configure-chap/IC740952.png)
