# <a name="azure-cognitive-services-support-and-feedback-options"></a>Opcje pomocy technicznej i opinii usługi Azure Cognitive Services

Czy dopiero zaczynasz eksplorować funkcjonalność usług Azure Cognitive Services? Być może implementujesz nową funkcję w aplikacji. Lub po skorzystaniu z usługi, czy masz sugestie, jak ją poprawić? Oto opcje, w których możesz uzyskać pomoc techniczną, być na bieżąco, przekazywać opinie i zgłaszać błędy dla usług Cognitive Services.

## <a name="get-support"></a>Uzyskiwanie pomocy technicznej

### <a name="create-an-azure-support-request"></a>Tworzenie wniosku o pomoc techniczną platformy Azure

<div class='icon is-large'>
    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

Zapoznaj się z zakresem [opcji pomocy technicznej platformy Azure i wybierz plan,](https://azure.microsoft.com/support/plans) który najlepiej pasuje, niezależnie od tego, czy jesteś programistą, który dopiero rozpoczyna podróż w chmurze, czy dużą organizacją wdrażającą aplikacje strategiczne o znaczeniu krytycznym dla firmy. Klienci platformy Azure mogą tworzyć żądania pomocy technicznej i zarządzać nimi w witrynie Azure portal.

* [Azure Portal](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [Portal Azure dla rządu Stanów Zjednoczonych](https://portal.azure.us)

### <a name="search"></a>Wyszukiwanie

<div class='icon is-large'>
    <img alt='Search' src='https://docs.microsoft.com/media/common/i_search.svg'>
</div>

Aby uzyskać szybsze wyniki, należy przeprowadzić wyszukiwanie w odpowiednich witrynach, takich jak [Przepełnienie stosu,](https://stackoverflow.com/) [dokumenty firmy Microsoft](https://docs.microsoft.com/azure/cognitive-services/)lub [przykłady kodu GitHub.](https://github.com/azure-samples) Użyj `site:` notacji zapytania w [ulubionej wyszukiwarce,](https://bing.com)na przykład:

```
{search keywords} site:stackoverflow.com
```

Gdzie `{search keywords}` jest kontekst wyszukiwania. Należy rozważyć użycie tych wyszukiwań o określonym zakresie odpowiednio:

 - Przepełnienie stosu:`site:stackoverflow.com`
 - Dokumenty firmy Microsoft:`site:docs.microsoft.com`
 - Próbki github:`site:github.com/azure-samples`

### <a name="post-a-question-on-stack-overflow"></a>Zamieść pytanie na temat Przepełnienia stosu

<div class='icon is-large'>
    <img alt='Stack Overflow' src='https://docs.microsoft.com/media/logos/logo_stackoverflow.svg'>
</div>

Jeśli nie możesz znaleźć odpowiedzi na problem z polem wyszukiwania w polu Przepełnienie stosu, prześlij nowe pytanie. Przepełnienie stosu jest preferowanym kanałem pytań związanych z rozwojem. To tam członkowie społeczności Stack Overflow i członkowie zespołu firmy Microsoft są bezpośrednio zaangażowani w pomoc w rozwiązywaniu problemów.

 Podczas formułowanie pytania należy użyć jednego z następujących tagów:

 - [Microsoft Cognitive](https://stackoverflow.com/questions/tagged/microsoft-cognitive)
 - [Azure Cognitive Services](https://stackoverflow.com/questions/tagged/azure-cognitive-services)

> [!TIP]
> Poniższe posty z Przepełnienie stosu zawierają wskazówki dotyczące tworzenia pytań i dodawania kodu źródłowego. Przestrzeganie tych wytycznych może zwiększyć szanse, że członkowie społeczności szybko oceniają twoje pytanie i odpowiedzą na nie:
> * [Jak zadać dobre pytanie?](https://stackoverflow.com/help/how-to-ask)
> * [Jak stworzyć minimalny, powtarzalny przykład?](https://stackoverflow.com/help/minimal-reproducible-example)

## <a name="report-bugs"></a>Zgłaszanie błędów

### <a name="create-a-github-issue-or-pull-request"></a>Tworzenie problemu z githubem lub żądanie ściągnięcia

<div class='icon is-large'>
    <img alt='GitHub' src='https://docs.microsoft.com/media/logos/logo_octokitty.svg'>
</div>

Poniżej znajdują się trzy typy repozytoriów, w których deweloper może zaksięgować problem lub utworzyć żądanie ściągnięcia. Aby utworzyć problem, użyj karty *Problemy* w odpowiednim repozytorium Usługi GitHub i wybierz pozycję *Nowy problem*. Jeśli chcesz zaproponować poprawkę, prześlij żądanie ściągnięcia, edytując plik bezpośrednio. Żądanie ściągnięcia będzie następnie siedzieć pod *ściągniętych żądań* kartę, dopóki właściciele repozytorium można przeglądać go.

* [Usługi kognitywne: Dotnet](https://github.com/Azure-Samples/cognitive-services-dotnet-sdk-samples)
