# Azure ゲスト OS リリースと SDK の互換性対応表
Cloud Services 向けの最新の Azure ゲスト OS リリースの最新情報を提供します。 この情報は、ゲスト OS が無効になる前にアップグレード パスを計画する際に役立ちます。

> [AZURE.IMPORTANT] このページは、クラウド サービス web ロールとワーカー ロール、ゲスト OS 上で実行されるに適用されます。 IaaS Virtual Machines には適用されません。 ゲスト OS の更新」の説明に従ってを自動的にロールを構成する場合は、 [Azure ゲスト OS の更新設定][], 、重要でこのページを確認することはありません。

## 最新情報

###### **2015 年 12 月 9 日**
12 月のゲスト OS ロールアウトは、2015 年 12 月 10 日に開始し、2016 年 1 月 8 日にリリースされる予定です。

###### **2015 年 11 月 12 日**  
2014 年 8 月 7 日に、Microsoft は .NET Framework 4、4.5、および 4.5.1 のサポートを 2016 年 1 月 12 日に終了することを発表しました。 お客様および開発者は、テクニカル サポートとセキュリティ更新プログラムを引き続き利用するために、2016 年 1 月 12 日までに .NET Framework 4.5.2 へのインプレース更新を完了することをお勧めします。 詳細については、Microsoft .NET Framework サポート ライフサイクル ポリシーをご覧ください。 

10 月 27 日に、Microsoft は、Azure ゲスト オペレーティング システム (ゲスト OS) ファミリ 2.x、3.x、および 4.x の .NET Framework が、11 月のゲスト OS リリースで .NET Framework 4.5.2 に更新されることを発表しました。 それ以来、.NET 4.5.2 がインストールされた OS リリースへの自動更新を延期し、テストの検証用に .NET 4.5.2 がインストールされたイメージを提供するよう求めるフィードバックがお客様から寄せられました。 

お客様の要件に適切に対応し、.NET 4.5.2 にスムーズにアップグレードできるように、Azure では、Azure ゲスト オペレーティング システム (ゲスト OS) ファミリ 2.x、3.x、および 4.x の .NET Framework が、2016 年 1 月のゲスト OS リリースで .NET Framework 4.5.2 に更新されます。 ゲスト OS ファミリ 2.x、3.x、4.x で実行され、自動更新が有効になっている Cloud Services は、.NET Framework 4.5.2 がインストールされた 2016 年 1 月のゲスト OS に更新されます。 11 月には、既定の OS にインストールされた .NET Framework は変更されません。 お客様が .NET 4.5.2 を使用して Cloud Service を検証できるように、手動デプロイメント用に、.NET 4.5.2 がインストールされた 11 月の OS バージョン 201511-02 の 2 つ目のセットが提供されます。

   これらには、11 月の MSRC と 10 月の Windows ロールアップが含まれます。

アップグレードする前に、.NET 4.5.2 での Cloud Service の検証に役立つオプションがいくつかあります。 

1.  テスト用の Cloud Service ロールを 201511-02 リリースにデプロイし、アプリケーションの互換性の検証を行います。 

2.  テスト用の Cloud Service ロールに .NET 4.5.2 をインストールし、アプリケーションの互換性の検証を行います。 参照してください [Install .NET on a Cloud Service Role] についてです。 

3.  オンプレミスのスタンドアロン VM または .NET 4.5.2 がインストールされた Azure VM で、アプリケーションの互換性の検証を行います。

## ゲスト OS のリリース情報

このセクションでは、現在サポートされているゲスト OS バージョンを示します。 ゲスト OS ファミリとバージョンには、リリース日、無効になる日、有効期限があります。 リリース日の時点で、ゲスト OS バージョンは、Azure クラシック ポータルで手動で選択できるようになります。 ゲスト OS は、"無効になる" 日付以降に Azure クラシック ポータルから削除されます。 その後は "移行中" になりますが、デプロイを更新できるように限定的にサポートされます。 有効期限とは、バージョンまたはファミリが Azure システムから完全に削除されるようスケジュールされている日付です。 削除すると、バージョンで動作している、期限切れになる場合は、停止されるクラウド サービスまたはで詳述するよう、新しいバージョンに強制的にアップグレード、 [Azure ゲスト OS のサポートと提供終了のポリシー][retirepolicy]します。 

マイクロソフトは、サポートされている各ゲスト OS ファミリについて少なくとも 2 つの最新バージョンをサポートしています。 既存のゲスト OS バージョンが無効になる日は、デプロイできるリリース バージョンが少なくとも 2 つ残るように延期することができます。

> [AZURE.WARNING] ゲスト OS ファミリ 1 の提供中止は 2013 年 6 月 1 日に開始され、すぐに終了する予定です。 このゲスト OS ファミリを使用した新規インストールの作成や古いインストールのアップグレードは行わないでください。 詳細については、次を参照してください [Azure ゲスト OS ファミリ 1 の提供終了情報。][fam1retire]


### ゲスト OS ファミリ、バージョン、およびリリースの説明
ゲスト OS ファミリは Microsoft Windows Server のリリース バージョンに基づいています。 ゲスト OS は、Azure Cloud Services が実行される、基になるオペレーティング システムです。 各ゲスト OS には、ファミリ、バージョン、リリース番号があります。 

 **ゲスト OS ファミリ** ゲスト OS の基にいる Windows Server オペレーティング システム リリースに対応しています。 たとえば、ファミリ 3 は Windows Server 2012 に基づいています。 

A **「ゲスト OS バージョン」** ファミリの OS イメージは、関連すると [Microsoft セキュリティ レスポンス センター (MSRC)][msrc] 修正プログラムの新しいゲスト OS バージョンが作成された時点で利用可能です。 すべての修正プログラムが含まれているわけではありません。 番号は 0 から始まり、新しい更新プログラムのセットが追加されるたびに 1 ずつ増えます。 末尾のゼロが表示されるのは、重要な場合のみです。 つまり、バージョン 2.10 は、バージョン 2.1 よりかなり後の異なるバージョンです。 

A **「ゲスト OS リリース」** ゲスト OS バージョンが再リリースを参照します。 再リリースは、変更を必要とする問題がテスト中に見つかった場合に行われます。 最新のリリースは、公開されているかどうかに関係なく、常に以前のリリースよりも優先されます。 Azure クラシック ポータルでは、ユーザーは指定したバージョンの最新のリリースのみを取得できます。 前のリリースで実行中のデプロイは、通常、バグの重大度に応じてアップグレードが強制されることはありません。 

次の例では、2 がファミリ、12 がバージョン、"rel2" がリリースです。

**ゲスト OS リリース** - 2.12 rel2

**このリリースの構成文字列** -WA-ゲスト-OS-2.12_201208-02

ゲスト OS の構成文字列には、組み込まれているのと同じ情報に加えて、そのリリースで考慮された MSRC 修正プログラムを示す日付が含まれます。 この例では、2012 年 8 月までに Windows Server 2008 R2 用に作成された MSRC 修正プログラムが含まれることが想定されます。 そのバージョンの Windows Server に特に適用される修正プログラムのみが含まれます。 たとえば、Microsoft Office に適用される MSRC 修正プログラムは、製品が Windows Server 基本イメージに属さないため、ここに含まれません。

## ゲスト OS システムの更新プロセス
このページには、今後予定されているゲスト OS のリリースに関する情報が含まれます。 "自動" 更新に設定している場合はクラウド サービスのロールが再起動されるため、リリースのタイミングを把握する必要があるとお客様から指摘を受けています。 ゲスト OS のリリースは、通常、毎月第 2 火曜日に発生する MSRC 更新プログラムのリリースから少なくとも 5 日以内に発生します。 新しいリリースには、ゲスト OS ファミリごとに関連するすべての MSRC 修正プログラムが含まれます。 

Microsoft Azure では、常に更新プログラムがリリースされています。 ゲスト OS は、パイプラインにおいて唯一のこのような更新です。 リリースは、ここに記載できないほどの多く要因の影響を受ける可能性があります。 さらに、Azure は、実際には数十万台のコンピューターで実行されています。 つまり、ロールが再起動される正確な日時を指定することはできません。 お知らせ、 [ゲスト OS 更新 RSS フィード][rss] 、最新の情報されていますが、その時点のおおよそのウィンドウを検討してください。 これは問題視されているため、再起動の制限や時間の指定をできるように取り組んでいます。 

ゲスト OS の新しいリリースが公開されると、Azure 全体で完全に反映されるまでに時間がかかります。 サービスが新しいゲスト OS に更新されると、更新ドメインに従って再起動されます。 "自動" 更新プログラムを使用するように設定されているサービスで、まずリリースが取得されます。 更新後、Azure クラシック ポータルでは、ご使用のサービスの新しいゲスト OS バージョンの一覧が表示されます。 この期間に再リリースが発生する場合があります。 一部のバージョンは長期間にわたってデプロイされることがあります。また、正式なリリース日以降何週間も自動アップグレードの再起動が行われないこともあります。 ゲスト OS が使用可能になると、ポータルや構成ファイルから明示的にそのバージョンを選択できます。 

再起動し、技術的な詳細のゲストとホスト OS の更新へのポインターに関する貴重な情報の多くは、MSDN のブログの記事を参照を参照してください。 [によるロール インスタンスが再起動 OS のアップグレード][restarts]します。

ゲスト OS を手動で更新する場合は、読み取り、 [ゲスト OS の提供終了のポリシー][retirepolicy]します。

###### **2014 年 11 月 11 日**

###### **2014 年 11 月 4 日**
10 月のリリース (4.13、3.20、2.32) は、2014 年 11 月 4 日にロールアウトされました。 これには、8 月と 9 月のリリースで問題の原因となった MSRC 修正プログラムが含まれます。 この問題を回避するために、10 月のリリースにはプレインストールされた .NET 3.5 および 3.5.1 が含まれていますが、無効になっています。 .NET 3.5 または 3.5.1 のインストールを試みるスクリプトは、実際にこれをもう一度有効にして、.NET インストールの "成功" を返します。ただし、MSRC 修正プログラムによって作成されたフル インストールの問題も回避します。 

**2014 年 10 月 20 日。2014 年 11 月 4 日を更新** -年 9 月リリース (4.12、3.19、2.31、および 1.39) が同じにより部分的にロールアウト [MSRC 修正プログラム MS14 046][MS14-046] ファミリ 3 または 4 に .NET 3.5 または 3.5.1 をインストールしようとしています。 これらのエラーの原因です。 .NET 3.5.x はどちらのファミリで正式にサポートされていませんが、一部のお客様のインストールではこれを利用し、変更が発表されたなかったために、Microsoft が動作の変更に応答しています。 前のゲスト OS が無効になる日 (6 月と 7 月) は、完全にリリースされたゲスト OS が少なくとも 2 つサポートされて使用できるように、それぞれ延期されます。 .NET インストールの問題の解決策は、2014 年 10 月のリリースで発表されました。

この 10 月のリリースには、プレインストールされている (ただし無効になっている) .NET 3.5 および 3.5.1 が含まれています。さらに、前に記載されている MSRC 修正プログラムも含まれています。 .NET 3.5 または 3.5.1 のインストールを試みるスクリプトは、実際にこれをもう一度有効にして、.NET インストールの "成功" を返します。ただし、MSRC 修正プログラムによって作成されたインストールの問題を回避もします。
