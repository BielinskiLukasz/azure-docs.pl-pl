# Azure Privileged Identity Management: セキュリティ レビューの実行方法

## セキュリティ レビューの実行
1 回のアクセス権限を確認するのには非常に簡単である、 [セキュリティの確認が開始された](active-directory-privileged-identity-management-how-to-start-security-review.md)します。

## レビュー担当者の場合: アクセスの許可または拒否

### 自分がレビューする
1. PIM のメイン メニューでクリックして **管理アクセス権を確認**します。 セキュリティ レビューの一覧が表示されます。
2. 選択、 **ユーザー** 順番でアクセスを変更します。 注: アクセスが実際に変更されます。  このプロセスは単に、ロールのアクセスを変更するユーザーのチェックリストを作成します。  少なくとも 1 人のユーザーが選択されて、 **のアクセスを承認** と **アクセスを拒否する** ボタンが有効になります。
3. いずれかをクリックして  **のアクセスを承認** または **アクセスを拒否する** 選択したユーザーにします。  通知がポータルのメイン メニューに表示され、レビューの一覧が表示されなくなります。  閉じる、 **レビュー Azure AD ロール** ブレードです。

### 自己レビュー
1. ユーザーは、自分のアクセスをレビューする必要があることを示す電子メールを受け取ります。  電子メールには、ポータルにログインするためのリンクが含まれます。
2. ユーザーを承認できますが、いったん] をクリックして、独自のアクセスを拒否するか、  **のアクセスを承認** または **アクセスを拒否する** ボタン。  名前が一覧に表示されなくなります。

## レビュー マネージャーの場合: セキュリティ レビューの管理

## レビューの完了または停止
1. PIM ダッシュボードに戻ります。
2. [で完了する場合、セキュリティの確認] をクリックして、 **セキュリティ レビュー** ] ボックスの一覧です。 セキュリティ レビューの詳細ブレードが表示されます。
3. をクリックして **レビューを停止** を完了するか、レビューを停止します。  レビューが保存されて、ブレードが表示されなくなります。

## レビューのエクスポート
Excel または CSV ファイルを使用できる他のプログラムで使用するために、レビューをエクスポートできます。

1. PIM ダッシュボードに戻ります。
2. をクリックして、 **セキュリティ レビュー** ダッシュ ボードのセクションです。   **セキュリティ レビュー** ブレードが表示されます。
3. エクスポートするセキュリティ レビューをクリックします。 セキュリティ レビューの詳細ブレードが表示されます。
4. クリックして、 **エクスポート** ] ボタンをクリックします。 CSV ファイルのダウンロードが開始します。

## レビューの削除

> AZURE です。警告 [がない削除が行われる前に警告が表示されたら、するようにしてした *します* そのレビューを実際に削除する]。

2. をクリックして、 **セキュリティ レビュー** ダッシュ ボードのセクションです。   **セキュリティ レビュー** ブレードが表示されます。
3. 削除するセキュリティ レビューをクリックします。 セキュリティ レビューの詳細ブレードが表示されます。
4. クリックして、 **削除** ] ボタンをクリックします。
