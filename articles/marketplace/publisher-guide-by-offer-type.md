# <a name="publishing-guide-by-offer-type"></a>Podręcznik publikowania dla różnych typów ofert

Po [wybraniu opcji publikowania](https://docs.microsoft.com/azure/marketplace/determine-your-listing-type)możesz wybrać typ oferty, który będzie używany do prezentacji oferty.

 *Typ oferty* definiuje strukturę oferty, która obejmuje metadane, artefakty i inną zawartość używaną do prezentowania oferty w portalu marketplace.

Przed utworzeniem oferty należy wybrać typ oferty. Typ oferty będzie odpowiadał typowi rozwiązania, aplikacji lub oferty usługi, które chcesz opublikować, a także jego wyrównaniu do produktów i usług firmy Microsoft.

Jeden typ oferty można skonfigurować inaczej, aby włączyć różne opcje publikowania, wezwania do działania, inicjowania obsługi administracyjnej lub cen. Opcja publikowania i konfiguracja typu oferty są również zgodne z wymaganiami dotyczącymi uprawnień i wymagań technicznych oferty.

Przed utworzeniem oferty należy zapoznać się z wymaganiami dotyczącymi uprawnień do witryny sklepu i typem oferty oraz wymaganiami dotyczącymi publikowania technicznego.

## <a name="list-of-offer-types"></a>Lista typów ofert

Typy ofert w portalu Azure Marketplace są wymienione w poniższej tabeli.

| **Typ oferty**    | **Opis**  |
| :------------------- | :-------------------|
| [**Maszyny wirtualne**](./marketplace-virtual-machines.md) | Użyj typu oferty maszyny wirtualnej podczas wdrażania urządzenia wirtualnego w subskrypcji skojarzonej z klientem. |
| [**Szablony rozwiązań**](./marketplace-solution-templates.md) | Użyj szablonu rozwiązania (znanego również jako aplikacja platformy Azure), gdy rozwiązanie wymaga dodatkowego wdrożenia i automatyzacji konfiguracji poza jedną maszyną wirtualną. Szablony rozwiązań mogą egozatruwać wiele różnych rodzajów zasobów platformy Azure, w tym między innymi maszyny wirtualne.  |
| [**Aplikacje zarządzane**](./marketplace-managed-apps.md) | Użyj aplikacji platformy Azure: typ oferty aplikacji zarządzanej, gdy wymagane są następujące warunki: <br> <ul> <li> Można wdrożyć rozwiązanie oparte na subskrypcji dla klienta przy użyciu maszyny Wirtualnej lub całego rozwiązania opartego na IaaS. </li> <li>Ty lub Twój klient wymagasz, aby rozwiązanie było zarządzane przez partnera. </li> <ul> |
| [**Aplikacje SaaS**](./marketplace-saas-applications-technical-publishing-guide.md) | Użyj typu oferty aplikacji SaaS, aby umożliwić klientowi zakup rozwiązania technicznego opartego na SaaS jako subskrypcji. |
| [**Oferty kontenerów**](./marketplace-containers.md) | Użyj typu oferty kontenera, gdy rozwiązanie jest obraz kontenera platformy Docker aprowizowana jako usługa kontenera platformy Azure oparta na usłudze Platformy Azure opartej na usłudze Kubernetes. |
| [**Moduły usługi Azure IoT Edge**](./iot-edge-module.md) | Moduły usługi Azure IoT Edge są najmniejszymi jednostkami obliczeniowymi zarządzanymi przez usługę IoT Edge i mogą zawierać usługi firmy Microsoft (takie jak usługa Azure Stream Analytics), usługi innych firm lub własny kod specyficzny dla rozwiązania. |
| [**Usługi konsultingowe**](./consulting-services.md) | Usługi konsultingowe pomagają łączyć klientów z usługami w celu obsługi i rozszerzania korzystania z usług platformy Azure, Dynamics 365 lub Power Suite.|
| [**Usługi Office 365, Dynamics 365 i Power BI**](./appsource-offer-publishing-guide.md) | Można publikować oferty appsource, które opierają się na dynamics 365, Office 365, usłudze Power BI i power apps.|
| [**Zintegrowane rozwiązania**](./integrated-solutions-for-publishers.md) | Można publikować zintegrowane, dostosowane do branży rozwiązania, które łączą technologię i usługi jako jedną ofertę.|

Aby uzyskać informacje na temat wymagań dotyczących logowania jednokrotnego, wyświetlając listę opcji i typów ofert, zobacz [**Wymagania usługi Azure AD**](./enable-appsource-marketplace-using-azure-ad.md).
