<a name="seller-insights-definitions"></a>Definicje statystyk sprzedawcy
=======================

Poniższa tabela zawiera definicje wielu terminów używanych w Statystykach sprzedawcy.

|  **Termin**                 |  **Definicja**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Typ licencji platformy Azure**                               | Typ umowy licencyjnej używanej przez klientów do zakupu platformy Azure.  Znany również jako Kanał.                                                  |
| **Typ licencji platformy Azure: dostawca rozwiązania w chmurze**      | Klient końcowy nabywa platformę Azure i ofertę marketplace za pośrednictwem dostawcy rozwiązań w chmurze, który działa jako twój sprzedawca.                 |
| **Typ licencji platformy Azure: przedsiębiorstwo**                   | Klient końcowy nabywa platformę Azure i ofertę marketplace za pośrednictwem umowy Enterprise Agreement podpisanej bezpośrednio z firmą Microsoft.                  |
| **Typ licencji platformy Azure: przedsiębiorstwo za pośrednictwem sprzedawcy**  | Klient końcowy nabywa platformę Azure i ofertę marketplace za pośrednictwem sprzedawcy, który ułatwia umowę Enterprise Agreement z firmą Microsoft.     |
| **Typ licencji platformy Azure: Płać zgodnie z oczekiwaniami**                | Klient końcowy nabywa platformę Azure i ofertę marketplace za pośrednictwem umowy "Zapłać zgodnie z tobą", podpisanej bezpośrednio z firmą Microsoft.                |
| **Kwota obciążenia (CC)**                              | Kwota pobrana od odbiorcy w *walucie odbiorcy* (CC) za rozliczenia.                                                                 |
| **Kwota obciążenia (PC)**                               | Kwota pobrana od odbiorcy w *walucie wypłaty* (PC).                                                                      |
| **Data obciążenia**                                      | Data obliczenia opłaty od klienta (zazwyczaj bezpośrednio po okresie użytkowania).                                             |
| **Nazwa wystąpienia w chmurze**                              | Usługa Microsoft Cloud, w której wystąpiło wdrożenie maszyny Wirtualnej.                                   |
| **Nazwa wystąpienia w chmurze: Usługa Azure Global**                | Publiczna globalna chmura Microsoft.                           |
| **Nazwa wystąpienia w chmurze: usługa Azure Government**        | Specyficzne dla rządu chmury firmy Microsoft dla jednego z następujących rządów: Chiny, Niemcy lub Stany Zjednoczone Ameryki.                           |
| **Stan kolekcji**  | W przypadku określonej opłaty najnowszy stan cyklu rozliczeniowego i okresowego pobierania opłat.  Na przykład stan może być w toku, zbieranie lub zwrot pieniędzy.                  |
| **Rozmiar rdzenia**  | Liczba rdzeni wirtualnych prezentowanych przez maszynę wirtualną, na podstawie na która jest uruchomiona oferta.               |
| **Miasto klienta**  | Nazwa miasta podana przez klienta. Może to być inna niż miasto w subskrypcji platformy Azure klienta.  |
| **Język komunikacji z klientem**        | Język preferowany przez klienta do komunikacji.                                  |
| **Nazwa firmy klienta**                  | Nazwa firmy podana przez klienta. Może to być inna niż miasto w subskrypcji platformy Azure klienta.                                  |
| **Kraj klienta**                       | Nazwa kraju/regionu podana przez klienta.  Może to być inny niż kraj/region w subskrypcji platformy Azure klienta.                               |
| **Waluta odbiorcy (CC)**                 | Waluta preferowana przez odbiorcę do wyceny i rozliczeń.                            |
| **Adres e-mail klienta**                         | Adres e-mail podany przez klienta końcowego.  Może to być inny niż adres e-mail w subskrypcji platformy Azure klienta.                   |
| **Imię klienta**                              | Imię podane przez klienta.  Może to być inna niż nazwa podana w subskrypcji platformy Azure klienta.                    |
| **Nazwisko klienta**                               | Nazwisko podane przez klienta.  Może to być inna niż nazwa podana w subskrypcji platformy Azure klienta.                    |
| **Identyfikator klienta**                               | Unikatowy identyfikator przypisany do odbiorcy.  Klient może mieć zero lub więcej subskrypcji portalu Azure Marketplace.                    |
| **Typ płatności odbiorcy**                            | Rodzaj instrumentu płatniczego używanego przez klienta.  Na przykład może to być karta, faktura lub czek elektroniczny.                             |
| **Kod pocztowy odbiorcy**                             | Kod pocztowy podany przez klienta.  Może to być inny niż kod pocztowy podany w subskrypcji platformy Azure klienta.                                                                 |
| **Stan klienta**                                   | Stan (adres) podany przez klienta.  Może to być inna niż nazwisko podane w subskrypcji platformy Azure klienta.                                                                 |
| **Data nabycia**                                    | W pierwszym dniu klient zakupił dowolną ofertę opublikowaną przez Ciebie.                 |
| **Data utraty**                                        | Ostatnia data anulowania przez odbiorcę ostatniej ze wszystkich wcześniej zakupionych ofert.                                                         |
| **Szacowana opłata rozszerzona (CC)**                   | Szacowana opłata rozszerzona za ilość jednostek użycia dla danej jednostki SKU (w walucie odbiorcy). Ta wartość może nie być dokładna z powodu błędów zaokrąglania lub obcinania.                            |
| **Szacowane obciążenie rozszerzone (PC)**                   | Szacowana opłata rozszerzona za ilość jednostek użycia dla danej jednostki SKU, na podstawie konwersji kursowej w dniu użycia, jest obliczana (w walucie wydawcy). Ta wartość może nie być dokładna z powodu błędów zaokrąglania lub obcinania.                  |
| **Szacowana wypłata (PC)**                            | Szacowana płatność za ilość jednostek użycia dla danej jednostki SKU, na podstawie konwersji kursowej w dniu obliczania użycia (w walucie wydawcy). Ta wartość może nie być dokładna z powodu błędów zaokrąglania lub obcinania.                        |
| **Szacowana cena (PC)**                             | Szacowana cena jednostki użycia dla danej jednostki SKU, na podstawie konwersji kursowej w dniu użycia jest obliczana (w walucie wydawcy). Ta wartość może nie być dokładna z powodu błędów zaokrąglania lub obcinania.                                             |
| **Jest nowym klientem**                                  | Wartość poinformuje Cię, czy jest to nowy klient zużywający lub nabywający jedną lub więcej ofert w Marketplace po raz pierwszy (lub nie).  Wartość będzie "Tak", jeśli jest to ten sam miesiąc kalendarzowy dla "Data nabycia".  Wartość będzie wynosić "Nie", jeśli klient zakupił dowolną ofertę przed zgłoszonym miesiącem kalendarzowym.                                                       |
| **Czy podgląd SKU**                                  | Wartość poinformuje Cię, jeśli oznaczyłeś jednostkę SKU jako "podgląd". Wartość będzie "Tak", jeśli jednostka SKU została odpowiednio oznaczona i tylko subskrypcje platformy Azure autoryzowane przez Ciebie można wdrożyć i używać tego obrazu.  Wartość będzie "Nie", jeśli jednostka SKU nie została zidentyfikowana jako "podgląd".                                          |
| **Czy kontakt promocyjny zgłoć**                   | Wartość poinformuje Cię, jeśli klient aktywnie zdecydował się na kontakt promocyjny od wydawców. W tej chwili nie przedstawiamy opcji klientom, więc wskazaliśmy "Nie" we wszystkich dziedzinach. Po wdrożeniu tej funkcji rozpoczniemy odpowiednie aktualizowanie.                                          |
| **Typ licencji marketplace**                         | Metoda rozliczeń oferty Marketplace.                             |
| **Typ licencji portalu Marketplace: rozliczane za pośrednictwem platformy Azure**   | Firma Microsoft jest twoim agentem dla tej oferty Marketplace i rozlicza klientów w Twoim imieniu. (Albo karta kredytowa PAYG, albo faktura enterprise)       |
| **Typ licencji marketplace: Przynieś własną licencję** | Maszyna wirtualna wymaga klucza licencyjnego dostarczonego przez klienta do wdrożenia. Firma Microsoft nie rozlicza klientów za korzystanie z tych ofert w portalu Marketplace. |
| **Typ licencji Marketplace: Bezpłatny**                   | Oferta Marketplace jest skonfigurowana tak, aby była bezpłatna dla wszystkich użytkowników. Firma Microsoft nie rozlicza klientów za korzystanie z tej oferty w marketplace.    |
| **Typ licencji marketplace: Microsoft jako sprzedawca**  | Firma Microsoft jest sprzedawcą tej oferty w marketplace.                               |
| **Nazwa firmy subskrypcyjnej marketplace**  | Nazwa firmy podana przez klienta w ramach subskrypcji platformy Azure.                           |
| **Identyfikator subskrypcji w portalu Marketplace**            | Unikatowy identyfikator skojarzony z subskrypcją platformy Azure, którego klient użył do zakupu oferty portalu Marketplace.  Dawniej był to identyfikator GUID subskrypcji platformy Azure.  W niektórych przypadkach, takich jak zamówienie SAAS, nie jest wymagana subskrypcja.           |
| **Opłata firmy Microsoft (CC)**                               | Opłata firmy Microsoft od transakcji w walucie klienta.                              |
| **Nazwa oferty**                                | Nazwa oferty Marketplace.                                                             |
| **Typ oferty**                                | Typ oferty microsoft marketplace.  Na przykład może to być aplikacja zarządzana, licencje maszyn wirtualnych lub kontener.                           |
| **Data anulowania zamówienia**                         | Data anulowania zamówienia w Marketplace.                                                       |
| **Identyfikator zamówienia**                                  | Unikatowy identyfikator zamówienia klienta dla usługi Marketplace.  Oferty oparte na użyciu maszyny wirtualnej nie są skojarzone z zamówieniem.                 |
| **Data zakupu zamówienia**                       | Data utworzenia zamówienia w marketplace.                                                        |
| **Zamówiona ilość**                       | Ilość zamówień dla tej samej jednostki SKU skojarzonej z subskrypcją platformy Azure.  W przypadku zamówień SAAS reprezentuje liczbę miejsc skojarzonych z tym wystąpieniem zamówienia.                                            |
| **Stan zamówienia**                              | Stan zamówienia w marketplace w momencie ostatniego odświeżania danych.  Na przykład zamówienie może być aktywne lub anulowane.             |
| **Stan zamówienia: Aktywny**                           | Klient zakupił zamówienie i nie anulował swojego zamówienia.   |
| **Stan zamówienia: Anulowano**                        | Klient wcześniej zakupił zamówienie, a następnie anulował swoje zamówienie.        |
| **Kwota wypłaty (PC)**                             | Kwota wypłacona w preferowanej walucie wypłaty (PC).                                       |
| **Waluta wypłaty (PC)**                           | Waluta używana do wypłat.                                                                |
| **Data wypłaty**                                    | Data wysłania żądania płatności od firmy Microsoft do wybranej instytucji finansowej.                                                                         |
| **Status wypłaty**                                  | Stan wypłaty w momencie ostatniego odświeżenia danych.                                   |
| **Status wypłaty: W toku**               | Transakcja nie jest gotowa do wypłaty. (Aby uzyskać więcej informacji, zobacz Stan zbierania)                             |
| **Status wypłaty: Wypłacone**                          | Transakcja została uwzględniona w poprzednim obliczeniu wypłaty. Wartości dodatnie są wypłacane, a wartości ujemne są kompensowane z całkowitą kwotą wymaganą. |
| **Status wypłaty: Nadchodząca wypłata**                   | Transakcja jest gotowa do wypłaty i zostanie uwzględniona w następnym dostępnym obliczeniu wypłaty.                      |
| **Cena (CC)**                                   | Cena jednostki użycia dla danej jednostki SKU (w walucie odbiorcy).                                                                  |
| **Poczta e-mail dostawcy**                                 | Adres e-mail dostawcy zaangażowanego w relację między firmą Microsoft a klientem końcowym. Jeśli klient jest przedsiębiorstwem za pośrednictwem sprzedawcy, będzie on sprzedawcą.  Jeśli dostawca rozwiązań w chmurze (CSP) jest zaangażowany, będzie to dostawca usług kryptograficznych.                                                                 |
| **Nazwa dostawcy**                                 | Nazwa dostawcy zaangażowanego w relację między firmą Microsoft a klientem końcowym. Jeśli klient jest przedsiębiorstwem za pośrednictwem sprzedawcy, będzie on sprzedawcą.  Jeśli dostawca rozwiązań w chmurze (CSP) jest zaangażowany, będzie to dostawca usług kryptograficznych.                                                                 |
| **Identyfikator URI zasobu**                                  | Ścieżka, w której wdrożono artefakt wdrożenia.                                            |
| **Numer jednostki magazynowej**                                           | Nazwa jednostki SKU zdefiniowana podczas publikowania. Oferta może mieć wiele jednostek SKU, ale jednostka SKU może być skojarzona tylko z jedną ofertą.              |
| **Typ rozliczeń jednostki SKU**                                 | Metoda rozliczeń jednostki SKU.                                                                                                             |
| **Typ rozliczeń jednostki SKU: Przynieś własną licencję**         | Maszyna wirtualna lub usługa wymaga klucza licencyjnego dostarczonego przez klienta do wdrożenia. Firma Microsoft nie rozlicza klientów za korzystanie z tych ofert w marketplace.   |
| **Typ rozliczeń usługi SKU: Microsoft jako sprzedawca**          | Firma Microsoft jest sprzedawcą tej jednostki SKU.   |
| **Typ rozliczenia jednostki SKU: bezpłatny**                           | Jednostka SKU jest skonfigurowana tak, aby była bezpłatna dla wszystkich użytkowników. Firma Microsoft nie rozlicza klientów za korzystanie z tej jednostki SKU.                           |
| **Typ rozliczenia jednostki SKU: Płatny**                           | Firma Microsoft jest twoim agentem dla tej jednostki SKU i rozlicza klientów w Twoim imieniu. (Albo karta kredytowa PAYG, albo faktura enterprise)                   |
| **Typ rozliczeń jednostki SKU: wersja próbna**                          | Klient jest w okresie próbnym i zostanie przekonwertowany na zapłacony, jeśli nie anuluje ani nie usunie.                                         |
| **Kwota podatku (CC)**                                  | Kwota podatku zastosowana do rachunku odbiorcy w walucie odbiorcy (DW).                                                               |
| **Data transakcji**                                 | Data transakcji zarejestrowana w raporcie wypłat.                                                                           |
| **Typ transakcji**                               | Typ transakcji, która jest zgłaszana.  Może to być na przykład korekta opłaty, zwrotu lub wypłaty.                                                                                        |
| **Data zakończenia okresu próbnego**                                 | Data zakończenia lub zakończenia okresu próbnego dla tego zamówienia.                                                                           |
| **Data wykorzystania**                                       | Data użycia przez klienta.                                                                                                             |
| **Ilość użycia**                                       | Zgłoszone użycie przez klienta jednostki SKU. W przypadku obrazów maszyn wirtualnych rekordy użycia reprezentują użycie dla zgłoszonego okresu dla tego rozmiaru maszyny Wirtualnej i jednostki SKU.                                               |
| **Odwołanie do użycia**                                       | Identyfikator jednego lub więcej dni użycia klienta dla danej jednostki SKU skojarzonej z zapisem w raporcie wypłat.                                               |
| **Typ użycia**                                       | Opis mierzonego użycia. (Znormalizowane użycie lub użycie pierwotne)                                                              |
| **Typ użycia: Znormalizowany zwyczaj**                | Godziny użycia znormalizowane w celu uwzględnienia liczby rdzeni maszyn wirtualnych zaangażowanych w użycie: [Liczba rdzeni maszyn wirtualnych] x [Godziny użycia surowca]. Maszyny wirtualne oznaczone jako "SHAREDCORE" używają 1/6 (lub 0,1666) jako mnożnika [Liczba rdzeni maszyn wirtualnych].                                        |
| **Typ użycia: Użycie nieprzetworzone**                        | Godziny użytkowania oferty w Marketplace.  Jest on przedstawiony w kategoriach "kalendarzowych" godzin.  Jest to liczba maszyn wirtualnych czasu zostały uruchomione.                           |
| **Jednostki użytkowania**                                   | Jednostka miary dla podanego użycia. Maszyny wirtualne są zawsze mierzone za pomocą godzinowych jednostek miary.                               |
| **Rozmiar maszyny Wirtualnej**                                       | Reprezentuje rozmiar sprzętu maszyny wirtualnej wyrównany z ofertą platformy Azure. Przykłady `Basic_A0`obejmują `Standard_A11` `Standard_D12`, `Standard_G4`, i .   |
|  |  |
