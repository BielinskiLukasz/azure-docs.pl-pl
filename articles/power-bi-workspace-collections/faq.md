# <a name="power-bi-workspace-collections-faq"></a>Kolekcji usługi Power BI w obszarze roboczym — często zadawane pytania

> [!IMPORTANT]
> Kolekcje obszarów roboczych usługi Power BI są przestarzałe i będą dostępne do czerwca 2018 roku lub do daty podanej w kontrakcie. Zachęcamy do zaplanowania migracji do usługi Power BI Embedded, aby uniknąć przerw w działaniu aplikacji. Aby uzyskać informacje dotyczące sposobu przeprowadzenia migracji danych do usługi Power BI Embedded, zobacz [How to migrate Power BI Workspace Collections content to Power BI Embedded (Migrowanie zawartości kolekcji obszarów roboczych usługi Power BI do usługi Power BI Embedded)](https://powerbi.microsoft.com/documentation/powerbi-developer-migrate-from-powerbi-embedded/).

## <a name="what-is-microsoft-power-bi-workspace-collections"></a>Co to jest obszar roboczy kolekcji usługi Microsoft Power BI?

Kolekcje obszarów roboczych usługi Power BI jest usługą platformy Azure, która umożliwia deweloperom osadzanie atrakcyjnych, w pełni interaktywne raporty i wizualizacje w aplikacjach, nie trzeba poświęcać czasu ani pieniędzy na tworzenie odpowiednich mechanizmów od podstaw przeznaczonych dla klientów. W efekcie powstał obszar roboczy kolekcji usługi Power BI dostępnych w ramach umowy SLA w centrach danych 9 na całym świecie. Ulepszyliśmy także funkcje usługi takie jak obsługa dla bezpieczeństwa danych przez zabezpieczenia na poziomie wiersza (RLS) w kolekcji obszarów roboczych usługi Power BI dla zaawansowanego filtrowania. Firma Microsoft ma również Uprościliśmy i Zaktualizowaliśmy kolekcji obszarów roboczych usługi Power BI, model cen.

## <a name="who-would-want-to-use-microsoft-power-bi-workspace-collections-and-why"></a>Kto może okazać się użycie kolekcji obszarów roboczych aplikacji Microsoft Power BI i dlaczego?

Obszar roboczy kolekcji usługi Microsoft Power BI są przeznaczone dla deweloperów aplikacji, którzy chcą oferować olśniewające, interaktywne środowiska wizualizacji danych dla swoich użytkowników na dowolnym urządzeniu, bez konieczności tworzenia samodzielnie. Za pomocą kolekcji obszarów roboczych usługi Power BI deweloperzy mogą dostarczać widoków zawsze w górę — od początku przy użyciu zapytania bezpośredniego. Deweloperzy mogą również programowe wdrażanie i zarządzanie nimi zautomatyzować usługi Power BI przy użyciu interfejsów API usługi Azure Resource Manager i interfejsów API usługi Power BI. Podobnie jak w przypadku wszystkich elementów usługi Power BI embedded usługi jest automatycznie skalowany zaspokoić potrzeby dotyczące użycia i wymagań aplikacji. Usługa kolekcje obszarów roboczych usługi Power BI oferuje model cenowy na podstawie użycia zgodnie z rzeczywistym użyciem.

## <a name="how-does-power-bi-workspace-collections-relate-to-the-power-bi-service"></a>Jaki jest związek między kolekcji obszarów roboczych usługi Power BI w usłudze Power BI?

Kolekcje obszarów roboczych usługi Power BI i usługa Power BI to osobne oferty. Funkcje kolekcji obszarów roboczych usługi Power BI zasilania na podstawie użycia model rozliczeń, jest wdrażany za pomocą witryny Azure portal i zaprojektowana w celu umożliwienia niezależnym dostawcom oprogramowania i osadzaj wizualizacje danych w aplikacjach dla swoich klientów do użycia. Usługa Power BI jest rozliczana i wdrażana za pośrednictwem Centrum administracyjnego usługi Microsoft 365, i jest to oferta analizy Biznesowej ogólnego zastosowania autonomiczny przeznaczony głównie dla użytku wewnętrznego przedsiębiorstw. Aby dowiedzieć się więcej na temat usługi Power BI, zobacz [www.powerbi.com](https://powerbi.microsoft.com).

## <a name="how-does-power-bi-workspace-collections-improve-my-app"></a>Jak kolekcje obszarów roboczych usługi Power BI może poprawić mojej aplikacji?

Aplikacje są znacznie większe możliwości, gdy można wykorzystać niesamowite, interaktywne wizualizacje danych, aby ułatwić podjęcie decyzji użytkownika, bezpośrednio w aplikacji. Kolekcje obszarów roboczych usługi Power BI pozwala zwiększyć swoją aplikację przy użyciu interaktywnych, zawsze aktualne, rozbudowanych wizualizacji danych, aby zwiększyć użyteczność aplikacji, dostępnych na urządzeniach oraz zadowolenia użytkowników i dostarczanie kontekstowych analizy przy użyciu jej obsługi ułatwiają realizację na dowolnym urządzeniu.

## <a name="are-there-any-rules-or-restrictions-about-how-i-can-use-power-bi-workspace-collections-in-my-app"></a>Istnieją żadnych zasad i ograniczeń dotyczących sposobu w mojej aplikacji można użyć z kolekcji obszarów roboczych usługi Power BI?

Kolekcje obszarów roboczych usługi Power BI są przeznaczone dla aplikacji, które są dostarczane dla podmiotów zewnętrznych. Jeśli chcesz użyć usługi kolekcji obszarów roboczych usługi Power BI do utworzenia wewnętrznej aplikacji biznesowej, każdy z użytkowników wewnętrznych musi usługi Power BI Pro subskrypcji użytkownika i organizacji obciążane opłatami za korzystanie z usługi kolekcji obszarów roboczych usługi Power BI Oprócz opłat za licencje subskrypcji użytkownika usługi Power BI Pro. Aby uniknąć naliczania opłat subskrypcji użytkownika usługi Power BI Pro i koszty użycia kolekcji obszarów roboczych usługi Power BI dla wewnętrznych aplikacji, usługa Power BI oferuje własne możliwości osadzania zawartości poza kolekcji obszarów roboczych usługi Power BI dla bez żadnych dodatkowych kosztów Posiadacze subskrypcji użytkownika usługi Power BI zasilania (witrynie dev.powerbi.com).

## <a name="can-power-bi-workspace-collections-be-used-to-create-internal-applications"></a>Może służyć do tworzenia aplikacji wewnętrznych kolekcji obszarów roboczych usługi Power BI?

Kolekcje obszarów roboczych usługi Power BI nie są przeznaczone tylko dla użytkowników zewnętrznych i nie powinny być używane w wewnętrznych aplikacjach biznesowych. W celu osadzenia zawartości usługi Power BI do użytku w wewnętrznych aplikacjach biznesowych należy użyć usługi Power BI, a wszyscy użytkownicy korzystający z tej zawartości muszą mieć ważną licencję subskrypcji użytkownika usługi Power BI Free lub Power BI Pro. Więcej informacji na temat sposobu integrowania aplikacji wewnętrznych przy użyciu usługi Power BI jest dostępna na [ https://dev.powerbi.com ](https://dev.powerbi.com).

## <a name="is-this-service-available-globally"></a>Jest to usługa dostępna globalnie?

Usługa kolekcje obszarów roboczych usługi Power BI jest teraz dostępna w większości centrów danych. Zawsze możesz sprawdzić dostępność najnowszej [tutaj](https://azure.microsoft.com/status/).

## <a name="what-is-the-available-sla-for-the-service"></a>Co to jest dostępna umowa SLA usługi?

Kolekcje obszaru roboczego usługi Power BI przy użyciu standardowych Azure — umowa SLA. Zobacz [umowy dotyczące poziomu usług](https://azure.microsoft.com/support/legal/sla/) Aby uzyskać więcej informacji.

## <a name="what-is-a-report-session-and-how-is-it-billed"></a>Co to jest sesja raportu i jak jest rozliczany?

Sesja jest zestawem interakcji między użytkownik końcowy a raportem kolekcji obszarów roboczych usługi Power BI. Każdorazowo, gdy raport kolekcji obszarów roboczych usługi Power BI jest wyświetlany użytkownikowi, inicjowana jest sesja, a dla posiadacza subskrypcji jest naliczana opłata za sesję. Sesje są rozliczane w stałej cenie niezależnej od liczby elementów wizualnych w raporcie ani częstotliwości odświeżania zawartości. Sesja kończy się po użytkownik zamknie raportu lub upłynie limit czasu sesji po godzinie.

## <a name="do-you-offer-any-tools-or-guidance-to-help-me-estimate-how-many-renderssession-i-should-expect-how-will-i-know-how-many-renders-have-been-completed"></a>Czy są dostępne wszystkie narzędzia i wskazówki ułatwią szacowanie liczby renderów/sesji mogę oczekiwać? Skąd mam wiedzieć, ile renderuje zostały ukończone?

Azure portal udostępnia rozliczeniowych, na ile renderuje / sesji raportu mogły zostać wykonane względem Twojej subskrypcji.

## <a name="do-i-need-a-power-bi-subscription-in-order-to-develop-applications-with-power-bi-workspace-collections-how-do-i-get-started"></a>Należy subskrypcji usługi Power BI w celu opracowywania aplikacji za pomocą kolekcji obszarów roboczych usługi Power BI? Jak rozpocząć?

Jako Deweloper aplikacji nie trzeba mieć subskrypcję usługi Power BI, aby tworzyć raporty i wizualizacje, które mają być używane w aplikacji. Potrzebujesz subskrypcji Microsoft Azure i bezpłatnych aplikacji Power BI Desktop.

Zobacz naszą dokumentację usługi, aby uzyskać szczegółowe informacje na temat sposobu korzystania z usługi kolekcji obszarów roboczych usługi Power BI.

## <a name="i-have-an-azure-subscription-can-i-use-power-bi-workspace-collections-using-my-existing-subscription"></a>Czy mogę mieć subskrypcję platformy Azure. Można użyć kolekcji obszarów roboczych usługi Power BI przy użyciu już posiadanej subskrypcji?

Tak. Można użyć istniejącej subskrypcji platformy Azure do aprowizacji i jest używana usługa kolekcje obszar roboczy usługi Microsoft Power BI.

## <a name="does-my-application-end-user-need-a-power-bi-license"></a>Czy Moje użytkownika końcowego aplikacji wymaga licencji usługi Power BI?

Nie. Użytkownicy końcowi aplikacji nie są wymagane do zakupu lub subskrypcji usługi Power BI oddzielnie Aby uzyskać dostęp do wizualizacji danych w aplikacji. W modelu kolekcji obszarów roboczych usługi Power BI, jako dostawca aplikacji jest naliczana za usługę za pośrednictwem mierników użycia platformy Azure. Zapoznaj się [strony cenowe i licencjonowania](https://go.microsoft.com/fwlink/?LinkId=760527).

## <a name="how-does-user-authentication-work-with-power-bi-workspace-collections"></a>Jak działa uwierzytelnianie użytkowników za pomocą kolekcji obszarów roboczych usługi Power BI?

Usługa kolekcje obszarów roboczych usługi Power BI używa tokenów aplikacji do uwierzytelniania i autoryzacji zamiast jawnego uwierzytelniania użytkowników końcowych. W modelu tokenu aplikacji Aplikacja zarządza uwierzytelniania i autoryzacji dla użytkowników końcowych. Następnie gdy jest to konieczne, tworzy aplikację

i wysyła tokenów aplikacji, który informuje o naszej usługi w celu renderowania żądanego raportu. Ten projekt nie wymaga aplikację do używania usługi Azure AD do uwierzytelniania i autoryzacji, użytkowników, choć można to zrobić. Dowiedz się więcej o tokenach aplikacji [tutaj](app-token-flow.md). Wprowadziliśmy także funkcji zabezpieczeń na poziomie wiersza (RLS) w kolekcji obszarów roboczych usługi Power BI zaawansowane zabezpieczenia scenariusze filtrowania.

## <a name="what-data-sources-are-currently-supported-with-power-bi-workspace-collections"></a>Do jakich źródeł danych są obecnie obsługiwane za pomocą kolekcji obszarów roboczych usługi Power BI?

Użyjemy do obsługi dostępu do źródeł danych w chmurze, korzystających z podstawowych poświadczeń za pomocą zapytania bezpośredniego. Oznacza to, że źródeł, takich jak bazy danych SQL Azure i magazyn danych SQL Azure są obecnie obsługiwane. Firma Microsoft może dodać obsługę innych źródeł danych i uzyskiwać dostęp do typów w najbliższych miesiącach. Aby uzyskać więcej informacji, zobacz [łączenie ze źródłem danych](connect-datasource.md).

## <a name="how-does-the-tenancy-model-work-for-power-bi-workspace-collections"></a>Jak działa model dzierżawy dla kolekcji obszarów roboczych usługi Power BI?

W modelu kolekcji obszarów roboczych usługi Power BI nie ma jawnych mieć klientów w dzierżawach usługi Azure AD. Możesz wybrać opcję wymagają usługi Azure AD dla swoich klientów, czy nie. W rezultacie Architektura aplikacji i infrastruktury jest co określa model obsługi dzierżawców wymaganych do kolekcji obszarów roboczych usługi Power BI.

Deweloperzy/pracowników nad lub kompilowania aplikacji musi mieć konto użytkownika usługi AAD, gdy do zarządzania subskrypcją platformy Azure i kolekcje obszarów roboczych w witrynie Azure portal. Programistycznych interfejsów API, aby umożliwić programistom importowania raportów, zmodyfikuj parametry połączenia i uzyskać osadzenia adresy URL używają tokenów aplikacji do uwierzytelniania, a w rezultacie nie wymagają usługi AAD.
