# <a name="choose-the-right-deployment-option-in-azure-sql"></a>Wybierz odpowiednią opcję wdrażania w usłudze Azure SQL

Dowiedz się, jak każda opcja wdrażania pasuje do platformy danych SQL platformy Azure firmy Microsoft i uzyskaj pomoc dopasowaną do odpowiednich opcji do wymagań biznesowych. Niezależnie od tego, czy nadajesz priorytet oszczędnościom kosztów, czy minimalnej administracji, ten artykuł pomoże Ci zdecydować, które podejście spełnia wymagania biznesowe, na których najbardziej Ci zależy.

## <a name="microsofts-azure-sql-data-platform"></a>Platforma danych SQL platformy Azure firmy Microsoft

Od migracji typu "lift-and-shift", przez modernizację istniejących aplikacji, po tworzenie nowoczesnych usług w chmurze — usługa Azure SQL to nowoczesna platforma SQL, która oferuje kilka opcji wdrażania, opartych na wiodącym w branży silniku programu Microsoft SQL Server. Usługa Azure SQL została zaprojektowana do obsługi szerokiej gamy wzorców aplikacji z różnymi poziomami kontroli nad podstawową platformą, aby spełnić najbardziej wymagające wymagania dotyczące migracji i modernizacji. Usługa Azure SQL eliminuje złożoność zarządzania różnymi kolekcjami aplikacji opartych na programie SQL Server na dużą skalę, zapewniając jedno, ujednolicone środowisko zarządzania.

Podczas każdej dyskusji dotyczącej baz danych Azure w zestawieniu z lokalnymi bazami danych programu SQL Server należy przede wszystkim pamiętać o tym, że można używać ich wszystkich. Platforma danych firmy Microsoft wykorzystuje technologię SQL Server i udostępnia ją na fizycznych komputerach lokalnych, środowiskach chmury prywatnej, hostowanych środowiskach chmury prywatnej innych firm i chmurze publicznej. Program SQL Server na maszynach wirtualnych platformy Azure (maszyny wirtualne SQL) umożliwia zaspokojenie unikatowych i zróżnicowanych potrzeb biznesowych dzięki połączeniu wdrożeń lokalnych i hostowanych w chmurze, przy użyciu tego samego zestawu produktów serwerowych, narzędzi programistycznych i wiedzy specjalistycznej w tych środowiskach.

   ![Opcje programu SQL Server w chmurze: program SQL Server w usłudze IaaS lub baza danych SQL w usłudze SaaS w chmurze.](./media/sql-database-paas-vs-sql-server-iaas/SQLIAAS_SQL_Server_Cloud_Continuum.png)

Jak widać na diagramie, każda oferta może charakteryzować się poziomem administracji nad infrastrukturą i stopniem efektywności kosztowej.

Na platformie Azure obciążenia programu SQL Server mogą być uruchomione jako usługa hostowana[(PaaS)](https://azure.microsoft.com/overview/what-is-paas/)lub infrastruktura hostowana[(IaaS).](https://azure.microsoft.com/overview/what-is-iaas/) W paaS masz wiele opcji wdrażania i warstwy usług w ramach każdej opcji wdrażania. Kluczowe pytanie, które należy zadać przy podejmowaniu decyzji między PaaS lub IaaS jest chcesz zarządzać bazą danych, stosować poprawki i wykonywać kopie zapasowe, czy chcesz delegować te operacje na platformę Azure?

W zależności od odpowiedzi dostępne są następujące opcje:

- [**Bazy danych SQL:**](sql-database-technical-overview.md)Najlepsze dla nowoczesnych aplikacji w chmurze, które chcą korzystać z najnowszych stabilnych funkcji programu SQL Server i mają ograniczenia czasowe w rozwoju i marketingu. W pełni zarządzany aparat bazy danych SQL, oparty na najnowszej stabilnej wersji programu SQL Server w wersji Enterprise. Jest to relacyjna baza danych jako usługa (DBaaS) hostowana w chmurze platformy Azure, która należy do kategorii branżowej *Platformy jako usługi (PaaS).* Baza danych SQL ma wiele opcji wdrażania, z których każda jest zbudowana na standardowym sprzęcie i oprogramowaniu, które jest własnością, hostowane i obsługiwane przez firmę Microsoft. Za pomocą programu SQL Server można używać wbudowanych funkcji i funkcji, które wymagają rozbudowanej konfiguracji (lokalnie lub na maszynie wirtualnej platformy Azure). Płatność za korzystanie z usługi SQL Database odbywa się zgodnie z rzeczywistym użyciem, a opcje pozwalają na skalowanie w górę lub w poziomie, dzięki czemu klienci mają do dyspozycji większe możliwości bez najmniejszych zakłóceń. Baza danych SQL ma kilka dodatkowych funkcji, które nie są dostępne w programie SQL Server, takich jak wbudowana wysoka dostępność, analizy i zarządzania.


  Bazy danych oferują następujące opcje wdrażania:
  - Jako [***pojedyncza baza danych***](sql-database-single-database.md) z własnym zestawem zasobów zarządzanych za pośrednictwem serwera bazy danych. Pojedyncza baza danych jest podobna do [zawartej bazy danych](https://docs.microsoft.com/sql/relational-databases/databases/contained-databases) w programie SQL Server. Ta opcja jest zoptymalizowana pod kątem nowoczesnego tworzenia aplikacji nowych aplikacji opartych na chmurze. Dostępne są opcje [hiperskali](sql-database-service-tier-hyperscale.md) i [bezserwerowe.](sql-database-serverless.md)
  - [***Pula elastyczna***](sql-database-elastic-pool.md), która jest zbiorem baz danych z udostępnionym zestawem zasobów zarządzanych za pośrednictwem serwera bazy danych. Pojedyncze bazy danych mogą być przenoszone do i z puli elastycznej. Ta opcja jest zoptymalizowana pod kątem nowoczesnego tworzenia aplikacji nowych aplikacji opartych na chmurze przy użyciu wzorca aplikacji SaaS z wieloma dzierżawcami. Pule elastyczne zapewniają ekonomiczne rozwiązanie do zarządzania wydajnością wielu baz danych, które mają wzorce użycia zmiennych.
  - [***Serwer bazy danych***](sql-database-servers.md), który służy do zarządzania grupami pojedynczych baz danych i pul elastycznych. Serwery baz danych działają jako centralny punkt administracyjny dla wielu pojedynczych lub branych baz danych, [loginów,](sql-database-manage-logins.md) [reguł zapory,](sql-database-firewall-configure.md) [reguł inspekcji,](sql-database-auditing.md) [zasad wykrywania zagrożeń](sql-database-threat-detection.md)i grup trybu [failover.](sql-database-auto-failover-group.md)

- [**Wystąpienia zarządzane sql:**](sql-database-managed-instance.md)Najlepsze dla większości migracji do chmury. Wystąpienie zarządzane to zbiór baz danych systemu i użytkowników z udostępnionym zestawem zasobów, który jest gotowy do podnoszenia i zmiany. Najlepsze rozwiązanie dla nowych aplikacji lub istniejących aplikacji lokalnych, które chcą korzystać z najnowszych stabilnych funkcji programu SQL Server i które są migrowane do chmury przy minimalnych zmianach. Wystąpienie zarządzane jest podobne do wystąpienia [aparatu bazy danych programu Microsoft SQL Server](https://docs.microsoft.com/sql/database-engine/sql-server-database-engine-overview) oferującego zasoby udostępnione dla baz danych i dodatkowych funkcji o zakresie wystąpienia. Wystąpienie zarządzane obsługuje migrację bazy danych z lokalnego przy minimalnej do żadnej zmianie bazy danych. Ta opcja zapewnia wszystkie zalety PaaS usługi Azure SQL Database, ale dodaje możliwości, które wcześniej były dostępne tylko w maszynach wirtualnych SQL. Obejmuje to natywną sieć wirtualną (VNet) i prawie 100% zgodności z lokalnym programem SQL Server. Wystąpienia zarządzane zapewniają pełny dostęp do programu SQL Server i zgodność funkcji do migracji serwerów SQL na platformę Azure.


- [**Maszyny wirtualne SQL:**](../virtual-machines/windows/sql/virtual-machines-windows-sql-server-iaas-overview.md)Najlepsze dla migracji i aplikacji wymagających dostępu na poziomie systemu operacyjnego. Maszyny wirtualne SQL są gotowe do podnoszenia i zmiany dla istniejących aplikacji, które wymagają szybkiej migracji do chmury przy minimalnych zmianach lub bez zmian. Maszyny wirtualne SQL oferują pełną kontrolę administracyjną nad wystąpieniem programu SQL Server i bazowym systemem operacyjnym w celu migracji na platformę Azure. Scenariusze szybkiego tworzenia i testowania, gdy nie chcesz kupować lokalnego nieprodukcyjnego sprzętu dla programu SQL Server. Maszyny wirtualne SQL należą do kategorii *branżowej Infrastruktura jako usługa (IaaS)* i umożliwiają uruchamianie programu SQL Server wewnątrz w pełni zarządzanej maszyny wirtualnej (VM) w chmurze platformy Azure. Maszyny wirtualne SQL działają również na standardowym sprzęcie, który jest własnością, hostuje i jest obsługiwany przez firmę Microsoft. Korzystając z maszyn wirtualnych SQL, można albo zapłacić zgodnie z rzeczywistym użyciem licencję programu SQL Server już zawartą w obrazie programu SQL Server lub łatwo użyć istniejącej licencji. W razie potrzeby można również zatrzymać lub wznowić maszynę wirtualną. Program SQL Server zainstalowany i hostowany w chmurze działa na maszynach wirtualnych systemu Windows Server lub Linux działających na platformie Azure, nazywanych również infrastrukturą jako usługą (IaaS). Maszyny wirtualne SQL jest dobrym rozwiązaniem do migracji lokalnych baz danych i aplikacji programu SQL Server bez żadnych zmian w bazie danych. Wszystkie najnowsze wersje programu SQL Server są dostępne do instalacji na maszynie wirtualnej IaaS. Najbardziej znaczącą różnicą w stosunku do baz danych SQL i wystąpień zarządzanych SQL jest to, że maszyny wirtualne programu SQL Server umożliwiają pełną kontrolę nad aparatem bazy danych. Można wybrać, kiedy rozpocząć konserwację/poprawki, zmienić model odzyskiwania na prosty lub rejestrowane zbiorczo, wstrzymać lub uruchomić usługę w razie potrzeby i można w pełni dostosować aparat bazy danych programu SQL Server. Dzięki tej dodatkowej kontroli pochodzi dodatkowa odpowiedzialność za zarządzanie maszyną wirtualną.

  Zoptymalizowane pod kątem migracji istniejących aplikacji na platformę Azure lub rozszerzania istniejących aplikacji lokalnych do chmury we wdrożeniach hybrydowych. Ponadto przy użyciu programu SQL Server na maszynie wirtualnej można tworzyć i testować tradycyjne aplikacje programu SQL Server. W przypadku maszyn wirtualnych SQL masz pełne prawa administracyjne za pośrednictwem dedykowanego wystąpienia programu SQL Server i maszyny wirtualnej opartej na chmurze. To doskonałe rozwiązanie, gdy firma ma już dostępne zasoby informatyczne do obsługi maszyn wirtualnych. Te funkcje umożliwiają stworzenie doskonale dopasowanego systemu, który spełni określone wymagania aplikacji związane z wydajnością i dostępnością.


Dodatkowe różnice są wymienione w poniższej tabeli, ale ***zarówno bazy danych, jak i wystąpienia zarządzane są zoptymalizowane pod kątem zmniejszenia ogólnych kosztów zarządzania do minimum w przypadku inicjowania obsługi administracyjnej wielu baz danych i zarządzania nimi.*** Obniża bieżące koszty administracyjne, ponieważ nie jest konieczne zarządzanie maszynami wirtualnymi, systemem operacyjnym ani oprogramowaniem bazy danych. Nie wymaga zarządzania uaktualnieniami, wysoką dostępnością ani [kopiami zapasowymi](sql-database-automated-backups.md). Ogólnie rzecz biorąc, usługa Azure SQL Database może znacząco zwiększyć liczbę baz danych zarządzanych przez pojedynczy zasób informatyczny lub projektowy. [Pule elastyczne](sql-database-elastic-pool.md) obsługują również architektury aplikacji wielodostępnych SaaS z funkcjami, w tym izolacją dzierżawy i możliwością skalowania w celu zmniejszenia kosztów przez udostępnianie zasobów w bazach danych. [Wystąpienie zarządzane](sql-database-managed-instance.md) zapewnia obsługę funkcji o zakresie wystąpienia umożliwiających łatwą migrację istniejących aplikacji, a także udostępnianie zasobów między bazami danych.

## <a name="business-motivations-for-choosing-databases-managed-instances-or-sql-virtual-machines"></a>Motywacje biznesowe do wybierania baz danych, wystąpień zarządzanych lub maszyn wirtualnych SQL

Istnieje kilka czynników, które mogą mieć wpływ na twoją decyzję o wyborze między różnymi ofertami danych:

- [Koszt](#cost) — opcja PaaS i IaaS obejmuje cenę podstawową, która obejmuje podstawową infrastrukturę i licencjonowanie. Jednak dzięki opcji IaaS musisz zainwestować dodatkowy czas i zasoby, aby zarządzać bazą danych, podczas gdy w PaaS otrzymujesz te funkcje administracyjne zawarte w cenie. Opcja IaaS umożliwia zamknięcie zasobów, gdy nie są one używane do zmniejszenia kosztów, podczas gdy wersja PaaS jest zawsze uruchomiona, chyba że jeśli upuścisz i ponownie utworzysz zasoby, gdy są potrzebne.
- [Administracja](#administration) — opcje PaaS skracają czas potrzebny do zainwestowania w administrowanie bazą danych. Jednak ogranicza również zakres niestandardowych zadań administracyjnych i skryptów, które można wykonać lub uruchomić. Na przykład clr nie jest obsługiwany z pojedynczych lub puli baz danych, ale jest obsługiwany dla wystąpienia zarządzanego. Ponadto żadne opcje wdrażania w PaaS obsługują użycie flag śledzenia.
- [Umowa na poziomie usług](#service-level-agreement-sla) — zarówno IaaS, jak i PaaS zapewniają wysoką, standardową umowę SLA. Opcja PaaS gwarantuje 99,99% umowy SLA, podczas gdy usługa IaaS gwarantuje 99,95% umowy SLA dla infrastruktury, co oznacza, że należy wdrożyć dodatkowe mechanizmy zapewniające dostępność baz danych. Rozwiązanie o wysokiej dostępności można zaimplementować w 99,99% przez utworzenie dodatkowego programu SQL Server na maszynie Wirtualnej i skonfigurowanie grup dostępności AlwaysOn.
- [Czas przejścia na platformę Azure](#market) — sql server w usłudze Azure VM jest dokładne dopasowanie środowiska, więc migracja z lokalnego do usługi Azure SQL VM nie różni się od przenoszenia baz danych z jednego serwera lokalnego do innego. Wystąpienie zarządzane umożliwia również niezwykle łatwą migrację; jednak może być kilka zmian, które należy zastosować przed migracją do wystąpienia zarządzanego.

Czynniki te zostaną omówione bardziej szczegółowo w poniższych sekcjach.

### <a name="cost"></a>Koszty

Ograniczone fundusze są często podstawowym czynnikiem decydującym o wyborze sposobu hostowania baz danych — niezależnie od tego, czy Twoja firma to start-up o ograniczonych środkach finansowych, czy należysz do zespołu firmowego, który musi zmieścić się w zaplanowanym budżecie. W tej sekcji dowiesz się o podstawach rozliczeń i licencjonowania na platformie Azure w odniesieniu do tych dwóch relacyjnych opcji bazy danych: bazy danych SQL i maszyn wirtualnych SQL. Poznasz również sposób obliczania całkowitego kosztu aplikacji.

#### <a name="billing-and-licensing-basics"></a>Podstawowe informacje dotyczące rozliczeń i licencjonowania

Obecnie **baza danych SQL** jest sprzedawana jako usługa i jest dostępna z kilkoma opcjami wdrażania i w kilku warstwach usług z różnymi cenami zasobów, z których wszystkie są rozliczane co godzinę według stałej stawki na podstawie warstwy usług i wybranego rozmiaru obliczeń. Aby uzyskać najnowsze informacje na temat bieżących obsługiwanych warstw usług, rozmiarów obliczeń i kwot magazynu, zobacz [model zakupów oparty na jednostce DTU](sql-database-service-tiers-dtu.md) i model zakupu oparty na [łanie wirtualnej](sql-database-service-tiers-vcore.md).

- Dzięki bazie danych SQL można wybrać warstwę usług, która odpowiada Twoim potrzebom, z szerokiego zakresu cen, począwszy od 5$/miesiąc dla warstwy podstawowej.
- Można utworzyć [pule elastyczne](sql-database-elastic-pool.md) do udostępniania zasobów między wystąpieniami bazy danych, aby zmniejszyć koszty i pomieścić skoki użycia.
- Za pomocą wystąpienia zarządzanego SQL można również przynieść własną licencję. Aby uzyskać więcej informacji na temat licencjonowania typu bring-your-own, zobacz [Mobilność licencji za pośrednictwem pakietu Software Assurance na platformie Azure](https://azure.microsoft.com/pricing/license-mobility/) lub skorzystaj z [kalkulatora korzyści hybrydowych platformy Azure,](https://azure.microsoft.com/pricing/hybrid-benefit/#sql-database) aby dowiedzieć się, jak **zaoszczędzić do 40%.**

Rozliczany jest również internetowy ruch wychodzący po zwykłych [stawkach transferu danych](https://azure.microsoft.com/pricing/details/data-transfers/). Można dynamicznie dostosowywać warstwy usług i rozmiary obliczeń, aby dopasować je do różnych potrzeb aplikacji w zakresie przepływności.

Dzięki **bazom danych SQL i wystąpieniom zarządzanym SQL**oprogramowanie bazy danych jest automatycznie konfigurowane, łatane i uaktualniane przez firmę Microsoft, co zmniejsza koszty administracyjne. Ponadto [wbudowana funkcja tworzenia kopii zapasowych](sql-database-automated-backups.md) pozwala osiągnąć znaczne oszczędności, zwłaszcza w przypadku dużej liczby baz danych.

Dostępne są wszystkie obsługiwane wersje programu SQL Server (2008R2, 2012, 2014, 2016) i edycje (Developer, Express, Web, Standard, Enterprise). Ponadto są dostępne wersje bring-your-own-license (BYOL) obrazów. W przypadku użycia obrazów dostarczanych przez platformę Azure koszty operacyjne zależą od rozmiaru maszyny wirtualnej, a także od wybranej wersji programu SQL Server. Więcej informacji na temat przenoszenia własnej licencji można znaleźć w temacie [Przenośność licencji za pośrednictwem programu Software Assurance w systemie Azure](https://azure.microsoft.com/pricing/license-mobility/). ### <a name="administration"></a>Administracja

Dla wielu firm decyzja o przeniesieniu usług do usługi w chmurze wiąże się zarówno z ograniczeniem złożoności w zakresie administracji, jak i obniżeniem kosztów. Dzięki iaaS i PaaS firma Microsoft administruje podstawową infrastrukturą i automatycznie replikuje wszystkie dane w celu zapewnienia odzyskiwania po awarii, konfiguruje i uaktualnia oprogramowanie bazy danych, zarządza równoważeniem obciążenia i wykonuje przezroczystą komunikację awaryjną, jeśli awarii serwera w centrum danych.

- Dzięki **bazom danych SQL i wystąpieniom zarządzanym SQL**można nadal administrować bazą danych, ale nie trzeba już zarządzać aparatem bazy danych, systemem operacyjnym ani sprzętem. Przykładowe elementy, którymi można dalej zarządzać to bazy danych, identyfikatory logowania, dostrajanie indeksów i zapytań oraz inspekcja i zabezpieczenia. Ponadto skonfigurowanie wysokiej dostępności do innego centrum danych wymaga minimalnej konfiguracji i administrowania.
- W **przypadku maszyn wirtualnych SQL**masz pełną kontrolę nad systemem operacyjnym i konfiguracją wystąpienia programu SQL Server. W przypadku korzystania z maszyny wirtualnej decydujesz o tym, kiedy należy zaktualizować/uaktualnić system operacyjny i oprogramowanie bazy danych i kiedy zainstalować dodatkowe oprogramowanie, np. oprogramowanie antywirusowe. Dostępnych jest kilka funkcji automatycznych, które znacznie upraszczają instalowanie poprawek, tworzenie kopii zapasowej i zapewnianie wysokiej dostępności. Ponadto możesz kontrolować rozmiar maszyny wirtualnej, liczbę dysków oraz ich konfiguracje magazynu. Platforma Azure umożliwia zmianę rozmiaru maszyny wirtualnej zgodnie z zapotrzebowaniem. Więcej informacji można znaleźć w temacie [Virtual Machine and Cloud Service Sizes for Azure](../virtual-machines/windows/sizes.md) (Rozmiary maszyn wirtualnych i usług w chmurze na platformie Azure).

### <a name="service-level-agreement-sla"></a>Umowa dotycząca poziomu usług (SLA)

Dla wielu działów IT wypełnienie zobowiązań wynikających z umowy dotyczącej poziomu usług (SLA) ma najwyższy priorytet. W tej sekcji opisano, jakie warunki umowy SLA stosuje się do poszczególnych opcji obsługi bazy danych.

W przypadku **bazy danych SQL**firma Microsoft udostępnia umowę SLA o dostępności 99,99%. Najnowsze informacje można znaleźć w artykule [SQL Database — umowa SLA](https://azure.microsoft.com/support/legal/sla/sql-database/).

W przypadku **maszyn wirtualnych SQL**firma Microsoft udostępnia umowę SLA o dostępności w wysokości 99,95%, która obejmuje tylko maszynę wirtualną. Umowa SLA nie obejmuje procesów (np. programu SQL Server) uruchomionych na maszynie wirtualnej i wymaga obsługi przynajmniej dwóch wystąpień maszyny wirtualnej w zbiorze dostępności. Najnowsze informacje znajdują się w artykule [Maszyny wirtualne — umowa SLA](https://azure.microsoft.com/support/legal/sla/virtual-machines/). Aby uzyskać wysoką dostępność bazy danych (HA) w maszynach wirtualnych, należy skonfigurować jedną z obsługiwanych opcji wysokiej dostępności w programie SQL Server, taką jak [Grupy dostępności zawsze włączone.](https://docs.microsoft.com/sql/database-engine/availability-groups/windows/always-on-availability-groups-sql-server) Użycie obsługiwanej opcji wysokiej dostępności nie zapewnia dodatkowej umowy SLA, ale umożliwia osiągnięcie dostępności bazy danych na poziomie > 99,99%.

### <a name="time-to-move-to-azure"></a><a name="market"></a>Czas przejść na platformę Azure

**Baza danych SQL** (pojedyncze bazy danych lub pule elastyczne) są właściwym rozwiązaniem dla aplikacji zaprojektowanych w chmurze, gdy produktywność programistów i szybki czas na rynek nowych rozwiązań mają kluczowe znaczenie. Dzięki funkcjonalności przypominającej model DBA jest doskonała dla architektów i deweloperów chmury, ponieważ zmniejsza potrzebę zarządzania bazowym systemem operacyjnym i bazą danych.

**Wystąpienie zarządzane SQL** znacznie upraszcza migrację istniejących aplikacji do usługi Azure SQL, umożliwiając szybkie wprowadzanie zmigrowanych aplikacji bazy danych na rynek na platformie Azure.

**Maszyny wirtualne SQL** są idealne, jeśli istniejące lub nowe aplikacje wymagają dużych baz danych lub dostępu do wszystkich funkcji w programie SQL Server lub Windows/Linux i chcesz uniknąć czasu i kosztów nabycia nowego sprzętu lokalnego. Jest to również dobre dopasowanie, gdy chcesz przeprowadzić migrację istniejących lokalnych aplikacji i baz danych na platformę Azure w stanie takim, w jakim — w przypadkach, gdy wystąpienie zarządzanego usługi Azure SQL Database nie jest dobrym rozwiązaniem. Ponieważ nie trzeba zmieniać warstw prezentacji, aplikacji i danych, można zaoszczędzić czas i budżet na ponowne zaprojektowanie istniejącego rozwiązania. Zamiast tego możesz skoncentrować się na migracji wszystkich rozwiązań do platformy Azure i przeprowadzeniu optymalizacji wydajności, których może wymagać platforma Azure. Więcej informacje zawiera artykuł [Performance Best Practices for SQL Server on Azure Virtual Machines](../virtual-machines/windows/sql/virtual-machines-windows-sql-performance.md) (Najlepsze praktyki dotyczące wydajności dla programu SQL Server w usłudze Azure Virtual Machines).
