# <a name="overview-of-the-azure-security-benchmark-blueprint-sample"></a>Omówienie przykładu strategii testu porównawczego zabezpieczeń platformy Azure

Przykład strategii testu porównawczego zabezpieczeń platformy Azure zapewnia wytyczne dotyczące ładu w usłudze [Azure Policy](../../../policy/overview.md), które ułatwiają ocenę określonych mechanizmów kontroli [testu porównawczego zabezpieczeń platformy Azure](../../../../security/benchmarks/overview.md). Ta strategia ułatwia klientom wdrażanie podstawowego zestawu zasad dla dowolnej architektury wdrożonej przez platformę Azure, w której planowane jest zaimplementowanie mechanizmów kontroli testu porównawczego zabezpieczeń platformy Azure.

## <a name="control-mapping"></a>Mapowanie kontrolek

Sekcja mapowania mechanizmów kontroli zawiera szczegółowe informacje dotyczące zasad w ramach tej strategii i o tym, jak te zasady pomagają spełnić wymagania różnych mechanizmów kontroli w teście porównawczym zabezpieczeń platformy Azure. Po przypisaniu do architektury zasoby są oceniane przez usługę Azure Policy pod względem braku zgodności z przypisanymi zasadami. Aby uzyskać więcej informacji, zobacz [Azure Policy](../../../policy/overview.md).
