# <a name="azure-status-overview"></a>Omówienie stanu platformy Azure

[Stan platformy Azure](https://status.azure.com/status/) zapewnia globalny widok kondycji usług i regionów platformy Azure. Dzięki statusowi platformy Azure można uzyskać informacje o dostępności usługi. Stan platformy Azure jest dostępny dla wszystkich, aby wyświetlić wszystkie usługi, które zgłaszają ich kondycję usługi, a także zdarzenia o szerokim zakresie wpływu. Jeśli jednak jesteś obecnym użytkownikiem platformy Azure, zdecydowanie zachęcamy do korzystania ze spersonalizowanego środowiska w [usłudze Azure Service Health.](https://aka.ms/azureservicehealth) Usługa Azure Service Health obejmuje wszystkie awarie, nadchodzące planowane działania konserwacyjne i porady dotyczące usług.

![Strona stanu platformy Azure](./media/azure-status-overview/azure-status.PNG)

## <a name="azure-status-updates"></a>Aktualizacje stanu platformy Azure

Strona stanu platformy Azure jest aktualizowana w czasie rzeczywistym w miarę zmiany kondycji usług platformy Azure. Jeśli pozostawisz otwartą stronę stanu platformy Azure, możesz kontrolować szybkość odświeżania strony za pomocą nowych danych. U góry widać, kiedy strona została ostatnio zaktualizowana.

![Odświeżanie stanu platformy Azure](./media/azure-status-overview/update.PNG)

## <a name="azure-status-history"></a>Historia stanu platformy Azure

Strona Stan platformy Azure zawsze zawiera najnowsze informacje o kondycji, ale można wyświetlać starsze zdarzenia na [stronie Historia stanu platformy Azure](https://status.azure.com/status/history/).

Począwszy od zdarzeń, które zostały opublikowane w listopadzie 2019 r., strona historii stanu pokazuje zdarzenia do 365 dni w przeszłości, w tym wstępną przyczynę główną, łagodzenie i kolejne kroki dla platformy Azure.

## <a name="rss-feed"></a>Kanał informacyjny RSS

Stan platformy Azure zapewnia również [źródło danych RSS](https://status.azure.com/status/feed/) zmian kondycji usług platformy Azure, które można subskrybować.
