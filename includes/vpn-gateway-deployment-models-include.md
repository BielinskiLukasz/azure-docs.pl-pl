Warto wiedzieć, że obecnie platforma Azure obsługuje dwa modele wdrażania: model wdrażania przy użyciu usługi Resource Manager i model klasyczny. Przed rozpoczęciem konfiguracji należy sprawdzić, czy stosowane są instrukcje dotyczące odpowiedniego modelu wdrażania. Te dwa modele nie są ze sobą w pełni zgodne.

Jeśli na przykład korzystasz z sieci wirtualnej, która została utworzona przy użyciu klasycznego modelu wdrażania, i chcesz dodać połączenie z siecią wirtualną, należy użyć metod wdrażania, które odpowiadają klasycznemu modelowi wdrażania, a nie modelowi wdrażania przy użyciu usługi Resource Manager. Jeśli korzystasz z sieci wirtualnej, która została utworzona za pomocą modelu wdrażania przy użyciu usługi Resource Manager, należy użyć metod wdrażania, które odpowiadają modelowi wdrażania przy użyciu usługi Resource Manager, a nie modelowi klasycznemu.

Aby uzyskać więcej informacji na temat modeli wdrażania, zobacz temat [Omówienie wdrażania przy użyciu usług Resource Manager oraz wdrażania klasycznego](../articles/resource-manager-deployment-model.md).
