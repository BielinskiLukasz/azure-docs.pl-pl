# <a name="yaml-configuration-options-to-customize-the-build-tasks"></a>Opcje konfiguracji YAML w celu dostosowania zadań kompilacji

W tym artykule wymieniono wszystkie opcje konfiguracji YAML dostępne w każdym z zadań kompilacji. Artykuł rozpoczyna się od zadań dla narzędzi do analizy kodu zabezpieczeń. Kończy się zadaniami przetwarzania końcowego.

| ToolLogsNotFoundAction | lista wyboru | zawsze | True | Standardowa | Błąd, Brak, Standard, Ostrzeżenie | Nie znaleziono akcji, które należy podjąć po zalogowaniu wybranego narzędzia (lub dowolnego narzędzia, jeśli zaznaczono opcję Wszystkie narzędzia), co oznacza, że narzędzie nie zostało uruchomione.<br/><br/>**Opcje:**<br/>**Brak:** Komunikat jest zapisywany do strumienia wyjściowego verbose dostępne tylko przez ustawienie zmiennej VSTS **system.debug** **true**.<br/>**Standard:** (Domyślnie) Zapisuje standardowy komunikat wyjściowy, że nie znaleziono dzienników dla narzędzia.<br/>**Ostrzeżenie:** Zapisuje żółty komunikat ostrzegawczy, że nie znaleziono dzienników dla narzędzia, który pojawia się na stronie podsumowania kompilacji jako ostrzeżenie.<br/>**Błąd:** Zapisuje czerwony komunikat o błędzie i zgłasza wyjątek, przerywając kompilację. Użyj tej opcji, aby zapewnić indywidualne opcje narzędzi, aby zapewnić, które narzędzia zostały uruchomiony.
