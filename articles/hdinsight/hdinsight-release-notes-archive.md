# <a name="archived-release-notes"></a>Zarchiwizowane informacje o wersji

> [!IMPORTANT]  
> Linux jest jedynym systemem operacyjnym używanym w połączeniu z usługą HDInsight w wersji 3.4 lub nowszą. Aby uzyskać więcej informacji, zobacz [artykuł HDInsight versioning](hdinsight-component-versioning.md).

## <a name="summary"></a>Podsumowanie

Azure HDInsight jest jedną z najpopularniejszych usług używanych przez klientów korporacyjnych na potrzeby analizy w narzędziach typu open source Apache Hadoop i Apache Spark na platformie Azure.

## <a name="release-date-12172019"></a>Data wydania: 12/17/2019

Ta wersja dotyczy zarówno hdinsight 3.6 i 4.0.

### <a name="new-features"></a>Nowe funkcje

#### <a name="service-tags"></a>Tagi usługi
Tagi usług upraszczają zabezpieczenia maszyn wirtualnych platformy Azure i sieci wirtualnych platformy Azure, umożliwiając łatwe ograniczanie dostępu do sieci do usług platformy Azure. Tagów usługi w regułach sieciowej grupy zabezpieczeń (NSG) można zezwalać lub odmawiać ruchu do określonej usługi platformy Azure globalnie lub na region platformy Azure. Platforma Azure zapewnia obsługę adresów IP leżących u podstaw każdego tagu. Tagi usługi HDInsight dla sieciowych grup zabezpieczeń (NSG) to grupy adresów IP dla usług zarządzania kondycjią i zarządzaniem. Te grupy pomagają zminimalizować złożoność tworzenia reguł zabezpieczeń. Klienci usługi HDInsight mogą włączyć tag usługi za pośrednictwem usługi Azure Portal, PowerShell i REST API. Aby uzyskać więcej informacji, zobacz [Tagi usługi sieciowej grupy zabezpieczeń (NSG) dla usługi Azure HDInsight](https://docs.microsoft.com/azure/hdinsight/hdinsight-service-tags).

#### <a name="custom-ambari-db"></a>Niestandardowa baza danych Ambari
HDInsight pozwala teraz na korzystanie z własnego bazy danych SQL dla Apache Ambari. Tę niestandardową bazę danych Ambari można skonfigurować z witryny Azure portal lub za pośrednictwem szablonu menedżera zasobów.  Ta funkcja umożliwia wybór odpowiedniego bazy danych SQL dla potrzeb przetwarzania i pojemności. Można również łatwo uaktualnić, aby dopasować wymagania dotyczące rozwoju firmy. Aby uzyskać więcej informacji, zobacz [Konfigurowanie klastrów HDInsight przy niestandardowym ambari DB](hdinsight-custom-ambari-db.md).

#### <a name="transport-layer-security-tls-12-enforcement"></a>Egzekwowanie zabezpieczeń warstwy transportu (TLS) 1.2
Transport Layer Security (TLS) i Secure Sockets Layer (SSL) to protokoły kryptograficzne, które zapewniają bezpieczeństwo komunikacji za pośrednictwem sieci komputerowej. Aby uzyskać więcej informacji, zobacz [Zabezpieczenia warstwy transportu](https://en.wikipedia.org/wiki/Transport_Layer_Security#SSL_1.0.2C_2.0_and_3.0). Podczas gdy klastry usługi Azure HDInsight akceptują połączenia TLS 1.2 w publicznych punktach końcowych HTTPS, protokół TLS 1.1 jest nadal obsługiwany w celu zapewnienia zgodności z powrotem ze starszymi klientami.

Począwszy od następnej wersji, będzie można wyrazić zgodę i skonfigurować nowe klastry HDInsight, aby akceptowały tylko połączenia TLS 1.2. 

W dalszej części roku, począwszy od 30.06.2020 r., usługa Azure HDInsight wymusza wersje TLS 1.2 lub nowszych dla wszystkich połączeń HTTPS. Zaleca się, aby upewnić się, że wszyscy klienci są gotowe do obsługi TLS 1.2 lub nowszych wersji.

#### <a name="moving-to-azure-virtual-machine-scale-sets"></a>Przenoszenie do zestawów skalowania maszyny wirtualnej platformy Azure

Zobacz więcej o [zestawach skalowania maszyny wirtualnej platformy Azure](https://docs.microsoft.com/azure/virtual-machine-scale-sets/overview).

#### <a name="hdinsight-identity-broker-hib-preview"></a>HDInsight Identity Broker (HIB) (wersja zapoznawcza)

Obecnie hib jest dostępny tylko dla klastrów wdrożonych za pośrednictwem szablonu usługi Azure Resource Management (ARM).

#### <a name="kafka-rest-api-proxy-preview"></a>Serwer proxy interfejsu API odpoczynku platformy Kafka (wersja zapoznawcza)

Serwer proxy interfejsu API rest platformy Kafka zapewnia wdrożenie jednego kliknięcia wysoce dostępnego serwera proxy REST z klastrem platformy Kafka za pośrednictwem zabezpieczonej autoryzacji AAD i protokołu OAuth. 

#### <a name="auto-scale"></a>Automatyczne skalowanie

Skalowanie automatyczne dla usługi Azure HDInsight jest teraz ogólnie dostępne we wszystkich regionach dla typów klastrów Apache Spark i Hadoop. Ta funkcja umożliwia zarządzanie obciążeniami analizy dużych zbiorów danych w bardziej opłacalny i produktywny sposób. Teraz możesz zoptymalizować korzystanie z klastrów HDInsight i płacić tylko za to, czego potrzebujesz.

W zależności od wymagań można wybierać między skalowaniem automatycznym opartym na obciążeniu i harmonogramie. Skalowanie automatyczne oparte na obciążeniu może skalować rozmiar klastra w górę i w dół na podstawie bieżących potrzeb zasobów, podczas gdy skalowanie automatyczne oparte na harmonogramie może zmieniać rozmiar klastra na podstawie wstępnie zdefiniowanego harmonogramu. 

Obsługa skalowania automatycznego dla obciążenia HBase i LLAP jest również publiczna wersja zapoznawcza. Aby uzyskać więcej informacji, zobacz [Automatyczne skalowanie klastrów usługi Azure HDInsight](https://docs.microsoft.com/azure/hdinsight/hdinsight-autoscale-clusters).

Aby uzyskać więcej informacji, zobacz [Usługa Azure HDInsight — przyspieszone zapisy dla oprogramowania Apache HBase](https://docs.microsoft.com/azure/hdinsight/hbase/apache-hbase-accelerated-writes).

#### <a name="custom-ambari-db"></a>Niestandardowa baza danych Ambari

Usługa HDInsight oferuje teraz nową pojemność, aby umożliwić klientom korzystanie z własnego bazy danych SQL db dla Ambari. Teraz klienci mogą wybrać odpowiedni bazę danych SQL dla Ambari i łatwo uaktualnić go na podstawie własnych wymagań dotyczących rozwoju firmy. #### <a name="f-series-virtual-machines-are-now-available-with-hdinsight"></a>Maszyny wirtualne serii F są teraz dostępne z hdinsight

Maszyny wirtualne serii F (maszyny wirtualne) są dobrym wyborem, aby rozpocząć pracę z HDInsight z wymaganiami przetwarzania światła. Oferując niższą cenę za godzinę, seria F zapewnia najlepszy w portfolio platformy Azure stosunek ceny do wydajności, określany na podstawie liczby jednostek ACU (Azure Compute Unit) na procesor wirtualny vCPU. ### <a name="known-issues"></a>Znane problemy

#### <a name="replication-is-broken-for-secure-hbase-with-default-installation"></a>Replikacja jest przerywana dla secure HBase z instalacją domyślną

W przypadku gry HDInsight 4.0 wykonaj następujące czynności:

1. Włącz komunikację między klastrami.
1. Zaloguj się do aktywnego nakrycie głowy.
1. Pobierz skrypt, aby włączyć replikację za pomocą następującego polecenia:

    ```
    sudo wget https://raw.githubusercontent.com/Azure/hbase-utils/master/replication/hdi_enable_replication.sh
    ```
1. Wpisz polecenie `sudo kinit <domainuser>`.
1. Wpisz następujące polecenie, aby uruchomić skrypt:

    ```
    sudo bash hdi_enable_replication.sh -m <hn0> -s <srclusterdns> -d <dstclusterdns> -sp <srcclusterpasswd> -dp <dstclusterpasswd> -copydata
    ```
W przypadku gry HDInsight 3.6 wykonaj następujące czynności:

1. Zaloguj się do aktywnego HMaster ZK.
1. Pobierz skrypt, aby włączyć replikację za pomocą następującego polecenia:
    ```
    sudo wget https://raw.githubusercontent.com/Azure/hbase-utils/master/replication/hdi_enable_replication.sh
    ```
1. Wpisz polecenie `sudo kinit -k -t /etc/security/keytabs/hbase.service.keytab hbase/<FQDN>@<DOMAIN>`.
1. Wpisz następujące polecenie:

    ```bash
    sudo bash hdi_enable_replication.sh -s <srclusterdns> -d <dstclusterdns> -sp <srcclusterpasswd> -dp <dstclusterpasswd> -copydata
    ```

#### <a name="phoenix-sqlline-stops-working-after-migrating-hbase-cluster-to-hdinsight-40"></a>Phoenix Sqlline przestaje działać po migracji klastra HBase do programu HDInsight 4.0

Wykonaj następujące czynności:

1. Upuść następujące tabele Phoenix:
    1. `SYSTEM.FUNCTION`
    1. `SYSTEM.SEQUENCE`
    1. `SYSTEM.STATS`
    1. `SYSTEM.MUTEX`
    1. `SYSTEM.CATALOG`
1. Jeśli nie możesz usunąć żadnej z tabel, uruchom ponownie usługę HBase, aby wyczyścić wszelkie połączenia z tabelami.
1. Uruchom ponownie polecenie `sqlline.py`. Phoenix ponownie utworzy wszystkie tabele, które zostały usunięte w kroku 1.
1. Ponowne generowanie tabel i widoków phoenix dla danych HBase.

#### <a name="phoenix-sqlline-stops-working-after-replicating-hbase-phoenix-metadata-from-hdinsight-36-to-40"></a>Phoenix Sqlline przestaje działać po replikowaniu metadanych HBase Phoenix z hdinsight 3.6 do 4.0

Wykonaj następujące czynności:

1. Przed wykonaniem replikacji przejdź do docelowego klastra `sqlline.py`4.0 i wykonaj polecenie . To polecenie wygeneruje tabele Phoenix, takie jak `SYSTEM.MUTEX` i `SYSTEM.LOG` które istnieją tylko w 4.0.
1. Upuść następujące tabele:
    1. `SYSTEM.FUNCTION`
    1. `SYSTEM.SEQUENCE`
    1. `SYSTEM.STATS`
    1. `SYSTEM.CATALOG`
1. Uruchamianie replikacji bazy danych HBase

#### <a name="mahout"></a>Mahout

W HDP-2.3.x i 2.4.x, zamiast wysyłać konkretne wydanie Apache Mahout, zsynchronizowaliśmy się z określonym punktem rewizji na pniu Apache Mahout. Ten punkt rewizji jest po wydaniu 0.9.0, ale przed wydaniem 0.10.0. Zapewnia to dużą liczbę poprawek błędów i ulepszeń funkcjonalnych w wersji 0.9.0, ale zapewnia stabilne wydanie funkcji Mahout przed całkowitą konwersją do nowego mahoutu opartego na platformie Spark w wersji 0.10.0.

Punkt rewizji wybrany dla Mahout w HDP 2.3.x i 2.4.x pochodzi z gałęzi "mahout-0.10.x" Apache Mahout, od 19 grudnia 2014 r., wersja 0f037cb03e77c096 w GitHub.

W HDP-2.5.x i 2.6.x usunęliśmy bibliotekę "commons-httpclient" z Mahout, ponieważ postrzegamy ją jako przestarzałą bibliotekę z możliwymi problemami z zabezpieczeniami, i uaktualniliśmy Hadoop-Client w Mahout do wersji 2.7.3, tej samej wersji używanej w HDP-2.5. W efekcie:

-   Wcześniej skompilowane zadania Mahout będą musiały zostać ponownie skompilowane w środowisku HDP-2.5 lub 2.6.

-   Istnieje niewielkie prawdopodobieństwo, że niektóre zadania Mahout mogą napotkać błędy "ClassNotFoundException" lub "nie można załadować klasy" związane z "org.apache.commons.httpclient", "net.java.dev.jets3t" lub powiązane prefiksy nazw klas. Jeśli te błędy się zdarzy, można rozważyć, czy ręcznie zainstalować potrzebne słoiki w ścieżce klasy dla zadania, jeśli ryzyko problemów z zabezpieczeniami w przestarzałej bibliotece jest dopuszczalne w twoim środowisku.

-   Istnieje jeszcze mniejsze prawdopodobieństwo, że niektóre zadania Mahout może wystąpić awarie w hbase-client mahout wywołania kodu do bibliotek hadoop-common, ze względu na problemy ze zgodnością binarnych. Niestety, nie ma sposobu, aby rozwiązać ten problem, z wyjątkiem powrotu do wersji HDP-2.4.2 Mahout, które mogą mieć problemy z bezpieczeństwem. Ponownie, powinno to być bardzo nietypowe, i jest mało prawdopodobne, aby wystąpić w danym mahout pakietu pracy.

      1.  Dodaj właściwości ranger.tomcat.ciphers w sekcji ranger-admin-site w konfiguracji Ambari Ranger z poniższą wartością:
            
          SSL\_\_RSA\_Z\_RC4\_128 MD5, SSL\_RSA\_Z\_RC4\_128\_\_\_\_\_\_\_\_\_\_SHA, TLS\_RSA\_Z\_AES 128 CBC SHA, SSL RSA Z 3DES EDE CBC SHA
        
      2.  Jeśli środowisko jest skonfigurowane dla Ranger-KMS, dodaj właściwości ranger.tomcat.ciphers w sekcji theranger-kms-site w konfiguracji Ambari Ranger z poniższą wartością:
            
          SSL\_\_RSA\_Z\_RC4\_128 MD5, SSL\_RSA\_Z\_RC4\_128\_\_\_\_\_\_\_\_\_\_SHA, TLS\_RSA\_Z\_AES 128 CBC SHA, SSL RSA Z 3DES EDE CBC SHA

- **RangerUI: Escape of policy condition text wprowadzony w formularzu zasad**
    
  **Dotyczy składnika:** Ranger
    
  **Opis problemu**
    
  Jeśli użytkownik chce utworzyć zasady z niestandardowymi warunkami zasad, a wyrażenie lub tekst zawiera znaki specjalne, wymuszanie zasad nie będzie działać. Znaki specjalne są konwertowane na ASCII przed zapisaniem zasad w bazie danych.
    
  **Znaki specjalne:**  &  &lt; &gt; " " \`
    
  Na przykład warunek tags.attributes\['type'\]='abc' zostanie przekonwertowany na następujące po zapisaniu zasad.
    
  tags.attds\[&\#x27;dsds&\#x27; \] \#x27;cssdfs&x27; =& \#
    
  Warunek zasad z tymi znakami można zobaczyć, otwierając zasady w trybie edycji.

      **Przykład:**
        
      ```bash
        curl -H "Content-Type: application/json" -X POST http://localhost:6080/service/plugins/policies -u admin:admin -d '{"policyType":"0","name":"P100","isEnabled":true,"isAuditEnabled":true,"description":"","resources":{"tag":{"values":["tags-test"],"isRecursive":"","isExcludes":false}},"policyItems":[{"groups":["public"],"conditions":[{"type":"accessed-after-expiry","values":[]},{"type":"tag-expression","values":["tags.attr['type']=='abc'"]}],"accesses":[{"type":"hive:select","isAllowed":true},{"type":"hive:update","isAllowed":true},{"type":"hive:create","isAllowed":true},{"type":"hive:drop","isAllowed":true},{"type":"hive:alter","isAllowed":true},{"type":"hive:index","isAllowed":true},{"type":"hive:lock","isAllowed":true},{"type":"hive:all","isAllowed":true}]}],"denyPolicyItems":[],"allowExceptions":[],"denyExceptions":[],"service":"tagdev"}'
      ```

### <a name="upgrading"></a>Uaktualnianie

Wszystkie te funkcje są dostępne w hdinsight 3.6. Aby uzyskać najnowszą wersję programu Spark, Kafka i R Server (Machine Learning Services), wybierz wersję Spark, Kafka, ML Services podczas [tworzenia klastra HDInsight 3.6](https://docs.microsoft.com/azure/hdinsight/hdinsight-hadoop-provision-linux-clusters). Aby uzyskać obsługę usługi ADLS, można wybrać typ pamięci masowej ADLS jako opcję. Istniejące klastry nie zostaną automatycznie uaktualnione do tych wersji.

Wszystkie nowe klastry utworzone po czerwcu 2018 r. automatycznie otrzymają ponad 1000 poprawek błędów we wszystkich projektach typu open source. Postępuj zgodnie z [tym](https://docs.microsoft.com/azure/hdinsight/hdinsight-upgrade-cluster) przewodnikiem, aby uzyskać najlepsze rozwiązania dotyczące uaktualniania do nowszej wersji HDInsight.
