# <a name="a-practical-guide-to-designing-secure-health-care-solutions-in-azure"></a>Praktyczny przewodnik dotyczący projektowania bezpiecznych rozwiązań opieki zdrowotnej na platformie Azure
Uruchomienia branżowe, Integratory systemów (SIs), niezależni dostawcy oprogramowania (ISV) i organizacje opieki zdrowotnej, biorąc pod uwagę przejście na platformę Azure, poszukują wskazówek, które ułatwiają uwzględnienie kontroli bezpieczeństwa w celu spełnienia wymagań dotyczących zgodności.

[Praktyczny przewodnik projektowania bezpiecznych rozwiązań opieki zdrowotnej w Microsoft Azure](https://aka.ms/azureindustrysecurity) pomaga zrozumieć, jak można poprawić zabezpieczenia rozwiązań za pomocą usług i funkcji platformy Azure, które można skonfigurować w zależności od wymagań.
Zawartość jest dzielona na trzy główne sekcje:

1. Wskazówki dotyczące korzystania z technologii chmurowej, w tym zarządzania ryzykiem, współdzielonej odpowiedzialności, ustanawiania systemu zarządzania zabezpieczeniami informacji, rozumienia branżowych i lokalnych przepisów oraz ustanawiania standardowych procedur operacyjnych.
2. Najważniejsze zasady zabezpieczeń, które są wyrównane do standardowego standardu zarządzania zabezpieczeniami informacji, takie jak ISO 27001, i standardowe procesy programistyczne, takie jak Microsoft Security Development (SDL).
3. Zastosowanie najważniejszych zasad do użycia w celu demonstrowania wyrównania z perspektywy architekta rozwiązań, gdzie wymagania dotyczące rozwiązań są wyrównane do standardu zarządzania zabezpieczeniami informacji.

Mamy nadzieję, że znajdziesz [praktyczny przewodnik dotyczący projektowania bezpiecznych rozwiązań opieki zdrowotnej](https://aka.ms/azureindustrysecurity) , a jeśli masz pytania lub sugestie, poinformuj nas, pozostawiając poniższy komentarz.
