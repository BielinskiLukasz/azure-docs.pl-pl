# <a name="comparing-appsource-and-the-azure-marketplace"></a>Porównanie usługi AppSource i portalu Azure Marketplace
Sklepów w [portalu Azure Marketplace](https://azuremarketplace.microsoft.com) i [Microsoft AppSource](https://appsource.microsoft.com) obsługi unikatowych wymagań klientów. Użyj sklepów na skierowane do klientów przez rolę. Możesz zaoferować odpowiednich rozwiązań lub usługą opartą na klienta.

## <a name="understanding-the-differences-between-storefronts"></a>Opis różnic między sklepów
Aby wybrać storefront, Rozpocznij od zidentyfikowania Docelowi odbiorcy oferty. Jeśli zostaną objęci odbiorcom w wielu sklepów, możesz opublikować jeden raz do sprzedaży w wielu sklepów.
 
W poniższej tabeli opisano zalety używania Sklepu:

| Korzyści | AppSource | Azure Marketplace |
|:--- |:--- | :--- |
| **Elastyczność rozliczeń** | Aprowizuje wersję próbną interfejsu. Obecnie nie oferuje opcji publikowania z włączoną commerce. Można użyć używanych bieżącego porządkowanie i rozliczeń infrastruktury bez dodatkowej inwestycji ani zmian. | W przypadku maszyn wirtualnych opcje rozliczania płatność za rzeczywiste użycie Użyj umowy Microsoft Enterprise Agreement (EA) lub bezpośredni modele sprzedaży w sieci web. Opcje cenowe obejmują subskrypcji warstwy bezpłatna, w jakiej oferta jest bezpłatny bezterminowo. Opcje cenowe obejmują również subskrypcji Wypróbuj teraz. Wypróbuj teraz subskrypcję jest bezpłatna w promocji przez ograniczony okres. Po zakończeniu okresu, maszyna wirtualna jest konwertowana na płatną subskrypcję. Bring Your Own License (BYOL) aktywacji jest również opcja, która umożliwia klientom.<br /><br/>Opcje rozliczania i następujących ofert maszyn wirtualnych aprowizowane zasoby platformy Azure są rozliczane bezpośrednio do klienta:<ul> <li>Aplikacje platformy Azure: aplikacja zarządzana</li> <li>Aplikacje platformy Azure: szablon rozwiązania</li> </ul> |
| **Połączenia z innymi partnerami** | Linki niezależnych dostawców oprogramowania, integratorzy systemów i dostawcom usług zarządzanych na potrzeby scenariuszy obejmujących konkretnej implementacji. Użytkownik może wspólnie Sprzedaj ją nowym klientom. | Obecnie nie można połączyć partnerów usług dostawcy lub dostarczania oferty. | 
| **Automatyzacja** | Linki niezależnych dostawców oprogramowania, integratorzy systemów i dostawcom usług zarządzanych na potrzeby scenariuszy obejmujących konkretnej implementacji. Użytkownik może wspólnie Sprzedaj ją nowym klientom. | Obecnie nie można połączyć partnerów usług dostawcy lub dostarczania oferty. Wykorzystaj automatycznych oprogramowania jako usługa (SaaS) z dodatku inicjowania obsługi administracyjnej. | **Wiele typów chmury** | Obecnie nie obsługuje usługi Azure Stack, Azure dla instytucji rządowych lub regionalnych chmury. | Publikowanie rozwiązań dla chmury publicznej i lokalnych przy użyciu następujących typów:<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>Regionalne chmury, w tym Chiny i Niemcy</li></ul> |
| **W kontekście prezentacji dla klientów** | Dotrzyj do wielu klientów za pośrednictwem środowiska aplikacji dla produktów firmy Microsoft, takich jak usługi Office 365, Dynamics 365 i usługi Power BI. | Udostępnić swoje rozwiązanie w środowisku platformy Azure w portalu dla wyszukiwania kontekstowych. Korzystanie z aplikacji i maszyn wirtualnych platformy Azure: szablon rozwiązania, opcje publikowania. |
