# <a name="billing-options"></a>Karta Opcje  

## <a name="commercial-considerations-on-the-marketplace"></a>Zagadnienia komercyjnych w witrynie marketplace  
Portalu marketplace nie udostępnia przychodu dla następujących typów listy.  
*   List  
*   Wersja próbna  
*   Transact przy użyciu modelu rozliczeń Bring Your Own License (BYOL)

Nie jesteś rachunku dodatkowe opłaty za udział w sklepy w witrynie marketplace.  
*   Aby uzyskać więcej informacji, odwiedź stronę Zasady uczestnictwo w programie Azure Marketplace strony znajduje się w [azure.microsoft.com/support/legal/marketplace/participation-policies](https://azure.microsoft.com/support/legal/marketplace/participation-policies).  

## <a name="pay-as-you-go-and-bring-your-own-license-billing-options"></a>Płatność za rzeczywiste użycie, a przełączyć własnych opcji rozliczeń licencji  
Jeśli korzystasz z modelu rozliczeń jako opcji publikowania, jest przychodu licencjonowania oprogramowania opartej na użyciu udostępnionych 80/20 między Tobą a firmą Microsoft. Jednej oferty może być cenach przy użyciu zarówno płatność za rzeczywiste użycie, jak i BYOL modelu rozliczeń. Dwa modele rozliczeń współistnieć na poziomie oferta jako osobne jednostki SKU. Skonfigurujesz modeli rozliczeń w ramach Twojej oferty w portalu dla partnerów chmury. 

Rozważ następujące przykłady.  
*   Po włączeniu opcji Płatność za rzeczywiste użycie, masz następujące wyniki.  
    | Bieżąca licencja koszt | 1,00 na godzinę |  
    |:--- |:--- |  
    | Koszt użycia platformy Azure (D1 1-rdzeni) | $0.14 na godzinę |  
    | *Jest on rozliczany klienta przez firmę Microsoft* | *$1.14 na godzinę* |  

    W tym scenariuszu Microsoft rachunków $1.14 na godzinę do użycia opublikowanego obrazu maszyny Wirtualnej.  
    | Opłaty firmy Microsoft | $1.14 na godzinę |  
    |:--- |:--- |  
    | Microsoft płaci 80% koszt licencji | $0,80 na godzinę |  
    | Microsoft zachowuje 20% koszt licencji | $0,20 na godzinę |  
    | Microsoft zachowuje koszt użycia platformy Azure | $0.14 na godzinę |  

*   Po włączeniu opcji BYOL, masz następujące wyniki.  
    | Bieżąca licencja koszt | Opłata licencji negocjował i rozliczony przez użytkownika |  
    |:--- |:--- |  
    | Koszt użycia platformy Azure (D1 1-rdzeni) | $0.14 na godzinę |  
    | *Jest on rozliczany klienta przez firmę Microsoft* | *$0.14 na godzinę* |  

    W tym scenariuszu Microsoft rachunków $0.14 na godzinę do użycia opublikowanego obrazu maszyny Wirtualnej.  
    | Opłaty firmy Microsoft | $0.14 na godzinę |  
    |:--- |:--- |  
    | Microsoft zachowuje koszt użycia platformy Azure | $0.14 na godzinę |  
    | Microsoft zachowuje 0% koszt licencji | 0,00 USD na godzinę |  

## <a name="single-billing-and-payment-methods"></a>Jednym rozliczeń i formy płatności  
Ważne zaletą używania Transact listę opcji jest, że koszty licencjonowania i użycia platformy Azure są rozliczane jednym bezpośrednio do klienta.  

W tym scenariuszu firma Microsoft rachunków i zbiera w Twoim imieniu. Microsoft rozliczeń eliminuje konieczność tworzenia własnych nabywania relacji z klientem. Jednym rozliczeń może zaoszczędzić czas i zasoby. Jednym rozliczeń mogą pomóc skupić się na lądowanie sprzedaży, zamiast zbieranie BOM. 

## <a name="enterprise-agreement"></a>Umowa Enterprise  
Jeśli klient Enterprise Agreement (EA) z firmą Microsoft, może używać administrator przedsiębiorstwa w Twojej płatności dla produktów firmy Microsoft. Może obciążyć produktów w tym użycie platformy Azure. Za pomocą programu EA zapłacenia jest przeznaczona dla organizacji, które mają być licencji oprogramowania i usług, co najmniej trzech lat w chmurze. Rozłożenia płatności zamiast jedną wprowadzania początkowych płatności. Jeśli używasz płatność za rzeczywiste użycie transact typ oferty, następnie rozliczeń koszty licencjonowania oprogramowania następuje co kwartał EA nadwyżkowe cyklu rozliczeniowego.  

### <a name="monetary-commitment"></a>Zobowiązanie pieniężne  
W przypadku klienta EA może dodać Azure z umową. Azure należy dodać do Umowy, wprowadzając góry zobowiązaniom pieniężnym na platformie Azure. Twoje zobowiązania pieniężnego jest używany przez cały rok. Twojej zawiera dowolną kombinację użycia szerokiej gamy usług Azure.
