## <a name="set-up-authentication"></a>Konfigurowanie uwierzytelniania

Aby uzyskać dostęp do usługi, należy podać klucz konta, token dostępu lub token y aktu usługi Azure Active Directory. Więcej informacji na ten temat można również przeczytać na [stronie koncepcji uwierzytelniania](/azure/spatial-anchors/concepts/authentication).

### <a name="account-keys"></a>Klucze konta

Klucze konta to poświadczenia, które umożliwiają aplikacji uwierzytelnienie za pomocą usługi Azure Spatial Anchors. Celem kluczy konta jest pomoc w szybkim rozpoczęciu pracy. Szczególnie w fazie rozwoju integracji aplikacji z kotwicami przestrzennymi platformy Azure. W związku z tym można użyć kluczy konta, osadzając je w aplikacjach klienckich podczas tworzenia. W miarę postępów poza programem zdecydowanie zaleca się przejście do mechanizmu uwierzytelniania, który jest na poziomie produkcyjnym, obsługiwany przez tokeny dostępu lub uwierzytelnianie użytkowników usługi Azure Active Directory. Aby uzyskać klucz konta do tworzenia, odwiedź konto Azure Spatial Anchors i przejdź do karty "Klucze".
