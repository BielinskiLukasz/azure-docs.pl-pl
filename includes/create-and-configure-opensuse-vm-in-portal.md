2. ウィンドウの下部にあるコマンド バー] をクリックして **新規**します。

3. [ **コンピューティング**, 、] をクリックして **仮想マシン**, 、] をクリックし、 **ギャラリーから**します。

    ![新しい仮想マシンの作成][Image1]

4. 下にある、 **SUSE** をグループ化から OpenSUSE 仮想マシン イメージを選択して、続行する矢印をクリックします。

5. 最初の **仮想マシンの構成** ページ。

    - 型 a **仮想マシン名**, 、"testlinuxvm"などです。
    - 確認、 **層** を選択し、 **サイズ**します。 階層により選択するサイズが決まります。 仮想マシンのサイズによって、使用料金や、構成オプション (接続できるデータ ディスクの数など) が変わります。 詳細については、「 [仮想マシンのサイズ](../articles/virtual-machines-size-specs.md)します。
    - 型 a **新しいユーザー名**, 、か、既定 **azureuser**します。 この名前が、Sudoers リスト ファイルに追加されます。
    - 種類を決定 **認証** を使用します。 一般的なパスワードのガイドラインについては、次を参照してください。 [強力なパスワード](http://msdn.microsoft.com/library/ms161962.aspx)します。

6. 次の **仮想マシンの構成** ページ。

    - 既定値を使用して **新しいクラウド サービスを作成する**です。
    -  **DNS 名** ボックスに、"testlinuxvm"などのアドレスの一部として使用する有効な DNS 名を入力します。
    -  **リージョン/アフィニティ グループ/仮想ネットワーク** ボックスで、この仮想イメージをホストするリージョンを選択します。
    -  **エンドポイント**, 、SSH エンドポイントを保持します。 この時点で他のエンドポイントを追加できます。また、仮想マシンの作成後にエンドポイントを追加、変更、削除できます。

    >[AZURE.NOTE] 仮想ネットワークを使用する仮想マシンの場合、 **必要があります** 仮想マシンを作成するときに、仮想ネットワークを指定します。 作成後に仮想マシンを仮想ネットワークに参加させることはできません。 詳細については、次を参照してください。 [仮想ネットワークの概要](virtual-networks-overview.md)します。

7.  最後の **仮想マシンの構成** ] ページで、既定の設定を保持して、[完了] にチェック マークをクリックします。

ポータルで新しい仮想マシンを一覧表示 **仮想マシン**します。 状態を報告 **(プロビジョニング)**, 、仮想マシンをセットアップします。 として状態を報告するときに **を実行している**, 、次の手順に進むことができます。

##仮想マシンへの接続

接続元コンピューターのオペレーティング システムに応じて、SSH または PuTTY を使用して仮想マシンと接続します。

- Linux を実行しているコンピューターでは、SSH を使用します。 コマンド プロンプトに、次のコマンドを入力します。

    `$ ssh newuser@testlinuxvm.cloudapp.net -o ServerAliveInterval=180`

    ユーザーのパスワードを入力します。

- Windows を実行しているコンピューターでは、PuTTY を使用します。 インストールされていない場合からダウンロード、 [PuTTY のダウンロード ページ][PuTTYDownload]します。

    保存 **putty.exe** お使いのコンピューター上のディレクトリにします。 コマンド プロンプトを開き、そのフォルダーに移動し、実行 **putty.exe**します。

    "Testlinuxvm.cloudapp.net"などのホスト名を入力しの「22」を入力、 **ポート**します。

    ![PuTTY の画面][Image6]  

##仮想マシンの更新 (オプション)

1. 仮想マシンに接続したら、必要に応じてシステムの更新プログラムとパッチをインストールできます。 更新プログラムを実行するには、次のように入力します。

    `$ sudo zypper update`

2. 選択 **ソフトウェア**, 、し **オンライン更新** を利用可能な更新を一覧表示します。 選択 **Accept** インストールを開始し、(省略可能な) を除くすべての新しい利用可能なパッチを適用します。

3. インストールが完了したら選択 **完了**します。  これでシステムが最新の状態になりました。
