# Azure への組織としてのサインアップ

最近まで、新しい Microsoft Azure サブスクリプションにサインアップするには、必ず Microsoft アカウント (Windows Live ID) を使用する必要がありました。 現在、Azure では、次に示す 2 つのアカウントのいずれかを使用してサインアップすることができます。

* **Microsoft アカウント** (で作成する個人向け) - Outlook (Hotmail)、Messenger、OneDrive、MSN、Xbox LIVE、または Office 365 などのサービスをクラウドとすべてのコンシューマー向け Microsoft 製品にアクセスします。 Outlook.com のメールボックスにサインアップすると、Microsoft アカウントが自動的に作成されます。 作成した Microsoft アカウントは、コンシューマーに関連した Microsoft クラウド サービスまたは Azure へのアクセスに使用できます。 [詳細情報](http://www.microsoft.com/account/default.aspx)

* **職場または学校アカウント** (企業または教育機関用に管理者が発行) - Azure、Microsoft Intune、Office 365 などのビジネス レベル Microsoft クラウド サービスのすべての小規模、中規模、およびエンタープライズへのアクセスを提供します。 このサービスのいずれかに組織としてサインアップすると、クラウド ベースのディレクトリが Azure Active Directory で自動的にプロビジョニングされ、組織が表示されます。 [詳細情報](active-directory-administer.md)

    このディレクトリが作成されたら、管理者がユーザーを作成し、それぞれがアクセスする必要があるクラウド サービス (Azure など) のサブスクリプションに基づいて各ユーザーにライセンスを付与できます。

組織として Azure にサインアップする必要がある場合は、 [**今すぐサインアップ**](https://azure.microsoft.com/pricing/purchase-options/)
