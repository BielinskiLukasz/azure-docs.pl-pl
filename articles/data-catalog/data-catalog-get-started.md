# Azure Data Catalog の概要

この記事は、Azure Data Catalog パブリック プレビューのシナリオと機能のエンド ツー エンドのチュートリアルです。 プレビューにサインアップしたら、以下の手順に従って、データ カタログを作成し、データソースの登録、注釈の設定、検出を行うことができます。

## チュートリアルの前提条件

このチュートリアルを始める前に、次の項目を用意する必要があります。

-   **Azure サブスクリプション** -、サブスクリプションを持っていない場合は、ほんの数分で無料の試用アカウントを作成することができます。 参照してください、 [無料評価版](http://azure.microsoft.com/pricing/free-trial/) の詳細については、資料です。
-   **Azure Active Directory** -Azure Data Catalog を使用して [Azure Active Directory](http://azure.microsoft.com/services/active-directory/) id およびアクセス管理します。
-   **データ ソース** - Azure Data Catalog は、データ ソース検出機能を提供し、チュートリアルを進めるには 1 つまたは複数のデータ ソースにアクセス権が必要です。 このチュートリアルは、Adventure Works サンプル データベースを使用して作成されていますが、馴染みがあり、自分のロールに関連するデータを操作したいと考える場合は、任意のサポートされているデータ ソースを使用できます。

この演習では、SQL Server Database Engine の Adventure Works サンプルと SQL Server Analysis Services Multidimensional をインストールします。 これらのサンプルは、その後の演習で使用します。

次に、Adventure Works をインストールする手順を示します。

### Adventure Works 2014 OLTP およびデータ ウェアハウス データベースをインストールします。

データベースは、 [CodePlex.com](http://msftdbprodsamples.codeplex.com/) し、次の手順に従って、インストールされていることができます、 [Adventure Works 2014 サンプル データベースの Readme](https://msftdbprodsamples.codeplex.com/downloads/get/880669)します。

この演習では、残りの演習で使用される Adventure Works サンプル データベースをインストールしました。 この演習をスキップし、独自のエンタープライズ データ ソースを使用するように選択した場合は、名前、タグ、およびその他のメタデータを覚えておくように準備しておいてください。

    ![register3][3]

## 演習 4: 登録されているデータ ソースへの注釈の設定

この演習では使用して、 **Azure Data Catalog** カタログに以前に登録されたデータ資産に注釈を付けるへの入り口です。 指定する注釈は、登録時に、データ ソースから抽出された構造型メタデータを補完し、拡張して、データ資産をさらに検出および理解しやすくします。 各 **Data Catalog** ユーザーは独自の注釈を指定できるすべてのユーザーを共有するデータの観点を持つは簡単にします。

### 次に、データ資産に注釈を設定する方法を示します

1.  ログイン、 **Azure Data Catalog** ポータルです。
2.  クリックして **検出**します。
3.  1 つ以上選択 **データ資産**します。 この例では次のように選択します **ProductPhoto**, 、「Product photos for marketing materials。"を入力し。
4.  入力、 **説明** を他の検出を支援され理由を理解して、選択したデータ資産を使用する方法です。 たとえば、「Product images」と入力します。 さらに、タグとビュー列を追加することもできます。
5.  これで、カタログに追加したわかりやすいメタデータを使用して、データ資産を検出するための検索とフィルター処理を試行できます。

    ![注釈][9]

この演習では、登録されているデータ資産にわかりやすい情報を追加して、カタログ ユーザーが理解できる用語を使用して、データ ソースを検出できるようにしました。

## 演習 5: メタデータのクラウドソーシング

この演習では、別のユーザーと協力して、カタログ内のデータ資産にメタデータを追加します。 Azure Data Catalog の注釈のクラウドソーシング アプローチにより、すべてのユーザーがタグ、説明、およびその他のメタデータを追加できるため、データ資産とその使用に関して、ある観点を持つすべてのユーザーが、その観点を他のユーザーからキャプチャし、使用してもらうことができます。

> [AZURE.NOTE] このチュートリアルで使用する別のユーザーを取得していない場合は、心配しないでください。 データ カタログにアクセスするすべてのユーザーが、選択に応じて、独自の観点を追加できます。 このメタデータのクラウドソーシング アプローチにより、カタログの内容と、カタログのメタデータの豊かさは時間とともに増加します。

### 次に、データ資産に関するメタデータをクラウドソーシングする方法を示します

オーディオの録音を繰り返す、 **登録されているデータ ソースの注釈を付ける** 上記の手順です。 同僚が ProductPhoto など、データ資産に説明を追加すると、複数の注釈が表示されます。

## 演習 6: データ ソースへの接続

この演習では使用して、 **Azure Data Catalog** ポータル Microsoft Excel を使用してデータ ソースに接続します。


> [AZURE.NOTE] 留意することが重要 **Azure Data Catalog** ユーザー アクセスを付与しないの実際のデータ ソースに、単に、ユーザーは簡単を検出し、理解しています。 ユーザーがデータ ソースに接続すると、選択したクライアント アプリケーションが、ユーザーの Windows 資格情報を使用するか、または必要に応じて、ユーザーに資格情報の入力を求めます。 ユーザーに以前にデータ ソースへのアクセス権が付与されていない場合、データ ソースに接続する前に、アクセス権を付与される必要があります。

### 次に、Excel からデータ ソースに接続する方法を示します

1.  ログイン、 **Azure Data Catalog** ポータルです。
2.  クリックして **検出**します。
3.  データ資産を選択します。 この例では ProductCategory を選択します。
4.  選択 **で開いている** > **Excel**します。

    ![connect1][10]
5.   **Microsoft Excel のセキュリティに関する通知** ウィンドウで、をクリックして **を有効にする**です。
6.  開いている、 **ProductCategory.odc** ファイルです。
7.  データ ソースが Excel で開きます。

    ![connect2][11]

この演習では、Azure Data Catalog を使用して検出されたデータ ソースに接続しました。  **Azure Data Catalog** ポータルではユーザーに統合されたクライアント アプリケーションを使用して直接接続するその **で開く.** メニューの [し、選択した接続の場所情報が資産のメタデータに含まれているすべてのアプリケーションを使用して接続することもできます。

## 演習 7: データ ソースのメタデータの削除

この演習では使用して、 **Azure Data Catalog** ポータル登録されているデータ資産からプレビュー データを削除して、カタログからデータ資産を削除します。

> [AZURE.NOTE] カタログの既定の動作では、すべてのユーザーを任意のデータ ソースを登録し、すべてのユーザーに登録されている任意のデータ資産の削除を許可するように許可です。 含まれる管理機能、 **標準エディションの Azure Data Catalog** 資産の所有権を制限する、資産を検出できるユーザーを取得および資産を削除できるユーザーを制限するための追加のオプションを提供します。

 **Azure Data Catalog**, 、個別の資産の削除からプレビューを除外したり、複数のアセットを削除します。

### 次に、複数のデータ資産を削除する方法を示します

1.  ログイン、 **Azure Data Catalog** ポータルです。
2.  クリックして **検出**します。
3.  1 つまたは複数のデータ資産を選択します。
4.  クリックして **削除**します。

この演習では、登録されているデータ資産をカタログから削除しました。

## 演習 8: 登録されているデータ ソースの管理

この演習では、管理機能を使用して、 **Azure Data Catalog** 検出およびそれらの資産を管理するユーザー コントロールをデータ資産の所有権を取得します。

注: この演習で説明している管理機能は、Free エディションではなく、Azure Data Catalog の Standard Edition でのみ使用できます。
 **Azure Data Catalog**, 、データ資産に共同所有者を追加するデータ資産の所有権を取得することができ、データ資産の可視性を設定します。

### 次に、データ資産の所有権を取得し、可視性を制限する方法を示します

1.  ログイン、 **Azure Data Catalog** ポータルです。
2.  クリックして **検出**します。
3.  1 つまたは複数のデータ資産を選択します。
4.   **プロパティ** パネル、 **管理** ] をクリックして **Take Ownership**します。
5.  可視性を制限するにはクリックして **所有者とこれらのユーザー**します。

    ![所有権][12]

この演習では、カタログの管理機能について学習し、選択したデータの資産の可視性を制限しました。
