# ストレージ エクスプローラー (プレビュー) を使用した Azure ストレージ リソースの管理

Microsoft Azure ストレージ エクスプローラー (プレビュー) は、Azure ストレージ アカウントを簡単に管理できるスタンドアロン ツールです。 Visual Studio でアプリを開発しているときなど、Azure ポータルの外部からストレージをスピーディに管理したい場合に便利です。 このプレビュー版では、Blob Storage を簡単に操作できます。 コンテナーの作成と削除、BLOB のアップロード、ダウンロード、および削除、すべてのコンテナーと BLOB に対する検索を実行できます。 高度な機能では、開発者や運用担当者が SAS キーとポリシーを操作できます。 Windows 開発者は、Azure ストレージ エミュレーターを使用して、ローカル開発ストレージ アカウントを用いたコード テストを実行することもできます。

ストレージ エクスプローラーでストレージ リソースの表示や管理を実行するには、サブスクリプションと外部ストレージ アカウントのいずれかで Azure ストレージ アカウントにアクセスできる必要があります。 ストレージ アカウントをお持ちでない場合は、数分でアカウントを作成できます。 MSDN サブスクリプションがある場合は、次を参照してください。 [サブスクライバー向けの Visual Studio の毎月の Azure クレジット](https://azure.microsoft.com/pricing/member-offers/msdn-benefits-details/)します。 そうしないと、 [無料評価版の 1 か月](https://azure.microsoft.com/pricing/free-trial/)します。

## Azure アカウントと Azure サブスクリプションの管理

ストレージ エクスプローラーで Azure ストレージ リソースを表示するには、1 つ以上の有効なサブスクリプションを持つ Azure アカウントにログインする必要があります。 複数の Azure アカウントがある場合は、ストレージ エクスプローラーでそれらのアカウントを追加し、ストレージ エクスプローラーのリソース ビューで追加するサブスクリプションを選択します。 これまでに Azure を使用したことがない場合、または必要なアカウントを Visual Studio に追加していない場合は、Azure アカウントにログインするように求められます。

### Azure アカウントをストレージ エクスプローラーに追加する

1.  選択、 **設定** 記憶域エクスプ ローラーのツールバー (歯車) アイコン。
1.  選択、 **アカウントを追加する** リンクします。 参照するストレージ リソースを所有する Azure アカウントにログインします。 ここで追加したアカウントは、アカウントを選択するドロップダウン リストで選択する必要があります。 選択したアカウントのすべてのサブスクリプションが、アカウント項目の下に表示されます。

    ![][0]

1.  参照し、選択するアカウントのサブスクリプションのチェック ボックスをオン、 **適用** ] ボタンをクリックします。 

    ![][1]

    選択したサブスクリプションの Azure ストレージ リソースがストレージ エクスプローラーに表示されます。

### 外部ストレージをアタッチする

1. アタッチするストレージ アカウントのアカウント名とキーを取得します。
    1.  Azure プレビュー ポータルで、アタッチするストレージ アカウントを選択します。
    1.   **管理** のセクションで、 **設定** Azure プレビュー ポータルでのウィンドウの選択、 **キー** ] ボタンをクリックします。
    1.  コピー、 **ストレージ アカウント名** と **プライマリ アクセス キー** 値。

        ![][2]

1.  ショートカット メニューを開き、 **ストレージ アカウント** 記憶域エクスプ ローラーでノードを選択、 **外部の記憶域をアタッチ** コマンドです。

    ![][3]

1. [ストレージ アカウント名を入力、 **アカウント名** ボックスと [プライマリ アクセス キー、 **アカウント キー** ボックス。 選択、 **Ok** 続行する] ボタンをクリックします。

    ![][4]

    外部ストレージが、ストレージ エクスプローラーに表示されます。 

    ![][5]

1. 外部ストレージを削除するには、外部ストレージのショートカット メニューの [デタッチ] をクリックします。

    ![][6]

## ストレージ リソースの表示と移動

ストレージ エクスプローラーで Azure ストレージ リソースに移動してその情報を表示するには、ストレージの種類を展開して、リソースを選択します。 表示される選択されたリソースに関する情報、 **アクション** と **プロパティ** 記憶域エクスプ ローラーの下部にあるタブです。

![][7]

-    **アクション** ] タブは選択した記憶域リソースを開く、コピー、または削除するなどの記憶域エクスプ ローラーで実行できるアクションを示します。 アクションは、リソースのショートカット メニューにも表示されます。

-    **プロパティ** ] タブは、型、ロケール、関連付けられたリソース グループおよび URL など、記憶域リソースのプロパティを表示します。

すべてのストレージ アカウントは、アクションを備える **ポータルで開く**します。 このアクションを選択すると、ストレージ エクスプローラーで選択したストレージ アカウントが、Azure プレビュー ポータルで表示されます。

その他のアクションとプロパティの値も、選択したリソースに基づいて表示されます。 たとえば、すべての Blob コンテナー、キュー、およびテーブルのノードがある、 **作成** アクション。 個々 の項目 (blob コンテナー) などのアクションはなどが含ま **開く**, 、**削除**, 、および **共有アクセス署名の取得**します。 BLOB エディターを開くアクションは、ストレージ アカウント BLOB を選択すると表示されます。

## ストレージ アカウントと BLOB コンテナーの検索

ストレージ アカウントを検索し、blob、Azure アカウントのサブスクリプションで特定の名前のコンテナー、内の名前を入力してください、 **検索** 記憶域エクスプ ローラーでボックスします。

![][8]

内の文字を入力すると、 **検索** ボックスで、ストレージ アカウントについてのみ、またはリソースのツリーでこれらの文字に一致する名前を持つ blob コンテナーが表示されます。 検索をクリアするには、選択、 **x** ボタンをクリックして、 **検索** ボックス。

## ストレージ アカウントの編集

ストレージ アカウントの内容を変更または追加を選択、 **エディターを開く** コマンドでは、記憶域の種類。 または選択された項目のショートカット メニューの [アクションは、いずれかを選択、 **アクション** 記憶域エクスプ ローラーの下部にあるタブをクリックします。

![][9]

BLOB コンテナー、キュー、テーブルの作成と削除を実行できます。 選択して、記憶域エクスプ ローラー内の blob を編集することも、 **Blob コンテナー エディターを開く** アクション。

### BLOB コンテナーを削除する

1.  選択、 **Blob コンテナー エディターを開く** アクション。 BLOB コンテナー エディターが右側のペインに表示されます。

    ![][10]

1.  選択、 **アップロード** ボタンをクリックしを選択し、 **ファイルのアップロード** コマンドです。

    ![][11]

    アップロードする複数のファイルが 1 つのフォルダー内にある場合は、[フォルダーのアップロード] を使用できます。

1.  **ファイルをアップロード** ] ダイアログ ボックスで、省略記号ボタンを選択 (**...**) をアップロードするファイルを選択する [ファイル] ボックスの右側にあるボタンをクリックします。 次に、アップロードする BLOB の種類 (ブロック、ページ、または追加) を選択します。 必要に応じて、BLOB コンテナー内のフォルダーにファイルをアップロードすることもできます。 フォルダーの名前を入力、 **フォルダー (省略可能) にアップロード** ボックス。 フォルダーが存在しない場合は、作成されます。

    ![][12]

    という新しいフォルダーに次のスクリーン ショットでは、次の 3 つのイメージ ファイルをアップロードした **マイの新しいファイル** で、 **イメージ** blob コンテナーです。

    ![][13]

    BLOB エディターのツール バーのボタンを使用して、ファイルの選択、ダウンロード、開く、コピー、削除などの操作を実行できます。  **アクティビティ** 、ダイアログ ボックスの下部のペインに、操作が成功しから見ると、成功のアクティビティのみを削除するか、まったくペインを消去することができるかどうかを示しています。 選択、 **+** の詳細な一覧を表示、アップロードしたファイルの横にあるアイコンは、ファイルをアップロードします。

## Shared Access Signature (SAS) の作成

一部の操作では、ストレージ リソースにアクセスするために SAS が必要になることがあります。 SAS はストレージ エクスプローラーを使用して作成できます。

1.  SAS を作成し、[項目を選択、 **共有アクセス署名の取得** コマンドを **アクション** ウィンドウまたは項目のショートカット メニュー。

    ![][14]

1.   **Shared Access Signature** ] ダイアログ ボックスでは、ポリシー、開始と終了日を選択し、タイム ゾーン。 また、読み取り専用、読み取り/書き込みなど、リソースに必要なアクセス レベルのチェック ボックスをオンにします。完了したら、選択、 **作成** クリックすると、SAS を作成します。

    ![][15]

1.   **Shared Access Signature** ] ダイアログ ボックスは、URL とストレージ リソースへのアクセスに使用することができますクエリー文字列と共にコンテナを一覧表示します。 選択、 **コピー** 文字列をコピーする] ボタンをクリックします。

    ![][16]

## SAS とアクセス許可の管理

Blob コンテナーへのアクセスを制御することができます、 **アクセス制御リストの管理** と **パブリック アクセス レベルの設定** コマンドです。 

-   [アクセス制御リストの管理] を使用すると、選択した BLOB コンテナーに対するアクセス ポリシー (ユーザーに読み取りや書き込みなどを許可するかどうか) の追加、編集、削除を実行できます。
-   [パブリック アクセス レベルの設定] を使用すると、パブリック ユーザーにリソースへのアクセスをどの程度許可するかを決定できます。  

-

1.  Blob コンテナーを選択し、[、 **アクセス制御リストの管理** コマンドのショートカット メニューでも、 **アクション** ウィンドウです。

    ![][17]

1.   **アクセス制御リスト** ] ダイアログ ボックスで、選択、 **追加** アクセス ポリシーを追加する] ボタンをクリックします。 アクセス ポリシーを選択し、必要なアクセス許可を選択します。 完了したら、選択、 **保存** ] ボタンをクリックします。

    ![][18]

1.  Blob コンテナーのアクセス レベルを設定するには、ストレージ エクスプ ローラー] を選択してクリックして、 **パブリック アクセス レベルの設定** コマンドのショートカット メニューでも、 **アクション** ウィンドウです。

    ![][19]

1.   **コンテナーのパブリック アクセス レベルの設定** ] ダイアログ ボックスで、オプション ボタンを選択すると、パブリックのユーザーに付与するアクセスのレベルを選択、 **適用** ] ボタンをクリックします。

    ![][20]
