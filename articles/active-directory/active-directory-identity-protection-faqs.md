# <a name="azure-active-directory-identity-protection-faq"></a>Ochronę tożsamości usługi Azure Active Directory — często zadawane pytania


## <a name="why-do-some-risk-events-have-closed-system-status"></a>Dlaczego niektóre zdarzenia o podwyższonym ryzyku ma stanu "Closed (system)"

**Odpowiedź:** są to zdarzenia, które usługi Azure Active Directory Identity Protection wykryte i później zamknięta, ponieważ zdarzenia zostały już uważane za ryzykowne. Te zdarzenia nie wchodzą w skład poziom ryzyka użytkownika. 

---

## <a name="do-i-need-to-be-a-global-admin-to-use-identity-protection-in-the-azure-portal"></a>Czy muszę być administratorem globalnym, aby korzystać z funkcji ochrony tożsamości w portalu Azure?
**Odpowiedź:** **nr**. Może być czytnik zabezpieczeń administratora zabezpieczeń lub administrator globalny, do korzystania z ochrony tożsamości.

**Odpowiedź:** zobacz [wprowadzenie do korzystania z usługi Azure Active Directory Premium](active-directory-get-started-premium.md) dla odpowiedzi na to pytanie.
