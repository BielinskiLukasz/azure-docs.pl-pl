# <a name="azure-functions-and-serverless-platform-security"></a>Środowisko Azure Functions i zabezpieczeń niekorzystającą platformy
## <a name="abstract"></a>abstrakcyjny
Większość przedsiębiorstw muszą znaczną ilość zasobów i do zarządzania serwerami, czasu, który dodaje kosztów. Jeśli przedsiębiorstwa mogą korzystać z mniej zasobów, do zarządzania serwerami, można skoncentrować się na tworzeniu wspaniałych aplikacji.  

Obliczanie niekorzystającą pomaga spełniają właśnie tę funkcję ponieważ infrastrukturę, która należy do uruchamiania i skalowania aplikacji odbywa się. Operacje obliczeniowe bez użycia serwera to abstrakcja serwerów, infrastruktury i systemów operacyjnych. Niekorzystającą przetwarzania danych jest wymuszany w reakcji na zdarzenia i wyzwalaczy, które są wszystkie odbywa się w czasie rzeczywistym near — w chmurze. 

Jako w pełni zarządzaną usługę serwer zarządzania i Planowanie wydajności są niewidoczne dla dewelopera. Niekorzystającą framework ułatwia opracowywanie i wdrażanie aplikacji bez serwera za pomocą usługi Azure Functions. Jest interfejsu wiersza polecenia (CLI) oferuje struktury i automatyzacji do tworzenia zaawansowanych, sterowane zdarzeniami, niekorzystającą architektury składa się z funkcji i zdarzeń. Funkcja Azure jest niezależną jednostkę wdrożenia, takich jak mikrousługi. Jest tylko kod, wdrożonych w chmurze, które najczęściej są zapisywane do wykonania w jednym zadaniu.

Pomimo korzyści niekorzystającą zabezpieczeń ma własną czynniki ryzyka postępowania w przypadku. Podejście niekorzystającą nie wprowadzić nowe problemy z zabezpieczeniami, ale wymaga o podejście do istniejących problemy z zabezpieczeniami. Ten dokument koncentruje się na te kwestie bezpieczeństwa: 
* Korzyści niekorzystającą platformy
* Problemy z zabezpieczeniami w przypadku komputerów bez serwera
* Ważnych kwestiach dotyczących zabezpieczeń i środki zaradcze w kontekście systemu Azure
* Zabezpieczanie niekorzystającą platformy firmy Microsoft
