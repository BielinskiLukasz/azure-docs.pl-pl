# <a name="azure-sql-database-features"></a>Funkcje bazy danych SQL platformy Azure

Usługa Azure SQL Database udostępnia wspólną bazę kodu z najnowszą stabilną wersją programu SQL Server. Większość standardowych funkcji języka SQL, przetwarzania zapytań i zarządzania bazami danych jest identyczna w programach SQL Server i usłudze Azure SQL Database. Funkcje, które są wspólne w programie SQL Server i wszystkie smaki usługi Azure SQL Database są:
  - Funkcje języka - [Sterowanie słowami kluczowymi języka przepływu](https://docs.microsoft.com/sql/t-sql/language-elements/control-of-flow), kursory , [typy](https://docs.microsoft.com/sql/t-sql/language-elements/cursors-transact-sql) [danych](https://docs.microsoft.com/sql/t-sql/data-types/data-types-transact-sql), [instrukcje DML](https://docs.microsoft.com/sql/t-sql/queries/queries), [predykaty](https://docs.microsoft.com/sql/t-sql/queries/predicates), [numery sekwencji](https://docs.microsoft.com/sql/relational-databases/sequence-numbers/sequence-numbers), [procedury przechowywane](https://docs.microsoft.com/sql/relational-databases/stored-procedures/stored-procedures-database-engine)i [zmienne](https://docs.microsoft.com/sql/t-sql/language-elements/variables-transact-sql).
  - Funkcje bazy danych - [Automatyczne dostrajanie (wymuszanie planu),](https://docs.microsoft.com/sql/relational-databases/automatic-tuning/automatic-tuning) [śledzenie zmian,](https://docs.microsoft.com/sql/relational-databases/track-changes/about-change-tracking-sql-server) [sortowanie bazy danych,](https://docs.microsoft.com/sql/relational-databases/collations/set-or-change-the-database-collation)zawarte bazy danych, zawarte [użytkownicy,](https://docs.microsoft.com/sql/relational-databases/databases/contained-databases) [kompresja danych,](https://docs.microsoft.com/sql/relational-databases/data-compression/data-compression)ustawienia [konfiguracji bazy danych,](https://docs.microsoft.com/sql/t-sql/statements/alter-database-scoped-configuration-transact-sql) [operacje indeksu online,](https://docs.microsoft.com/sql/relational-databases/indexes/perform-index-operations-online) [partycjonowanie](https://docs.microsoft.com/sql/relational-databases/partitions/partitioned-tables-and-indexes)i [tabele czasowe](https://docs.microsoft.com/sql/relational-databases/tables/temporal-tables) [(patrz przewodnik wprowadzenie).](sql-database-temporal-tables.md) [Contained users](https://docs.microsoft.com/sql/relational-databases/security/contained-database-users-making-your-database-portable)
  - Funkcje zabezpieczeń — [role aplikacji](https://docs.microsoft.com/sql/relational-databases/security/authentication-access/application-roles), [Dynamiczne maskowanie danych](https://docs.microsoft.com/sql/relational-databases/security/dynamic-data-masking) [(zobacz przewodnik po rozpoczęciu](sql-database-dynamic-data-masking-get-started.md)pracy), [Zabezpieczenia na poziomie wiersza](https://docs.microsoft.com/sql/relational-databases/security/row-level-security)i Wykrywanie zagrożeń — zobacz przewodniki wprowadzania do pracy dla [pojedynczej bazy danych i pul elastycznych](sql-database-threat-detection.md) oraz [wystąpienie zarządzane](sql-database-managed-instance-threat-detection.md).
  - Możliwości wielu modeli — [przetwarzanie wykresów](https://docs.microsoft.com/sql/relational-databases/graphs/sql-graph-overview), [dane JSON](https://docs.microsoft.com/sql/relational-databases/json/json-data-sql-server) [(patrz przewodnik po rozpoczęciu](sql-database-json-features.md)pracy), [OPENXML](https://docs.microsoft.com/sql/t-sql/functions/openxml-transact-sql), [przestrzenne,](https://docs.microsoft.com/sql/relational-databases/spatial/spatial-data-sql-server) [OPENJSON](https://docs.microsoft.com/sql/t-sql/functions/openjson-transact-sql)i [indeksy XML](https://docs.microsoft.com/sql/t-sql/statements/create-xml-index-transact-sql).

Usługa Azure SQL Database zarządza bazami danych i gwarantuje ich wysoką dostępność. Niektóre funkcje, które mogą mieć wpływ na wysoką dostępność lub nie mogą być używane w świecie PaaS mają ograniczone funkcje w usłudze Azure SQL Database. Ponadto niektóre funkcje bazy danych zależą od typu usługi Azure SQL Database, który tworzysz. Te funkcje są opisane w poniższych tabelach. Za pomocą usługi Azure SQL Database można utworzyć bazę danych jako część [wystąpienia zarządzanego](sql-database-managed-instance.md), jako pojedynczej bazy danych lub jako część puli elastycznej. Jeśli potrzebujesz więcej szczegółów na temat różnic, można je znaleźć na oddzielnych stronach dla [pojedynczej bazy danych i pul elastycznych](sql-database-transact-sql-information.md) lub [wystąpienia zarządzanego](sql-database-managed-instance-transact-sql-information.md).

Za pomocą różnych metod migracji można przenosić dane między bazami danych programu SQL Server, Single Database i Managed Instance. Niektóre metody są **w trybie online** i pobrania wszystkich zmian, które są wprowadzane w źródle podczas uruchamiania migracji, podczas gdy w **trybie offline** metody należy zatrzymać obciążenie, które modyfikuje dane w źródle, gdy migracja jest w toku.
