# 仮想ネットワーク (VNet) で使用される DNS サーバーの管理

VNet で使用される DNS サーバーの一覧は、管理ポータルまたはネットワーク構成ファイルで管理できます。 VNet ごとに最大 12 台の DNS サーバーを追加できます。 DNS サーバーを指定する際に重要なのは、環境に適した順で DNS サーバーが一覧表示されていることを確認することです。 DNS サーバーの一覧はラウンド ロビンに対応していません。 指定した順序で使用されます。 一覧の先頭にある DNS サーバーに到達できる場合は、DNS サーバーが正しく動作しているかどうかに関係なく、クライアントはその DNS サーバーを使用します。 仮想ネットワーク用に DNS サーバーの順序を変更するには、DNS サーバーを一覧から削除して、希望の順序になるように再度追加します。

>[AZURE.WARNING] DNS の一覧が更新されると、新しい DNS サーバー設定するために、仮想ネットワークにある仮想マシンを再起動する必要があります。 仮想マシンは、再起動されるまで、現在の構成を引き続き使用します。

## 管理ポータルによる仮想ネットワークの DNS サーバー一覧の編集

1. ログオン、 **管理ポータル**します。

1. ナビゲーション ウィンドウでをクリックして **ネットワーク**, 、仮想ネットワークの名前をクリックして、 **名前** 列です。

1. クリックして **構成**します。

1.  **DNS サーバー**, 、次を構成することができます。

    - **サーバーの登録 (追加)、新しい DNS – に** のボックスに名前と IP アドレスを入力するだけです。 これにより、DNS サーバーは、仮想ネットワークの DNS サーバーの一覧に追加され、さらに Azure に登録されます。

    - **– 以前に登録された DNS サーバーを追加する** Azure で既に DNS サーバーが登録されている場合は、設定済みの一覧から選択できます。

    - **DNS サーバー – 仮想ネットワークから削除する** を削除するサーバーの横に X] をクリックします。 この操作では、仮想ネットワークの一覧からサーバーが削除されるだけです。 DNS サーバーは、他の仮想ネットワークで使用するために Azure に登録されたままになります。 サブスクリプションから DNS サーバーを削除するには、 **ネットワークの DNS サーバー]、[** ページです。

    - **DNS サーバーの順序変更 – に** 希望する順序で再度すべての DNS サーバーが表示され、それを追加で削除します。 ラウンド ロビン DNS の一覧ではないことに注意してください。

    - **DNS サーバー: 名前を変更する** の一覧で、DNS サーバーを強調表示し、新しい名前を入力します。 これにより、新しい DNS サーバーが Azure に登録されると同時に、仮想ネットワークの DNS サーバーの一覧にも追加されます。 古い DNS サーバーとその IP アドレスは Azure に登録されたままになります。 削除できます、 **DNS サーバー** ページに他の仮想ネットワークで使用する、しない場合。

1. クリックして **保存** 新しい DNS サーバーの構成を保存するページの下部にあります。

1. 仮想ネットワークにある仮想マシンを再起動して、新しい DNS の設定を取得できるようにします。

## ネットワーク構成ファイルによる DNS サーバー一覧の編集

ネットワーク構成ファイルを使用して DNS サーバーの一覧を編集するには、最初に管理ポータルから構成設定をエクスポートします。 次に、ネットワーク構成ファイルを編集し、管理ポータルでもう一度インポートします。 このプロセスを完了するための手順の概要は次のとおりです。

1. 仮想ネットワークの設定をネットワーク構成ファイルにエクスポートします。 詳細と、ネットワーク構成設定をエクスポートする手順については、次を参照してください。 [ネットワーク構成ファイルへの仮想ネットワーク設定のエクスポート](virtual-networks-using-network-configuration-file.md)します。

1. 仮想ネットワークの DNS サーバーの情報を指定します。 DNS サーバーを指定する方法の詳細については、次を参照してください。 [仮想ネットワーク構成ファイルで DNS サーバーを指定する](virtual-networks-specifying-a-dns-settings-in-a-virtual-network-configuration-file.md)です。 ネットワーク構成ファイルの詳細については、次を参照してください。 [Azure 仮想ネットワーク構成スキーマ](https://msdn.microsoft.com/library/azure/jj157100.aspx) と [ネットワーク構成ファイルを使用して仮想ネットワークを構成](virtual-networks-using-network-configuration-file.md)します。

1. ネットワーク構成ファイルをインポートします。 詳細と、ネットワーク構成ファイルをインポートする手順は、「 [ネットワーク構成ファイルをインポート](virtual-networks-using-network-configuration-file.md)します。

1. 仮想ネットワークにある仮想マシンを再起動して、新しい DNS の設定を取得できるようにします。
